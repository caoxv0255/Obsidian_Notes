## 📚 模块 1：背包问题模型——从物品选择到状态空间

### 两种基本模型

| 模型 | 定义 | 特点 |
|------|------|------|
| **01 背包** | 每种物品最多选一次（要么选 0 件，要么选 1 件） | 物品选择是二元的，状态转移时考虑"不选"或"选" |
| **完全背包** | 每种物品可以选无限次（只要不超过总容量） | 物品可以重复选取，状态转移时考虑"不选"或"选多次" |

### 状态设计核心

`dp[i][j]`：考虑前 i 件物品，在总容量为 j 的限制下，能够获得的最大价值（或能否达到某种状态）。

**01 背包状态转移：**
```
dp[i][j] = max(dp[i-1][j],                      // 不选第 i 件物品
               dp[i-1][j-weight[i]] + value[i]) // 选第 i 件物品
```

**完全背包状态转移：**
```
dp[i][j] = max(dp[i-1][j],                      // 不选第 i 件物品
               dp[i][j-weight[i]] + value[i])  // 选第 i 件物品（注意这里是 dp[i] 而非 dp[i-1]）
```

**关键区别**：完全背包中，因为物品可以重复选取，所以在选择第 i 件物品后，仍然可以继续考虑第 i 件物品（即 `dp[i][j-weight[i]]`）。

## 📚 模块 2：01 背包经典例题——LeetCode 416 分割等和子集

### 问题描述

LeetCode 416. 分割等和子集：给你一个只包含正整数的数组 `nums`，判断是否可以将数组分割成两个子集，使得两个子集的元素和相等。

### 问题转化

1. 计算数组总和 `total_sum`，奇数则直接返回 `false`
2. 目标：从数组中选出一个子集，使其和为 `target = total_sum // 2`
3. 这就变成了 01 背包问题：物品重量 = 元素值，背包容量 = target

### 代码实现（C++）

```cpp
class Solution {
public:
    bool canPartition(vector<int>& nums) {
        int total_sum = accumulate(nums.begin(), nums.end(), 0);
        if (total_sum % 2 != 0) return false;
        
        int target = total_sum / 2;
        vector<bool> dp(target + 1, false);
        dp[0] = true;
        
        for (int num : nums) {
            for (int j = target; j >= num; --j) {
                dp[j] = dp[j] || dp[j - num];
                if (dp[target]) return true;
            }
        }
        return dp[target];
    }
};
```

## 📚 模块 3：完全背包经典例题——LeetCode 494 目标和

### 问题描述

LeetCode 494. 目标和：给数组中每个整数前添加 `'+'` 或 `'-'`，构造表达式，求运算结果等于 target 的不同表达式数目。

### 问题转化

设添加 `'+'` 的数字之和为 `positive_sum`，添加 `'-'` 的数字之和为 `negative_sum`：
- `positive_sum - negative_sum = target`
- `positive_sum + negative_sum = total_sum`
- 解得：`positive_sum = (total_sum + target) / 2`

问题转化为：从 nums 中选择若干个数（每个数只能选一次），使得和恰好为 `positive_sum`，问有多少种方案。

### 代码实现（C++）

```cpp
class Solution {
public:
    int findTargetSumWays(vector<int>& nums, int target) {
        int total_sum = accumulate(nums.begin(), nums.end(), 0);
        if (abs(target) > total_sum || (total_sum + target) % 2 != 0) return 0;
        
        int positive_sum = (total_sum + target) / 2;
        vector<int> dp(positive_sum + 1, 0);
        dp[0] = 1;
        
        for (int num : nums) {
            for (int j = positive_sum; j >= num; --j) {
                dp[j] += dp[j - num];
            }
        }
        return dp[positive_sum];
    }
};
```

## 📚 模块 4：竞赛专项——CF 189A. Cut Ribbon

### 问题描述

Polycarpus 有一条长度为 n 的丝带，想剪成若干段，每段长度只能是 a、b 或 c，求最多能得到的段数。

### 问题分析

这是一个**完全背包**问题：
- 背包容量：丝带长度 n
- 物品：三种长度 a, b, c，每种可以选无限次
- 价值：每选一段，段数 +1
- 目标：恰好装满 n，最大化段数

### 代码实现（C++）

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    int len[3] = {0};
    cin >> n;
    for (int &num : len) cin >> num;
    
    vector<int> dp(n + 1, -1);
    dp[0] = 0;
    
    for (int num : len) {
        for (int j = num; j <= n; ++j) {
            if (dp[j - num] != -1) {
                if (dp[j] == -1) {
                    dp[j] = dp[j - num] + 1;
                } else {
                    dp[j] = max(dp[j], dp[j - num] + 1);
                }
            }
        }
    }
    
    if (dp[n] == -1) cout << 0 << endl;
    else cout << dp[n] << endl;
    return 0;
}
```

## 📚 模块 5：空间优化原理

### 为什么 01 背包倒序，完全背包正序？

- **01 背包倒序**：保证在第 i 轮循环中，`dp[j]` 使用的是上一轮（i-1）的 `dp[j - w]`，避免同一物品被重复选取
- **完全背包正序**：保证在第 i 轮循环中，`dp[j]` 使用的是本轮（i）的 `dp[j - w]`，即允许同一物品被重复选取

```
01背包（倒序）：
 dp[10] = max(dp[10], dp[10-w] + v)  // dp[10-w] 来自上一轮，物品 i 未被选过

完全背包（正序）：
 dp[10] = max(dp[10], dp[10-w] + v)  // dp[10-w] 可能已经在本轮被更新，即物品 i 可能已被选过
```

## ❓ 常见问题

**Q：01 背包和完全背包的区别仅仅是遍历顺序不同吗？**

A：不仅仅是遍历顺序，更重要的是**状态定义**。01 背包中 `dp[i][j]` 只依赖 `dp[i-1][...]`（上一行），而完全背包依赖 `dp[i][...]`（本行）。遍历顺序是实现这种依赖关系的一维数组优化技巧。

**Q：如何判断一个实际问题是否是背包问题？**

A：如果问题同时满足以下两个特征，很可能是背包问题：
- 涉及**选择**（选或不选、选几个）
- 有**容量限制**（总重量、总长度、总金额等）
- 求最值（最大价值、最多数量）或计数（多少种方法）
