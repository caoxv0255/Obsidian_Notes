### 模块1：序列问题概述——从子序列到状态设计

**子序列 vs 子数组**

| | 子序列 | 子数组 |
|--|--------|--------|
| 定义 | 删除某些元素后保持相对顺序 | 连续的一段 |
| 特点 | 元素可以不连续 | 必须连续 |

示例：序列 [2,5,3,7]：
- 子序列：[2,3,7]（非连续）
- 子数组：[5,3,7]（连续）

**DP状态设计思维**

- **LIS**：`dp[i]` = 以第 i 个元素结尾的最长递增子序列长度
- **LCS**：`dp[i][j]` = text1 前 i 字符与 text2 前 j 字符的最长公共子序列长度

**与背包问题的对比**

| 维度 | 背包问题 | 序列问题 |
|------|---------|---------|
| 状态维度 | 一维/二维 | 一维/二维 |
| 遍历顺序 | 倒序(01)/正序(完全) | 正序 |
| 状态意义 | 容量约束最优选择 | 序列匹配或排序最优 |

### 模块2：最长递增子序列（LIS）——LeetCode 300

**问题**：给你一个整数数组 nums，找到其中最长严格递增子序列的长度。

**核心思想**：以每个元素结尾的最长递增子序列，通过考察该元素之前的所有元素来构建。

**状态设计**：`dp[i]` = 以 `nums[i]` 结尾的最长递增子序列长度

**状态转移**：
```cpp
for (int j = 0; j < i; j++) {
    if (nums[j] < nums[i]) {
        dp[i] = max(dp[i], dp[j] + 1);
    }
}
```

**初始化**：`dp[i] = 1`（每个元素自身至少长度为1）

**复杂度**：O(n²) 时间，O(n) 空间

**示例推导**（nums = [10,9,2,5,3,7,101,18]）：

| i | nums[i] | dp[i] | 说明 |
|---|---------|-------|------|
| 0 | 10 | 1 | 初始化 |
| 1 | 9 | 1 | 9 < 10，不能接 |
| 2 | 2 | 1 | 2 < 10,9 |
| 3 | 5 | 2 | 5 > 2，dp[2]+1=2 |
| 4 | 3 | 2 | 3 > 2，dp[2]+1=2 |
| 5 | 7 | 3 | 7 > 5,3，dp[3]+1=3 |
| 6 | 101 | 4 | 101 > 7，dp[5]+1=4 |
| 7 | 18 | 4 | 18 > 7，dp[5]+1=4 |

### 模块3：最长公共子序列（LCS）——LeetCode 1143

**问题**：给定两个字符串 text1 和 text2，返回这两个字符串的最长公共子序列的长度。

**核心思想**：如果当前字符相等 → LCS 长度 = 对角线 + 1；如果不相等 → 取左边和上边的最大值。

**状态设计**：`dp[i][j]` = text1 前 i 个字符与 text2 前 j 个字符的 LCS 长度

**状态转移**：
```cpp
if (text1[i-1] == text2[j-1]) {
    dp[i][j] = dp[i-1][j-1] + 1;
} else {
    dp[i][j] = max(dp[i-1][j], dp[i][j-1]);
}
```

**初始化**：`dp[0][j] = 0`，`dp[i][0] = 0`

**示例推导**（text1 = "abcde", text2 = "ace"）：

| | "" | a | c | e |
|--|----|---|---|---|
| "" | 0 | 0 | 0 | 0 |
| a | 0 | 1 | 1 | 1 |
| b | 0 | 1 | 1 | 1 |
| c | 0 | 1 | 2 | 2 |
| d | 0 | 1 | 2 | 2 |
| e | 0 | 1 | 2 | 3 |

最终 `dp[5][3] = 3`

### 模块4：优化思路（可选）

**LIS的O(n log n)优化：贪心+二分**

维护 `tails[k]` = 长度为 k+1 的递增子序列的最小尾部元素。遍历 nums：大于末尾则追加，否则二分替换。

```cpp
vector<int> tails;
for (int num : nums) {
    if (tails.empty() || num > tails.back()) {
        tails.push_back(num);
    } else {
        auto it = lower_bound(tails.begin(), tails.end(), num);
        *it = num;
    }
}
return tails.size();
```

**LCS的空间优化：滚动数组**

`dp[i][j]` 只依赖 `dp[i-1][j-1]`、`dp[i-1][j]`、`dp[i][j-1]`，用两行数组滚动即可将空间降到 O(min(m,n))。

## 思考

1. **为什么 LCS 用二维 dp[i][j] 而不是一维？**
   - 因为需要比较 text1 的前 i 字符和 text2 的前 j 字符，匹配状态需要两个维度
   - 一维会丢失"前缀"信息

2. **为什么 LIS 可以用二分优化，但 LCS 不行？**
   - LIS 的贪心策略：维护最小尾部数组 tails，可以在 O(n log n) 找到最长长度
   - LCS 是两个字符串的匹配问题，无法用单调性优化，只能用滚动数组省空间

3. **背包问题和序列问题的本质区别？**
   - 背包：选择物品，每个物品只用一次（0-1）或无数次（完全）
   - 序列：在序列上逐步扩展状态，LIS 看单个数组内递增关系，LCS 看两个数组的匹配关系
