## 💡 学习与收获

### 核心收获（3个）
1. **双栈实现队列的核心思想**：使用输入栈（inStack）和输出栈（outStack）分工协作。push直接压入输入栈，pop/peek时若输出栈为空，则将输入栈所有元素转移到输出栈（这样栈底元素变成栈顶，实现FIFO）。关键优势：平摊O(1)时间复杂度，每个元素最多转移一次
2. **队列旋转法实现栈**：push时将新元素入队后，旋转队列n次（将队头元素重新入队），使新元素成为队头（即栈顶）。这个旋转过程本质上是调整队列顺序，模拟栈的LIFO特性。pop/top直接操作队头即可，时间复杂度：push O(n)，pop/top O(1)
3. **最小栈的辅助栈法**：使用两个栈同步维护——数据栈存储所有元素，最小栈存储当前最小值。push时若新值≤最小栈顶则压入新值，否则压入当前最小值（重复压入保证同步）。所有操作O(1)，空间换时间的经典优化

### 疑问与思考（1个）
- 在双栈实现队列中，为什么pop/peek是"平摊O(1)"而非"最坏O(n)"？理解了平摊分析：虽然某次pop可能触发O(n)转移，但每个元素在整个生命周期中最多被转移一次，因此分摊到每个操作的平均成本是O(1)

### 感悟与总结（1个）
- 栈和队列的相互实现是数据结构设计的经典案例，展现了"用基础结构构建高级结构"的思想。通过双栈分工、队列旋转等技巧，可以在不改变底层实现的前提下，改变数据的访问模式（LIFO ↔ FIFO）。最小栈则展示了"辅助数据结构"的威力——用额外空间换取时间性能，这是算法优化的重要范式

### 重点知识点
- **双栈法实现队列**：输入栈（inStack）接收新元素，输出栈（outStack）弹出旧元素。关键转移：outStack空时将inStack全部转移到outStack
- **队列旋转法实现栈**：push后旋转队列n次，使新元素成为队头。每次push O(n)，pop/top O(1)
- **最小栈-辅助栈法**：两个栈同步维护，数据栈存储原始值，最小栈存储当前最小值。所有操作O(1)，空间O(n)
- **最小栈-差值存储法**：存储当前值与历史最小值的差值，用long避免溢出。空间O(n)，但实现较复杂
- **最小栈-单栈配对法**：用stack<pair<int, int>>存储{原始值, 当前最小值}，初始化时压入{0, INT_MAX}哨兵
- **emplace() 与 push() 的区别**：emplace() 在容器中直接构造元素，避免临时对象的创建和拷贝/移动；push() 先构造临时对象，再拷贝/移动到容器中。emplace() 通常性能更优（特别是对于复杂类型），且可以直接传递构造函数参数（如 stk.emplace(val, min(getMin(), val))）。对于基本类型（如 int），两者性能差异可以忽略
- **平摊时间复杂度**：每个元素最多被转移一次，因此平均到每个操作是O(1)。即使某次操作是O(n)，但平摊后是O(1)
- **空间复杂度**：双栈队列O(n)，队列栈O(n)，最小栈辅助栈法O(2n)，差值存储法O(n)更优但复杂

```cpp
class MyQueue {
public:
    stack<int> s1;  // 输入栈
    stack<int> s2;  // 输出栈
    MyQueue() {}

    void push(int x) { s1.push(x); }

    int pop() {
        if (s2.empty()) {
            while (!s1.empty()) {
                s2.push(s1.top());
                s1.pop();
            }
        }
        int num = s2.top();
        s2.pop();
        return num;
    }

    int peek() {
        int num = this->pop();
        s2.push(num);
        return num;
    }

    bool empty() { return s1.empty() && s2.empty(); }
};
```

**LeetCode 155 - 最小栈（辅助栈法 - 标准解）**
```cpp
class MinStack {
public:
    stack<int> data;  // 数据栈
    stack<int> min;   // 最小栈
    MinStack() {}

    void push(int val) {
        if (min.empty() || min.top() >= val) {
            min.push(val);
        } else {
            min.push(min.top());
        }
        data.push(val);
    }

    void pop() {
        min.pop();
        data.pop();
    }

    int top() { return data.top(); }

    int getMin() { return min.top(); }
};
```

**LeetCode 155 - 最小栈（差值存储法 - 空间优化）**
```cpp
class MinStack {
public:
    stack<long> diff;
    long minval;
    MinStack() {}

    void push(int val) {
        if (diff.empty()) {
            diff.push(0);
            minval = val;
        } else {
            long diffval = static_cast<long>(val - minval);
            diff.push(diffval);
            if (diffval < 0) {
                minval = val;
            }
        }
    }

    void pop() {
        if (diff.empty()) {
            return;
        }
        long diffval = diff.top();
        diff.pop();
        if (diffval < 0) {
            minval = minval - diffval;
        }
    }

    int top() {
        if (diff.empty()) {
            return -1;
        }
        long diffval = diff.top();
        if (diffval >= 0) {
            return static_cast<int>(minval + diffval);
        } else {
            return static_cast<int>(minval);
        }
    }

    int getMin() { return static_cast<int>(minval); }
};
```

|------|------------------------|------------------------|------------------------|

**平摊时间复杂度解释**：
- LeetCode 232的pop/peek虽然可能触发O(n)转移，但每个元素最多被转移一次，因此平摊O(1)
- 类比： amortized analysis，类似动态数组的扩容，最坏情况偶发但平均成本低

- LeetCode 225：O(n)，一个队列存储n个元素

### Python实现对比

**LeetCode 232 - 用栈实现队列（Python版）**
```python
from collections import deque

class MyQueue:
    def __init__(self):
        self.in_stack = []
        self.out_stack = []

    def push(self, x: int) -> None:
        self.in_stack.append(x)

    def pop(self) -> int:
        self._transfer_if_needed()
        return self.out_stack.pop()

    def peek(self) -> int:
        self._transfer_if_needed()
        return self.out_stack[-1]

    def empty(self) -> bool:
        return len(self.in_stack) == 0 and len(self.out_stack) == 0

    def _transfer_if_needed(self):
        if not self.out_stack:
            while self.in_stack:
                self.out_stack.append(self.in_stack.pop())
```

**LeetCode 225 - 用队列实现栈（Python版）**
```python
from collections import deque

class MyStack:
    def __init__(self):
        self.q = deque()

    def push(self, x: int) -> None:
        n = len(self.q)
        self.q.append(x)
        for _ in range(n):
            self.q.append(self.q.popleft())

    def pop(self) -> int:
        return self.q.popleft()

    def top(self) -> int:
        return self.q[0]

    def empty(self) -> bool:
        return len(self.q) == 0
```

**LeetCode 155 - 最小栈（Python版）**
```python
class MinStack:
    def __init__(self):
        self.data_stack = []
        self.min_stack = []

    def push(self, val: int) -> None:
        self.data_stack.append(val)
        if not self.min_stack or val <= self.min_stack[-1]:
            self.min_stack.append(val)
        else:
            self.min_stack.append(self.min_stack[-1])

    def pop(self) -> None:
        self.data_stack.pop()
        self.min_stack.pop()

    def top(self) -> int:
        return self.data_stack[-1]

    def getMin(self) -> int:
        return self.min_stack[-1]
```
