## 📚 模块1：LeetCode 148 排序链表——归并排序的链表实现

### 问题描述

给你链表的头结点 head，请将其按升序排列并返回排序后的链表。要求时间复杂度 O(n log n)，空间复杂度 O(1)（递归栈空间不计入）。

### 核心挑战：链表的分治操作

与数组不同，链表无法随机访问，因此需要：
- **找到链表中点**：快慢指针法
- **分割链表**：将链表从中点处断开
- **递归排序**：分别排序左右两半链表
- **合并有序链表**：迭代合并两个已排序链表

### 完整代码模板（C++）

```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    ListNode* sortList(ListNode* head) {
        // 递归终止条件：空节点或单个节点
        if (head == nullptr || head->next == nullptr) {
            return head;
        }
        
        // 步骤1：使用快慢指针找到链表中点
        ListNode* slow = head;
        ListNode* fast = head->next;
        while (fast != nullptr && fast->next != nullptr) {
            slow = slow->next;
            fast = fast->next->next;
        }
        
        // 步骤2：分割链表（将链表从中点后断开）
        ListNode* mid = slow->next;
        slow->next = nullptr;
        
        // 步骤3：递归排序左右两半
        ListNode* left = sortList(head);
        ListNode* right = sortList(mid);
        
        // 步骤4：合并两个有序链表
        return mergeTwoLists(left, right);
    }
    
private:
    // 合并两个有序链表（迭代法）
    ListNode* mergeTwoLists(ListNode* l1, ListNode* l2) {
        ListNode dummy(0);  // 虚拟头节点，简化边界处理
        ListNode* tail = &dummy;
        
        while (l1 != nullptr && l2 != nullptr) {
            if (l1->val <= l2->val) {
                tail->next = l1;
                l1 = l1->next;
            } else {
                tail->next = l2;
                l2 = l2->next;
            }
            tail = tail->next;
        }
        
        // 拼接剩余部分
        tail->next = (l1 != nullptr) ? l1 : l2;
        return dummy.next;
    }
};
```

### 逐步调试示例（以链表 4→2→1→3 为例）

**初始链表**：4 → 2 → 1 → 3

**第一层递归 sortList(4→2→1→3)**
- 快慢指针找中点：slow从4开始，fast从2开始
  - 第一轮：slow=2, fast=1
  - 第二轮：slow=1, fast=3
  - 第三轮：fast=nullptr，停止
- 中点：slow=1，即节点值为1的节点
- 分割链表：mid = slow->next = 3，断开slow->next = nullptr
- 左半链表：4→2→1
- 右半链表：3
- 递归调用：
  - left = sortList(4→2→1)
  - right = sortList(3) → 直接返回3（单个节点）

**回溯合并过程**：
- 第三层：合并2→4和1 → 1→2→4
- 第二层：合并1→2→4和3 → 1→2→3→4

**最终结果**：1 → 2 → 3 → 4

### 复杂度分析

| 维度 | 链表归并排序 | 数组归并排序 |
|------|-------------|-------------|
| 时间复杂度 | O(n log n) | O(n log n) |
| 空间复杂度 | O(1)（仅指针变量） | O(n)（需要临时数组） |
| 递归栈深度 | O(log n) | O(log n) |
| 稳定性 | 稳定 | 稳定 |

**关键认知**：链表归并排序的空间复杂度优势在于合并时只需调整指针，无需额外数组。这是链表相比数组在排序上的一个重要优势。

## ⚡ 模块2：LeetCode 215 数组中的第K个最大元素——快速选择算法

### 问题描述

给定整数数组 nums 和整数 k，请返回数组中第 k 个最大的元素。要求时间复杂度 O(n)。

### 核心思想：快速选择（QuickSelect）

快速选择是快速排序的变种，只递归处理包含目标元素的那一侧，从而将平均时间复杂度从 O(n log n) 降低到 O(n)。

**算法步骤**：
1. 随机选择一个基准值（pivot）
2. 分区操作：将数组分为小于基准、等于基准、大于基准三部分
3. 判断第k大元素所在区间：
   - 如果在大于基准的区域，递归处理该区域
   - 如果在小于基准的区域，递归处理该区域并调整k值
   - 如果在等于基准的区域，直接返回基准值

### 完整代码模板（C++）

```cpp
#include <vector>
#include <cstdlib>  // for rand()
#include <ctime>    // for time()
using namespace std;

class Solution {
public:
    int findKthLargest(vector<int>& nums, int k) {
        // 第k大等价于第 (nums.size() - k) 小（0-based）
        return quickSelect(nums, 0, nums.size() - 1, nums.size() - k);
    }
    
private:
    // 快速选择主函数
    int quickSelect(vector<int>& nums, int left, int right, int k_smallest) {
        if (left == right) {
            return nums[left];
        }
        
        // 随机选择基准值，避免最坏情况
        int pivotIndex = left + rand() % (right - left + 1);
        pivotIndex = partition(nums, left, right, pivotIndex);
        
        if (k_smallest == pivotIndex) {
            return nums[k_smallest];
        } else if (k_smallest < pivotIndex) {
            // 目标在左半部分
            return quickSelect(nums, left, pivotIndex - 1, k_smallest);
        } else {
            // 目标在右半部分
            return quickSelect(nums, pivotIndex + 1, right, k_smallest);
        }
    }
    
    // 分区函数（Lomuto变种，返回基准值的最终位置）
    int partition(vector<int>& nums, int left, int right, int pivotIndex) {
        int pivotValue = nums[pivotIndex];
        // 将基准值交换到最右端
        swap(nums[pivotIndex], nums[right]);
        
        int storeIndex = left;  // 存储小于基准值的区域边界
        
        for (int i = left; i < right; i++) {
            if (nums[i] < pivotValue) {
                swap(nums[storeIndex], nums[i]);
                storeIndex++;
            }
        }
        
        // 将基准值放回正确位置
        swap(nums[storeIndex], nums[right]);
        return storeIndex;
    }
};
```

### 逐步调试示例（以数组 [3,2,1,5,6,4], k=2 为例）

**问题**：找第2大的元素（即第5小的元素，因为n=6，k_smallest=6-2=4）

**第一轮**：
- 随机选择pivotIndex，假设选到3（值5）
- 分区操作后：[3,2,1,4,5,6]
- pivotIndex = 4（基准值5的最终位置）
- 判断：k_smallest=4 == pivotIndex=4 → 返回nums[4]=5

**结果**：第2大的元素是5

### 快速选择 vs 堆排序（优先队列）对比

| 算法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| 快速选择 | 平均O(n)，最坏O(n²) | 平均O(log n)递归栈 | 随机数据，需要原地修改 |
| 堆排序 | O(n log k) | O(k)或O(n) | 数据流，不需要修改原数组 |
| 排序后取第k个 | O(n log n) | O(1)或O(n) | k接近n/2，且允许排序 |

**工程选择建议**：
- 如果数据可修改且需要最优平均性能 → 快速选择（随机化基准）
- 如果数据不可修改或需要稳定最坏情况 → 堆排序（维护大小为k的最小堆）
- 如果k接近n/2且允许O(n log n) → 直接排序取第k个

## 🔄 模块3：分治思想在链表与数组上的应用对比

### 实现差异总结

| 维度 | 链表实现 | 数组实现 |
|------|---------|---------|
| 中点查找 | 快慢指针O(n) | 索引计算O(1) |
| 分割操作 | 修改指针O(1) | 计算边界O(1) |
| 合并操作 | 调整指针O(n) | 需要临时数组O(n) |
| 空间复杂度 | O(1)额外空间 | O(n)或O(log n)递归栈 |
| 缓存友好性 | 差（非连续内存） | 好（连续内存） |

### 分治思想的通用模式

**分治算法框架**：
1. **分解**：将原问题分解为若干子问题
   - 链表：快慢指针找中点，断开链表
   - 数组：计算中间索引，划分区间
2. **解决**：递归解决子问题
   - 终止条件：子问题规模为1（天然有序）
3. **合并**：将子问题的解合并为原问题的解
   - 链表：合并两个有序链表（迭代）
   - 数组：合并两个有序子数组（需要临时空间）

**关键认知**：分治思想的威力在于问题分解的标准化。一旦掌握"分-治-合"的通用框架，就能将其迁移到不同数据结构上，只需调整具体实现细节。

## 📝 模块4：自测题——代码补全与应用场景分析

### 第一部分：代码补全题（快速选择分区优化）

**三向分区（Dutch National Flag）优化**：

```cpp
void threeWayPartition(vector<int>& nums, int left, int right, int pivotValue,
                      int& lessEnd, int& greaterStart) {
    int i = left;
    lessEnd = left - 1;
    greaterStart = right + 1;
    
    while (i < greaterStart) {
        if (nums[i] < pivotValue) {
            lessEnd++;
            swap(nums[i], nums[lessEnd]);
            i++;
        } else if (nums[i] == pivotValue) {  // 填空1：判断等于基准值
            i++;           // 填空2：移动i指针
        } else {  // nums[i] > pivotValue
            greaterStart--;
            swap(nums[i], nums[greaterStart]);  // 填空3：交换元素
            // 注意：交换后i不增加，需要再次判断交换来的值
        }
    }
}
```

### 第二部分：应用场景分析

**场景1**：单向链表存储10万个学生成绩（0-100整数），按成绩升序排序
- **选择**：归并排序
- **理由**：时间复杂度O(n log n)满足需求，空间复杂度O(1)（链表归并只需调整指针），稳定性重要

**场景2**：大小为100万的整数数组，找出第5000大的元素
- **选择**：快速选择算法（随机化基准）
- **理由**：平均时间复杂度O(n)最快，空间复杂度O(log n)递归栈可接受

**场景3**：大量重复元素（0,1,2三种值），找第k大的元素
- **选择**：三向分区快速选择
- **理由**：将数组分为小于、等于、大于三部分，避免对重复元素进行不必要的递归

## 📝 模块5：快速选择算法复杂度分析与优化

### 时间复杂度深度分析

**平均情况O(n)的数学直觉**：

快速选择每次递归只处理一侧子问题。假设每次分区大致均匀：
- 第一次分区：处理n个元素
- 第二次分区：处理约n/2个元素
- 第三次分区：处理约n/4个元素
- ...

总工作量：n + n/2 + n/4 + ... ≈ 2n = O(n)

**最坏情况O(n²)**：

当每次分区都极不均匀（如最小或最大元素总是被选为基准），递归树退化为链表：
- 第一次分区：处理n个元素
- 第二次分区：处理n-1个元素
- ...
- 第n次分区：处理1个元素

总工作量：n + (n-1) + ... + 1 = n(n+1)/2 = O(n²)

### 优化策略

| 策略 | 原理 | 实现复杂度 | 效果 |
|------|------|-----------|------|
| 随机化基准 | 随机选择基准，避免固定模式导致最坏情况 | 简单 | 将最坏情况转化为低概率事件 |
| 三数取中 | 取左、中、右三个元素的中值作为基准 | 中等 | 进一步降低不平衡分区概率 |
| 三向分区 | 将数组分为小于、等于、大于三部分 | 中等 | 对重复元素多的场景大幅优化 |
| 切换到插入排序 | 当子数组规模小于阈值（如16）时使用插入排序 | 简单 | 减少递归深度，提高缓存友好性 |
| 尾递归优化 | 递归处理较大一侧，循环处理较小一侧 | 中等 | 将递归栈深度限制为O(log n) |

**工程实现建议**：
- **必做优化**：随机化基准（简单有效）
- **推荐优化**：当k较小时使用堆排序（稳定O(n log k)），当k接近n时使用快速选择
- **进阶优化**：三向分区（如果数据重复率高）
