## 📚 模块1：LeetCode 46 全排列——排列问题的回溯框架

### 问题描述

给定一个不含重复数字的整数数组 nums，返回其所有可能的全排列。你可以按任意顺序返回答案。

### 核心思想：交换法生成全排列

排列问题的本质是顺序敏感：[1,2,3] 与 [1,3,2] 是不同的排列。经典的回溯解法通过"交换元素"来生成所有排列：

- 第一层递归：依次将每个元素交换到第一个位置
- 后续递归：对剩余子数组继续执行相同操作
- 状态恢复：每次递归返回后，交换回原状态，保证其他分支的正确性

### 完整代码模板（C++）

```cpp
#include <vector>
using namespace std;

class Solution {
public:
    vector<vector<int>> permute(vector<int>& nums) {
        vector<vector<int>> result;
        backtrack(nums, 0, result);
        return result;
    }
    
private:
    void backtrack(vector<int>& nums, int start, vector<vector<int>>& result) {
        // 终止条件：已生成一个完整排列
        if (start == nums.size()) {
            result.push_back(nums);
            return;
        }
        
        // 遍历所有可能的交换位置
        for (int i = start; i < nums.size(); i++) {
            // 选择：将 nums[i] 交换到当前位置 start
            swap(nums[start], nums[i]);
            
            // 递归：生成剩余部分的排列
            backtrack(nums, start + 1, result);
            
            // 撤销：恢复原状态
            swap(nums[start], nums[i]);
        }
    }
};
```

### 递归树可视化解释（前两层）

以 nums = [1,2,3] 为例，画出前两层递归树：

```
第一层 (start=0)
       [1,2,3]   ← 初始状态
         |
         +--- i=0: 交换 nums[0]⇔nums[0]（不改变）
         |       [1,2,3]
         |       |
         |       +--- 递归进入第二层 (start=1)
         |
         +--- i=1: 交换 nums[0]⇔nums[1]
         |       [2,1,3]
         |       |
         |       +--- 递归进入第二层 (start=1)
         |
         +--- i=2: 交换 nums[0]⇔nums[2]
                 [3,2,1]
                 |
                 +--- 递归进入第二层 (start=1)

第二层 (start=1)
以第一层的 [1,2,3] 分支为例：
       [1,2,3]   ← 当前状态
         |
         +--- i=1: 交换 nums[1]⇔nums[1]（不改变）
         |       [1,2,3]
         |       |
         |       +--- 递归进入第三层 (start=2)
         |
         +--- i=2: 交换 nums[1]⇔nums[2]
                 [1,3,2]
                 |
                 +--- 递归进入第三层 (start=2)
```

### 关键认知

- 每个节点表示当前的部分排列（nums[0..start-1] 已固定）
- 每条边表示一次交换操作
- 叶子节点（start == nums.size()）即为一个完整排列
- 树的高度 = 数组长度，每个节点的分支数递减（第一层n个分支，第二层n-1个分支...）

### 时间复杂度分析

设数组长度为 n：

- 递归树节点总数：n! + n!/(1!) + n!/(2!) + ... + 1 ≈ e × n!（其中 e ≈ 2.718）
- 每个叶子节点对应一个排列，共 n! 个
- **时间复杂度**：O(n × n!)
  - 生成每个排列需要 O(n) 时间（复制或交换）
  - 共 n! 个排列
- **空间复杂度**：O(n)（递归栈深度）

**零压力提示**：掌握交换法模板即达标，复杂度分析为可选深化内容。

## 📚 模块2：LeetCode 78 子集——组合问题的回溯框架

### 问题描述

给你一个整数数组 nums，数组中的元素互不相同。返回该数组所有可能的子集（幂集）。解集不能包含重复的子集。

### 核心思想：选择/不选择法生成子集

组合问题的本质是顺序不敏感：[1,2] 与 [2,1] 被视为相同的子集。经典的回溯解法通过"选择或不选择当前元素"来生成所有子集：

- 每个元素有两种选择：加入当前子集，或不加入
- 递归树是二叉树：每个节点对应一个决策点
- 路径记录：从根到叶子的路径即为一个子集

### 完整代码模板（C++）

```cpp
#include <vector>
using namespace std;

class Solution {
public:
    vector<vector<int>> subsets(vector<int>& nums) {
        vector<vector<int>> result;
        vector<int> current;
        backtrack(nums, 0, current, result);
        return result;
    }
    
private:
    void backtrack(vector<int>& nums, int index, vector<int>& current, vector<vector<int>>& result) {
        // 终止条件：已处理完所有元素
        if (index == nums.size()) {
            result.push_back(current);
            return;
        }
        
        // 选择1：不包含当前元素
        backtrack(nums, index + 1, current, result);
        
        // 选择2：包含当前元素
        current.push_back(nums[index]);
        backtrack(nums, index + 1, current, result);
        current.pop_back(); // 撤销选择
    }
};
```

### 递归树可视化解释（前两层）

以 nums = [1,2,3] 为例，画出前两层递归树：

```
第一层 (index=0)
        []
         |
         +--- 不选 nums[0] → []
         |       |
         |       +--- 递归进入第二层 (index=1)
         |
         +--- 选 nums[0] → [1]
                 |
                 +--- 递归进入第二层 (index=1)

第二层 (index=1)
以第一层的 [] 分支为例：
        []
         |
         +--- 不选 nums[1] → []
         |       |
         |       +--- 递归进入第三层 (index=2)
         |
         +--- 选 nums[1] → [2]
                 |
                 +--- 递归进入第三层 (index=2)
```

### 关键认知

- 每个节点表示当前已做的决策（哪些元素被选中）
- 每条边表示一个决策（选或不选）
- 叶子节点（index == nums.size()）即为一个完整子集
- 树的高度 = 数组长度，每个节点有2个分支（选/不选）
- 总叶子数 = 2^n（对应2^n个子集）

## 📚 模块3：排列与组合问题的递归树差异对比

| 维度 | 排列问题（LeetCode 46） | 组合问题（LeetCode 78） |
|-----|------------------------|----------------------|
| 决策类型 | 交换位置 | 选择/不选择 |
| 顺序敏感性 | 敏感（[1,2]≠[2,1]） | 不敏感（[1,2]=[2,1]） |
| 递归树结构 | 多叉树，分支数递减 | 二叉树，每个节点固定2分支 |
| 叶子节点数 | n!（阶乘） | 2^n（指数） |
| 典型时间复杂度 | O(n × n!) | O(n × 2^n) |
| 状态记录方式 | 修改原数组（交换法） | 维护当前路径（current向量） |

### 工程建议

- **排列问题**：优先掌握交换法，理解状态恢复的必要性
- **组合问题**：优先掌握选择/不选择法，理解路径记录的通用性
- **两者共性**：都需要明确终止条件、选择列表、递归调用、状态恢复

## 📚 模块4：通用回溯模板——路径、选择列表、结束条件

### 三要素框架

所有回溯问题都可以抽象为三个核心要素：

- **路径（path）**：已做出的选择序列，对应递归树中从根到当前节点的路径
- **选择列表（choices）**：当前可做的选择集合
- **结束条件（termination）**：满足该条件时，将当前路径加入结果集

### 通用代码模板（C++）

```cpp
// 伪代码框架
void backtrack(路径, 选择列表, 结果集) {
    if (满足结束条件) {
        结果集.push_back(路径);
        return;
    }
    
    for (选择 in 选择列表) {
        // 剪枝：跳过无效选择（可选）
        if (选择无效) continue;
        
        // 做选择
        路径.push_back(选择);
        从选择列表中移除该选择; // 或标记为已使用
        
        // 递归
        backtrack(路径, 选择列表, 结果集);
        
        // 撤销选择
        从选择列表中恢复该选择; // 或标记为未使用
        路径.pop_back();
    }
}
```

### 在LeetCode 46与78中的应用

| 要素 | LeetCode 46（排列） | LeetCode 78（子集） |
|-----|---------------------|---------------------|
| 路径 | nums[0..start-1]（已固定的前缀） | current向量（当前选中的元素） |
| 选择列表 | nums[start..n-1]（可交换到位置start的元素） | {包含nums[index], 不包含nums[index]}（二选一） |
| 结束条件 | start == nums.size()（所有位置已固定） | index == nums.size()（所有元素已处理） |
| 做选择 | swap(nums[start], nums[i]) | current.push_back(nums[index]) 或 直接递归（不选） |
| 撤销选择 | swap(nums[start], nums[i])（恢复） | current.pop_back()（如选择包含） |

**关键认知**：掌握三要素框架后，可以系统分析任何回溯问题，快速确定实现思路。

## 📚 模块5：剪枝策略深化——重复元素剪枝

### 问题升级：LeetCode 47 全排列 II

**问题描述**：给定一个可包含重复数字的整数数组 nums，返回所有不重复的全排列。

### 核心挑战：避免重复排列

当数组包含重复元素时，简单的交换法会产生重复排列。例如 nums = [1,1,2]：

- 交换 nums[0] 与 nums[1]（两个1交换）会产生相同的排列
- 需要额外的剪枝策略来跳过重复分支

### 剪枝策略：先排序后跳过

- **预处理**：对数组排序，使相同元素相邻
- **剪枝条件**：当 nums[i] 与 nums[start] 交换时，如果 nums[i] 在区间 [start, i-1] 中出现过，则跳过
- **实现**：使用哈希表记录，或利用排序特性判断

### 完整代码模板（C++，含剪枝）

```cpp
#include <vector>
#include <algorithm>
using namespace std;

class Solution {
public:
    vector<vector<int>> permuteUnique(vector<int>& nums) {
        vector<vector<int>> result;
        sort(nums.begin(), nums.end()); // 关键：先排序
        backtrack(nums, 0, result);
        return result;
    }
    
private:
    void backtrack(vector<int>& nums, int start, vector<vector<int>>& result) {
        if (start == nums.size()) {
            result.push_back(nums);
            return;
        }
        
        for (int i = start; i < nums.size(); i++) {
            // 剪枝：跳过重复元素
            if (i != start && nums[i] == nums[start]) {
                continue;
            }
            
            // 更精确的剪枝：检查 nums[i] 是否在 [start, i-1] 中出现过
            bool duplicate = false;
            for (int j = start; j < i; j++) {
                if (nums[j] == nums[i]) {
                    duplicate = true;
                    break;
                }
            }
            if (duplicate) continue;
            
            swap(nums[start], nums[i]);
            backtrack(nums, start + 1, result);
            swap(nums[start], nums[i]);
        }
    }
};
```

### 剪枝效果分析

以 nums = [1,1,2] 为例，无剪枝时递归树：

- 第一层3个分支（i=0,1,2），产生重复
- 最终排列数：3! = 6，但实际不重复排列只有3个：[1,1,2], [1,2,1], [2,1,1]

**剪枝后**：

- 当 start=0，i=1 时（两个1交换），剪枝跳过
- 递归树分支减少，效率提升

**时间复杂度**：最坏仍为 O(n × n!)，但平均情况因剪枝大幅改善。

**零压力提示**：LeetCode 47为可选挑战题，掌握基础剪枝思路即可。

## 📚 模块6：强度渐进——记忆化搜索概念铺垫

### 从回溯到动态规划的桥梁

回溯法通过穷举所有可能路径解决问题，但当问题存在重叠子问题时，会产生大量重复计算。例如斐波那契数列：

- 计算 fib(5) 需要 fib(4) 和 fib(3)
- 计算 fib(4) 又需要 fib(3) 和 fib(2)
- fib(3) 被重复计算多次

### 记忆化搜索（Memoization）基本思想

- **记录已计算结果**：使用数组或哈希表存储子问题的解
- **避免重复计算**：再次遇到相同子问题时，直接返回存储的结果
- **递归+缓存 = 记忆化搜索**

### 简单示例：斐波那契数列的记忆化实现

```cpp
#include <vector>
using namespace std;

class Fibonacci {
public:
    int fib(int n) {
        vector<int> memo(n + 1, -1);
        return helper(n, memo);
    }
    
private:
    int helper(int n, vector<int>& memo) {
        if (n <= 1) return n;
        
        // 如果已计算过，直接返回
        if (memo[n] != -1) return memo[n];
        
        // 计算并缓存结果
        memo[n] = helper(n - 1, memo) + helper(n - 2, memo);
        return memo[n];
    }
};
```

### 时间复杂度对比

| 方法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|-----|-----------|-----------|---------|
| 朴素递归 | O(2^n) | O(n) | 小规模问题 |
| 记忆化搜索 | O(n) | O(n) | 存在重叠子问题 |
| 动态规划 | O(n) | O(n)（可优化为O(1)） | 大多数优化问题 |

### 与回溯问题的关联

回溯问题中，如果不同路径会到达相同的状态（如相同的剩余元素集合），则存在重叠子问题，可考虑记忆化优化。例如：

- **排列问题**：不同交换顺序可能产生相同的剩余元素集合（当有重复元素时）
- **组合问题**：不同选择顺序可能产生相同的剩余元素集合

**关键认知**：记忆化搜索是回溯法与动态规划之间的过渡技术，通过缓存中间结果避免重复计算，为后续动态规划专题做准备。

**零压力提示**：本节仅需理解记忆化搜索的基本概念，知道其存在价值即可，具体实现将在动态规划专题深入学习。

- 顺序敏感性：排列顺序敏感（[1,2]≠[2,1]），子集顺序不敏感（[1,2]=[2,1]）
- 递归树结构：排列是多叉树（分支数递减），子集是二叉树（每个节点固定2分支）
- 结果数量：排列有n!个，子集有2^n个
- 状态记录：排列常修改原数组（交换法），子集常维护当前路径（push/pop）

**子集剪枝思路**：
- 前提：如果问题附加约束（如子集和≤某值）
- 条件：当当前子集和 + 剩余元素最小值 > 目标值时，可提前终止
- 实现：先排序，计算前缀和，在递归中判断剩余元素能否满足约束

**排列数量记忆化**：
- 状态定义：dp[mask] 表示已选中集合为 mask（位掩码）时，剩余元素的排列数量
- 状态转移：dp[mask] = Σ dp[mask|(1<<i)]（对所有未选中的i）
- 初始状态：dp[全选] = 1（一个空排列）
- 复杂度：O(n × 2^n)，比朴素O(n!)优化明显
