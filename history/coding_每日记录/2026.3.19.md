## 📚 模块1：动态规划核心概念——从分治、回溯到DP的思维跃迁

### 动态规划的本质

动态规划（Dynamic Programming）不是一种具体的算法，而是一种思想和方法论。它通过将复杂问题分解为重叠的子问题，并存储子问题的解（避免重复计算），最终得到原问题的最优解。

### 三个核心特征

| 特征 | 定义 | 与分治/回溯的对比 |
|------|------|------------------|
| 重叠子问题 | 在递归求解过程中，相同的子问题被反复计算多次 | 分治：子问题相互独立，无重叠 |
| 最优子结构 | 问题的最优解可以由其子问题的最优解构造出来 | DP要求子问题的最优解能组合出全局最优解 |
| 无后效性 | 当前状态确定后，后续决策不受之前决策的影响 | 回溯法中状态会随着路径变化，DP状态一旦确定就不再改变 |

### DP与回溯的本质区别

| 维度 | 回溯法 | 动态规划 |
|------|--------|----------|
| 计算模式 | 暴力搜索 + 剪枝，探索所有可能路径 | 记录子问题最优解，避免重复计算 |
| 时间复杂度 | 指数级（剪枝后可能仍很大） | 多项式级（消除重复计算） |
| 目标 | 找所有解或一个解 | 找最优解 |
| 状态处理 | 状态随着递归路径不断变化和恢复 | 状态一旦计算完成就固定，可重复使用 |

**本质区别：对重叠子问题的处理方式**

- 回溯法：会重复计算相同的子问题
- 动态规划：记录已计算的结果，避免重复

### 如何识别DP问题？

在实际问题中，如果满足以下至少两个特征，就很可能是DP问题：

1. 问题可以分解为子问题
2. 子问题会重复出现
3. 存在最优子结构
4. 问题涉及"最值"或"计数"

## 📚 模块2：记忆化搜索（自顶向下）——以LeetCode 509斐波那契数为例

### 斐波那契数（LeetCode 509）

**题目描述**：给定n，计算F(n)，其中F(0)=0，F(1)=1，F(n)=F(n-1)+F(n-2)

**解法对比**：

```python
# 朴素递归（指数级，不推荐）
def fib_naive(n: int) -> int:
    if n <= 1:
        return n
    return fib_naive(n - 1) + fib_naive(n - 2)

# 记忆化搜索（使用装饰器）
from functools import lru_cache

@lru_cache(maxsize=None)
def fib_memo_decorator(n: int) -> int:
    if n <= 1:
        return n
    return fib_memo_decorator(n - 1) + fib_memo_decorator(n - 2)

# 记忆化搜索（手动缓存）
def fib_memo_manual(n: int) -> int:
    if n <= 1:
        return n
    memo = [-1] * (n + 1)
    
    def dfs(i: int) -> int:
        if memo[i] != -1:
            return memo[i]
        memo[i] = dfs(i - 1) + dfs(i - 2)
        return memo[i]
    
    return dfs(n)
```

**性能对比**：

| n值 | 朴素递归 | 记忆化搜索 | 优化倍数 |
|-----|---------|-----------|---------|
| 30 | ~300ms | <1ms | 300+倍 |
| 40 | ~30000ms | <1ms | 30000+倍 |

## 📚 模块3：递推（自底向上）——以LeetCode 70爬楼梯为例

### 爬楼梯（LeetCode 70）

**题目描述**：爬n阶楼梯，每次爬1或2阶，有多少种方法？

**状态定义**：`dp[i]`表示爬到第i阶楼梯的不同方法数

**状态转移方程**：`dp[i] = dp[i-1] + dp[i-2]`（当i ≥ 2）

**边界条件**：`dp[0] = 1，dp[1] = 1`

```cpp
// C++递推实现
int climbStairs(int n) {
    if (n <= 1) return 1;
    vector<int> dp(n + 1);
    dp[0] = dp[1] = 1;
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i-1] + dp[i-2];
    }
    return dp[n];
}

// 空间优化版本
int climbStairs_optimized(int n) {
    if (n <= 1) return 1;
    int prev2 = 1, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        int current = prev1 + prev2;
        prev2 = prev1;
        prev1 = current;
    }
    return prev1;
}
```

**与斐波那契数的联系**：爬楼梯的解实际上是斐波那契数列的平移版本：`dp[n] = F(n+1)`

### Codeforces 455A - Boredom（C++实现）

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    if (!(cin >> n)) return 0;
    
    vector<int> a(n);
    int max_val = 0;
    for (int i = 0; i < n; i++) {
        cin >> a[i];
        max_val = max(max_val, a[i]);
    }
    
    vector<long long> cnt(max_val + 1, 0);
    for (int x : a) {
        cnt[x]++;
    }
    
    vector<long long> dp(max_val + 1, 0);
    if (max_val >= 1) dp[1] = cnt[1];
    
    for (int i = 2; i <= max_val; i++) {
        dp[i] = max(dp[i-1], dp[i-2] + (long long)i * cnt[i]);
    }
    
    cout << dp[max_val] << "\n";
    return 0;
}
```
