# 计算机网络核心练习

这份练习围绕分层、传输、路由和应用层展开，目的是让你能把协议知识讲明白并能用于实际分析。

## 练习 1：分层与封装

### 题目

请解释网络分层的意义，并按顺序写出一次 HTTP 请求在各层经过的封装过程。

### 参考要点

- 分层可以控制复杂度，明确职责边界。
- HTTP 数据会被 TCP 加上首部，再被 IP 封装，再进入链路层帧。

## 练习 2：三次握手与四次挥手

### 题目

分别说明 TCP 三次握手和四次挥手的原因，并回答：

1. 为什么不能用两次握手替代三次握手？
2. 为什么关闭连接时常常不是一次报文就结束？

### 参考要点

- 握手要同步双方状态和初始序号。
- 关闭是全双工的，发送方向和接收方向要分别关闭。

## 练习 3：子网划分

### 题目

给定地址 `192.168.10.0/24`，请划分出 4 个等大小子网，并写出每个子网的网络地址、可用主机范围和广播地址。

### 参考要点

- `/24` 划分成 4 个子网后前缀变为 `/26`。
- 每个子网有 64 个地址，去掉网络地址和广播地址后有 62 个可用主机地址。

## 练习 4：HTTP 与 DNS

### 题目

解释浏览器访问 `https://example.com` 时，DNS、TCP、TLS、HTTP 分别做了什么。

### 参考要点

- DNS 将域名解析为 IP。
- TCP 建立连接。
- TLS 提供加密和认证。
- HTTP 负责请求和响应资源。

## 练习 5：性能分析

### 题目

解释时延、RTT、吞吐量、带宽和丢包率的区别，并说明为什么高带宽不一定意味着低时延。

### 参考要点

- 带宽是容量，时延是传输时间。
- 排队、处理和传播都会影响时延。

## 综合题：设计一个文件传输协议

### 题目

请设计一个面向局域网的文件传输协议，至少考虑：

- 消息头格式
- 分片与重传
- 校验机制
- 断点续传
- 错误处理

### 作答要求

- 画出报文格式
- 写出发送与接收流程
- 分析协议的可靠性与局限

## 复习建议

- 每道题都结合抓包或命令行输出来理解。
- 多画封装关系图和状态机图。
- 题目答案尽量写成“协议流程 + 原理说明 + 现实场景”。
