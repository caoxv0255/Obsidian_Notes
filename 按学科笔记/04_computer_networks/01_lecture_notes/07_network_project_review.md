# 网络项目与复盘

## 1. 为什么要做项目

网络知识很容易停留在“会背协议”层面。项目的价值，是逼迫你把协议、编程、性能与故障定位连成一体。

## 2. 课程中的项目主线

建议按以下顺序推进：

1. 抓包观察
2. 套接字通信
3. 静态服务端
4. 性能测试
5. 安全与日志

## 3. 一个完整的网络小项目应该有什么

- 问题定义
- 协议设计
- 接口设计
- 错误处理
- 日志与调试
- 性能评估

## 4. 复盘模板

每次项目结束后，至少回答以下问题：

- 这个项目解决了什么问题
- 核心瓶颈在哪里
- 哪些协议概念被真正用到了
- 哪些错误是网络层、哪些错误是应用层
- 如果重做一次，如何简化设计

## 5. 课程最终目标

学完网络后，你应当能够：

- 看懂一次网络请求的完整路径
- 写出一个可工作的 TCP/HTTP 小程序
- 通过抓包和日志定位常见网络问题
- 理解性能、安全和可观测性的关系

## 6. 项目式学习入口

### 综合项目建议

- 实现一个带协议解析、连接管理与日志的迷你文件传输服务。
- 或实现一个带缓存、重试和统计信息的任务分发客户端。

## 7. 学习检查

- 能否把课程中的协议知识整合成一个项目？
- 能否说明自己的网络程序在不同层分别做了什么？
- 能否写出项目复盘而不是只写“完成了”？
