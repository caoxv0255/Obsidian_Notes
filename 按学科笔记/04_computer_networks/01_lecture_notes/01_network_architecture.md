# 网络体系结构与分层模型

## 1. 网络的本质

计算机网络的核心问题，是如何让不同主机上的进程在不可靠的通信链路上可靠地交换信息。网络不是单纯的“把机器连起来”，而是要同时解决：

- 寻址：数据要发给谁
- 路由：数据怎么到达对方
- 可靠性：传输过程中丢了怎么办
- 性能：延迟、吞吐量、抖动如何控制
- 安全性：数据如何防篡改、伪造和窃听

## 2. 分层思想

分层的意义在于控制复杂度。每一层只负责一部分问题，并通过标准接口与上下层交互。

典型分层模型：

- 应用层：HTTP、DNS、SMTP、TLS
- 传输层：TCP、UDP
- 网络层：IP、路由、转发
- 链路层：以太网、Wi-Fi、ARP、交换
- 物理层：比特传输、信号编码、介质

分层的优点：

- 抽象清晰
- 易于替换实现
- 便于标准化
- 便于分工协作

## 3. 一次网页访问经历了什么

以访问一个网页为例：

1. 用户在浏览器输入 URL
2. 浏览器通过 DNS 获取服务器 IP
3. 浏览器与服务器建立 TCP 连接
4. 发送 HTTP 请求
5. 服务器返回 HTTP 响应
6. 浏览器解析 HTML、CSS、JavaScript 并继续发起后续请求

这条路径把应用层、传输层、网络层和链路层全部串起来了。

## 4. 寻址、封装与解封装

网络通信通常伴随着封装：

- 应用层生成应用数据
- 传输层加上 TCP/UDP 首部，形成段
- 网络层加上 IP 首部，形成包
- 链路层加上帧首部和尾部，形成帧
- 物理层把帧变成比特流发送出去

接收方执行相反的解封装过程。

理解封装的关键在于：每层都增加自己的控制信息，但不会直接关心下层的实现细节。

## 5. 三个最重要的网络层次

### 5.1 链路层

链路层负责一跳内的数据传输。典型问题包括：

- 如何在共享介质上访问链路
- 如何用 MAC 地址标识同一网络中的设备
- 如何通过交换机转发帧

### 5.2 网络层

网络层负责跨网络转发。它要解决：

- IP 编址
- 子网划分
- 路由选择
- 分片与重组

### 5.3 传输层

传输层连接的是进程而不是主机。TCP 提供可靠、面向连接的字节流，UDP 提供简单、无连接的数据报服务。

## 6. TCP 为什么重要

TCP 是理解现代互联网的核心协议之一。它解决了以下问题：

- 连接建立与释放
- 丢包重传
- 流量控制
- 拥塞控制
- 字节流重组

TCP 的复杂之处在于，它同时考虑“端到端可靠性”和“网络整体效率”。

## 7. DNS、HTTP 与 TLS 的协作

### DNS

DNS 将域名映射到 IP 地址，是“人类可读名称”与“机器可路由地址”的桥梁。

### HTTP

HTTP 定义客户端与服务器如何交换资源请求和响应。它是应用层协议的代表。

### TLS

TLS 在不可靠网络之上提供加密、认证与完整性保护。它不是独立于应用的“附加功能”，而是现代 Web 的基础组件。

## 8. 性能指标

理解网络不能只看能不能通，还要看快不快、稳不稳：

- 时延：从发送到接收所需时间
- 吞吐量：单位时间传输的数据量
- 丢包率：丢失的数据比例
- 抖动：时延波动程度
- RTT：往返时延

## 9. 典型误区

- 以为“网络就是网线”：实际上网络是一整套协议与控制系统。
- 以为“IP 地址等于设备身份”：IP 只是位置相关标识，可能会变化。
- 以为“TCP 一定慢”：TCP 的设计目标是可靠与可控，不是单纯追求极低延迟。
- 以为“DNS 只是查地址”：DNS 还承担负载均衡、服务发现等作用。

## 10. 课堂例题

### 例题 1：为什么需要分层？

因为如果所有问题都放在一个协议里解决，系统会变得极其复杂。分层让每层只处理自己的职责范围，并通过接口连接。

### 例题 2：TCP 和 UDP 的差异是什么？

TCP 提供可靠、有序、面向连接的字节流；UDP 只提供简单的数据报传输，不保证可靠性和顺序。

## 11. 项目式学习入口

### 项目 A：抓包实验

- 目标：学会看懂真实网络通信。
- 做法：抓一次 DNS 查询、一次 TCP 建连、一次 HTTP 请求。
- 输出：协议字段说明、报文流向图、分析笔记。

### 项目 B：TCP 回显服务

- 目标：把套接字编程和协议行为结合起来。
- 做法：实现一个客户端和服务端，完成连接、收发、关闭。
- 输出：接口说明、状态图、测试记录。

### 项目 C：静态文件服务器

- 目标：理解 HTTP 请求处理的基本流程。
- 做法：支持 GET 请求、文件返回、404 处理和简单日志。
- 输出：请求处理流程图、报文示例、功能说明。
