# 网络层与路由

## 1. 网络层解决什么问题

网络层负责把数据从一个网络转发到另一个网络。它解决的是“跨网络到达”的问题，而不是单一链路上的问题。

## 2. IP 地址

IP 地址是网络层的核心标识。它描述的是主机在网络中的位置，而不是硬件身份证。

IP 地址的主要作用：

- 标识主机
- 支持路由转发
- 支持子网划分

## 3. 子网与前缀

网络通常按前缀划分。子网掩码或前缀长度决定哪些位是网络号，哪些位是主机号。

子网划分的目的：

- 降低广播范围
- 便于地址管理
- 支持层次化路由

## 4. 路由与转发

- 转发：路由器根据路由表把包送到下一跳
- 路由：决定路由表怎么形成

转发是局部行为，路由是全局决策。

## 5. 路由协议思想

不同路由协议追求不同目标：

- 距离向量：简单，按邻居信息逐步收敛
- 链路状态：全局视图更强，但开销更高

现实网络通常是多种机制协同工作。

## 6. NAT 与私有地址

NAT 允许局域网内部主机共享一个或少数公网地址。

优点：

- 节省 IPv4 地址
- 隐藏内部拓扑

缺点：

- 破坏端到端可见性
- 对某些协议不友好

## 7. 分片与重组

不同链路的 MTU 不同。若 IP 包过大，可能需要分片或避免分片。

分片会增加复杂度和丢包风险，因此现代网络更强调路径 MTU 发现和端到端控制。

## 8. 例题

### 例题 1：为什么 IP 需要前缀？

因为前缀允许层次化管理地址，支持子网和聚合路由。

### 例题 2：NAT 为什么会影响端到端连接？

因为地址和端口被中间设备改写，外部主机无法直接看到内部主机的真实身份。

## 9. 项目式学习入口

### 项目 A：子网划分练习

- 目标：熟悉前缀与主机范围。
- 做法：给定一个网络地址，推导可用子网和广播地址。
- 输出：地址表、推导过程和结论。

### 项目 B：路由表模拟器

- 目标：理解下一跳与最长前缀匹配。
- 做法：输入多条路由，模拟转发过程。
- 输出：路由表、匹配规则说明。
