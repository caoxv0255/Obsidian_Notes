# 应用层协议

## 1. 应用层的角色

应用层直接面向用户和应用程序。它定义的是“应用如何交换数据”，而不是“数据如何在链路上传输”。

## 2. DNS

DNS 的作用是把域名映射为 IP 地址。它把人类可读的名称系统与网络可路由的地址系统连接起来。

DNS 常见用途：

- 名称解析
- 负载分配
- 服务发现

## 3. HTTP

HTTP 是 Web 的基础协议。它定义客户端如何请求资源，服务器如何响应资源。

典型请求-响应流程：

1. 客户端发起请求
2. 服务器返回状态码、头部和内容
3. 浏览器根据内容继续发起后续请求

## 4. 请求方法与状态码

- GET：获取资源
- POST：提交数据
- PUT：更新资源
- DELETE：删除资源

状态码常见含义：

- 200：成功
- 301/302：重定向
- 404：资源不存在
- 500：服务器内部错误

## 5. TLS

TLS 为应用层提供加密、身份认证和完整性保护。

它的作用包括：

- 防窃听
- 防篡改
- 防伪装

## 6. 应用层不是孤立的

一次 Web 访问通常涉及：

- DNS 解析
- TCP 建连
- TLS 握手
- HTTP 请求
- 内容下载与资源加载

因此，应用层学习不能脱离传输层和网络层。

## 7. 例题

### 例题 1：为什么浏览器访问网页时不只是发送 HTTP？

因为在 HTTP 之前通常需要 DNS、TCP 和可能的 TLS 协商，整个过程是分层协作的结果。

### 例题 2：HTTP 为什么适合 Web？

因为它简单、可扩展、文本化、适合请求-响应模式，并且容易被缓存和代理机制利用。

## 8. 项目式学习入口

### 项目 A：静态网站服务器

- 目标：理解 HTTP 请求处理。
- 做法：实现 GET、404、日志和头部解析。
- 输出：协议解析器、响应生成器、测试集。

### 项目 B：域名解析实验

- 目标：理解 DNS 查询流程。
- 做法：观察不同域名的解析结果与缓存行为。
- 输出：解析记录、延迟数据、分析笔记。
