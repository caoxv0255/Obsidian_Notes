# 网络安全与可观测性

## 1. 安全要解决什么问题

网络安全的目标包括：

- 保密性：别人看不懂
- 完整性：别人改不了
- 认证性：知道对方是谁
- 可用性：服务不容易被打垮

## 2. 常见威胁

- 窃听
- 篡改
- 伪造
- 重放
- 中间人攻击
- 拒绝服务攻击

## 3. TLS 的核心价值

TLS 为应用层通信提供安全传输能力。它把明文通信变成受保护的通信，是现代 Web、API 和大多数在线服务的基础。

## 4. 对称加密与非对称加密

- 对称加密：速度快，适合大量数据加密
- 非对称加密：便于密钥交换和身份认证，但开销更大

真实系统通常把二者结合使用。

## 5. 认证与授权

- 认证：你是谁
- 授权：你能做什么

这两个概念不能混淆。

## 6. 可观测性

安全不只靠加密，还要能观察和定位问题。

常见手段：

- 日志
- 指标
- 抓包
- 告警
- 审计

## 7. 例题

### 例题 1：为什么加密不等于安全？

因为安全还需要认证、授权、完整性和可用性。只加密并不能防止服务被滥用。

### 例题 2：为什么日志对安全很重要？

因为日志可以帮助定位攻击、审计行为和恢复故障现场。

## 8. 项目式学习入口

### 项目 A：HTTPS 抓包观察

- 目标：理解 TLS 的作用。
- 做法：观察一次 HTTPS 请求的握手和数据传输。
- 输出：握手流程图、字段分析、结论。

### 项目 B：简化认证系统

- 目标：理解认证与授权。
- 做法：设计一个带登录和访问控制的小型 Web 应用。
- 输出：权限模型、日志和测试记录。
