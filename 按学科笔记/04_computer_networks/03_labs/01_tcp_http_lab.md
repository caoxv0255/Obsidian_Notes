# 计算机网络实验：TCP、HTTP 与抓包

## 实验目标

通过抓包和小程序，把 TCP 连接、HTTP 请求和网络性能真正观察出来。

## 实验 A：抓包观察网页访问

### 实验要求

选择一个网页，使用抓包工具观察从域名解析到网页内容加载的全过程，并标出：

- DNS 查询
- TCP 三次握手
- TLS 握手（如有）
- HTTP 请求和响应

### 记录重点

- 每一步对应的协议层
- 报文的源地址、目的地址、端口和序号
- 是否发生重传或重定向

## 实验 B：TCP 回显服务

### 实验要求

实现一个最简单的 TCP 客户端和服务端：

- 服务端接受连接并回显收到的内容
- 客户端发送字符串并输出回显结果

### 观察重点

- 连接建立过程
- 收发数据是否按字节流处理
- 关闭连接时的状态变化

## 实验 C：HTTP 静态服务器

### 实验要求

实现一个支持 GET 请求的静态文件服务器，能够返回本地文件和 404 页面。

### 功能要求

- 解析请求行
- 解析常见请求头
- 返回响应状态码
- 输出访问日志

## 实验结果记录模板

### 1. 现象

- 抓包里观察到了什么
- 程序运行结果是什么

### 2. 协议解释

- 这一帧/这一段/这个请求属于哪一层
- 它的作用是什么

### 3. 问题定位

- 如果失败，问题在哪一层
- 如何通过抓包或日志定位

### 4. 改进

- 能否减少 RTT
- 能否减少请求次数
- 能否更清晰地输出日志

## 实验验收标准

- 能说清一次网页访问的完整协议链
- TCP 回显服务能正常双向收发
- HTTP 服务器能返回正确状态码和内容
