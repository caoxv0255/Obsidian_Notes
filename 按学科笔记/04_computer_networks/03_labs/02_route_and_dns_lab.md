# 计算机网络实验：路由与 DNS

## 实验 A：路由路径观察

使用 `traceroute` 或 `tracert` 观察到目标主机的路径，记录每一跳的延迟变化和可能的丢包。

### 观察重点

- 每一跳的意义。
- RTT 波动的原因。
- 终点不可达时的表现。

## 实验 B：DNS 查询链路

使用 `nslookup`、`dig` 或浏览器开发者工具观察一次域名解析，记录递归查询与缓存命中现象。

### 观察重点

- 查询了哪些记录类型。
- 解析结果是否被缓存。
- 不同 DNS 服务器返回是否一致。
