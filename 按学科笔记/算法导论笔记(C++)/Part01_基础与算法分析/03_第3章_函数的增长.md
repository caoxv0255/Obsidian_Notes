# 第3章：函数的增长

## 概述

函数的增长是算法分析的基础。在比较算法效率时，我们不关注具体的运行时间（这依赖于硬件、编译器、输入数据等因素），而是关注算法运行时间随输入规模增长的**渐近行为**（asymptotic behavior）。

### 为什么需要渐近分析？

1. **硬件无关性**：消除具体机器和编译器的影响
2. **简化分析**：忽略常数因子和低阶项，关注主要趋势
3. **预测性能**：预测算法在大规模输入下的表现
4. **算法比较**：提供统一的比较标准

### 常见的时间复杂度

| 复杂度 | 名称 | 典型算法 | n=10³ 时的操作数 |
|--------|------|----------|------------------|
| O(1) | 常数时间 | 数组访问 | 1 |
| O(log n) | 对数时间 | 二分查找 | ~10 |
| O(n) | 线性时间 | 线性搜索 | 1,000 |
| O(n log n) | 线性对数时间 | 归并排序 | ~10,000 |
| O(n²) | 平方时间 | 冒泡排序 | 1,000,000 |
| O(n³) | 立方时间 | 矩阵乘法 | 1,000,000,000 |
| O(2ⁿ) | 指数时间 | 暴力搜索 | 不可行 |
| O(n!) | 阶乘时间 | 旅行商问题（暴力） | 不可行 |

## 渐近记号

### O-记号（Big-O Notation）

**定义**：$f(n) = O(g(n))$ 如果存在正常数 $c$ 和 $n_0$，使得对于所有 $n \geq n_0$，有 $0 \leq f(n) \leq c \cdot g(n)$。

**直观理解**：$f(n)$ 的增长速度**不超过** $g(n)$（上界）

**示例**：
- $n^2 + 3n + 2 = O(n^2)$
- $100n + 1000 = O(n)$
- $n \log n = O(n^2)$（虽然这不是紧确界）

**性质**：
- 传递性：若 $f(n) = O(g(n))$ 且 $g(n) = O(h(n))$，则 $f(n) = O(h(n))$
- 加法：若 $f_1(n) = O(g_1(n))$ 且 $f_2(n) = O(g_2(n))$，则 $f_1(n) + f_2(n) = O(\max(g_1(n), g_2(n)))$
- 乘法：若 $f_1(n) = O(g_1(n))$ 且 $f_2(n) = O(g_2(n))$，则 $f_1(n) \cdot f_2(n) = O(g_1(n) \cdot g_2(n))$

### Ω-记号（Big-Omega Notation）

**定义**：$f(n) = \Omega(g(n))$ 如果存在正常数 $c$ 和 $n_0$，使得对于所有 $n \geq n_0$，有 $0 \leq c \cdot g(n) \leq f(n)$。

**直观理解**：$f(n)$ 的增长速度**不低于** $g(n)$（下界）

**示例**：
- $n^2 + 3n + 2 = \Omega(n^2)$
- $100n + 1000 = \Omega(n)$
- $n^2 = \Omega(n \log n)$

**性质**：
- $f(n) = \Omega(g(n))$ 当且仅当 $g(n) = O(f(n))$

### Θ-记号（Big-Theta Notation）

**定义**：$f(n) = \Theta(g(n))$ 如果存在正常数 $c_1, c_2$ 和 $n_0$，使得对于所有 $n \geq n_0$，有 $0 \leq c_1 \cdot g(n) \leq f(n) \leq c_2 \cdot g(n)$。

**直观理解**：$f(n)$ 的增长速度**与** $g(n)$ **相同**（紧确界）

**示例**：
- $n^2 + 3n + 2 = \Theta(n^2)$
- $100n + 1000 = \Theta(n)$
- $\frac{1}{2}n^2 - 3n = \Theta(n^2)$

**性质**：
- $f(n) = \Theta(g(n))$ 当且仅当 $f(n) = O(g(n))$ 且 $f(n) = \Omega(g(n))$
- 对称性：$f(n) = \Theta(g(n))$ 当且仅当 $g(n) = \Theta(f(n))$

### o-记号（Little-o Notation）

**定义**：$f(n) = o(g(n))$ 如果对于任意正常数 $c > 0$，存在 $n_0 > 0$，使得对于所有 $n \geq n_0$，有 $0 \leq f(n) < c \cdot g(n)$。

**直观理解**：$f(n)$ 的增长速度**严格低于** $g(n)$（非渐近紧确的上界）

**示例**：
- $n = o(n^2)$
- $n \log n = o(n^2)$
- $2^n = o(3^n)$

### ω-记号（Little-omega Notation）

**定义**：$f(n) = \omega(g(n))$ 如果对于任意正常数 $c > 0$，存在 $n_0 > 0$，使得对于所有 $n \geq n_0$，有 $0 \leq c \cdot g(n) < f(n)$。

**直观理解**：$f(n)$ 的增长速度**严格高于** $g(n)$（非渐近紧确的下界）

**示例**：
- $n^2 = \omega(n)$
- $n^2 = \omega(n \log n)$

## 常用函数及其增长

### 对数函数

**性质**：
- $\log_b n = \frac{\log_a n}{\log_a b}$（换底公式）
- $\log(ab) = \log a + \log b$
- $\log(a^b) = b \log a$
- $\log n$ 增长非常慢，$\log_2 10^6 \approx 20$

**在算法中的应用**：
- 二分查找：$O(\log n)$
- 平衡树操作：$O(\log n)$
- 快速排序的平均情况：$O(n \log n)$

### 多项式函数

**定义**：$p(n) = a_k n^k + a_{k-1} n^{k-1} + \cdots + a_0$

**性质**：
- $n^a = O(n^b)$ 当且仅当 $a \leq b$
- 多项式函数的增长速度由最高次项决定
- $n^k + n^{k-1} + \cdots + 1 = \Theta(n^k)$

**示例**：
- $n^2 + 100n = \Theta(n^2)$
- $0.01n^3 + n^2 = \Theta(n^3)$

### 指数函数

**定义**：$f(n) = a^n$，其中 $a > 1$

**性质**：
- 指数函数增长极快
- $a^n = o(b^n)$ 当且仅当 $a < b$
- $n^k = o(a^n)$ 对于任何 $k > 0$ 和 $a > 1$

**在算法中的应用**：
- 通常表示低效算法
- 暴力搜索：$O(2^n)$
- 动态规划的某些应用：$O(2^n)$

### 阶乘函数

**定义**：$n! = 1 \times 2 \times 3 \times \cdots \times n$

**性质**：
- 增长比指数函数更快
- Stirling公式：$n! = \sqrt{2\pi n}\left(\frac{n}{e}\right)^n \left(1 + \Theta\left(\frac{1}{n}\right)\right)$
- $n! = \omega(2^n)$ 且 $n! = o(n^n)$

**在算法中的应用**：
- 排列和组合问题
- 旅行商问题的暴力解法：$O(n!)$

### 欧拉函数

**定义**：$\phi(n)$ 表示小于等于 $n$ 的正整数中与 $n$ 互质的数的个数

**性质**：
- $\phi(n) = n \prod_{p|n}\left(1 - \frac{1}{p}\right)$，其中 $p$ 是 $n$ 的质因数
- $\phi(n) = \Omega(n / \log \log n)$

## 复杂度比较

### 复杂度层次

从低到高的复杂度层次：

```
O(1) < O(log n) < O(√n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!) < O(nⁿ)
```

### 复杂度函数的增长比较

| n | log n | √n | n | n log n | n² | 2ⁿ |
|---|-------|-----|---|---------|-----|-----|
| 1 | 0 | 1 | 1 | 0 | 1 | 2 |
| 10 | 3.3 | 3.2 | 10 | 33 | 100 | 1,024 |
| 100 | 6.6 | 10 | 100 | 664 | 10,000 | 1.27×10³⁰ |
| 1,000 | 10 | 31.6 | 1,000 | 9,966 | 1,000,000 | ∞ |
| 10,000 | 13.3 | 100 | 10,000 | 132,877 | 100,000,000 | ∞ |

**结论**：对于大规模输入，对数时间算法非常高效，而指数时间算法通常不可行。

/**
 * @brief 计算复杂度函数值
 */
struct ComplexityFunctions {
    // O(1) - 常数时间
    static long long constant(int n) {
        return 1;
    }
    
    // O(log n) - 对数时间
    static long long logarithmic(int n) {
        return static_cast<long long>(std::log2(n));
    }
    
    // O(n) - 线性时间
    static long long linear(int n) {
        return n;
    }
    
    // O(n log n) - 线性对数时间
    static long long linearithmic(int n) {
        return n * static_cast<long long>(std::log2(n));
    }
    
    // O(n²) - 平方时间
    static long long quadratic(int n) {
        return static_cast<long long>(n) * n;
    }
    
    // O(n³) - 立方时间
    static long long cubic(int n) {
        return static_cast<long long>(n) * n * n;
    }
    
    // O(2ⁿ) - 指数时间
    static long long exponential(int n) {
        if (n > 63) return LLONG_MAX; // 防止溢出
        return 1LL << n;
    }
    
    // O(n!) - 阶乘时间
    static long long factorial(int n) {
        if (n > 20) return LLONG_MAX; // 20! 超过 64 位整数范围
        long long result = 1;
        for (int i = 2; i <= n; ++i) {
            result *= i;
        }
        return result;
    }
};

// ============ 实际算法复杂度演示 ============

/**
 * @brief 线性搜索 - O(n)
 */
long long linear_search(const std::vector<int>& arr, int target) {
    long long operations = 0;
    for (size_t i = 0; i < arr.size(); ++i) {
        operations++;
        if (arr[i] == target) {
            break;
        }
    }
    return operations;
}

/**
 * @brief 二分查找 - O(log n)
 */
long long binary_search(const std::vector<int>& arr, int target) {
    long long operations = 0;
    int left = 0;
    int right = arr.size() - 1;
    
    while (left <= right) {
        operations++;
        int mid = left + (right - left) / 2;
        
        if (arr[mid] == target) {
            return operations;
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    
    return operations;
}

/**
 * @brief 冒泡排序 - O(n²)
 */
long long bubble_sort(std::vector<int>& arr) {
    long long operations = 0;
    int n = arr.size();
    
    for (int i = 0; i < n - 1; ++i) {
        for (int j = 0; j < n - i - 1; ++j) {
            operations++;
            if (arr[j] > arr[j + 1]) {
                std::swap(arr[j], arr[j + 1]);
            }
        }
    }
    
    return operations;
}

/**
 * @brief 归并排序 - O(n log n)
 */
long long merge_sort(std::vector<int>& arr, int left, int right) {
    long long operations = 0;
    
    if (left >= right) return operations;
    
    int mid = left + (right - left) / 2;
    
    operations += merge_sort(arr, left, mid);
    operations += merge_sort(arr, mid + 1, right);
    
    // 合并
    std::vector<int> temp;
    int i = left, j = mid + 1;
    
    while (i <= mid && j <= right) {
        operations++;
        if (arr[i] <= arr[j]) {
            temp.push_back(arr[i++]);
        } else {
            temp.push_back(arr[j++]);
        }
    }
    
    while (i <= mid) {
        temp.push_back(arr[i++]);
        operations++;
    }
    
    while (j <= right) {
        temp.push_back(arr[j++]);
        operations++;
    }
    
    for (int k = 0; k < temp.size(); ++k) {
        arr[left + k] = temp[k];
    }
    
    return operations;
}

/**
 * @brief 比较不同复杂度函数的增长
 */
void compare_complexity_growth(const std::vector<int>& sizes) {
    std::cout << "=== 复杂度函数增长比较 ===" << std::endl;
    std::cout << std::left;
    std::cout << std::setw(10) << "n"
              << std::setw(15) << "O(1)"
              << std::setw(15) << "O(log n)"
              << std::setw(15) << "O(n)"
              << std::setw(15) << "O(n log n)"
              << std::setw(15) << "O(n²)"
              << std::setw(15) << "O(n³)" << std::endl;
    std::cout << std::string(100, '-') << std::endl;
    
    for (int n : sizes) {
        std::cout << std::setw(10) << n
                  << std::setw(15) << ComplexityFunctions::constant(n)

                  << std::setw(15) << ComplexityFunctions::linear(n)
                  << std::setw(15) << ComplexityFunctions::linearithmic(n)
                  << std::setw(15) << ComplexityFunctions::quadratic(n)
                  << std::setw(15) << ComplexityFunctions::cubic(n) << std::endl;
    }
    std::cout << std::endl;
}

/**
 * @brief 实际算法性能测试
 */
void test_algorithm_performance() {
    std::cout << "=== 实际算法性能测试 ===" << std::endl;
    
    // 测试不同规模的输入
    std::vector<int> test_sizes = {100, 1000, 10000, 100000};
    
    for (int n : test_sizes) {
        std::cout << "\n输入规模: " << n << std::endl;
        
        // 准备测试数据
        std::vector<int> arr(n);
        for (int i = 0; i < n; ++i) {
            arr[i] = i;
        }
        
        // 测试线性搜索
        auto start = std::chrono::high_resolution_clock::now();
        long long ops1 = linear_search(arr, n - 1);
        auto end = std::chrono::high_resolution_clock::now();
        auto duration1 = std::chrono::duration_cast<std::chrono::microseconds>(end - start);
        
        // 测试二分查找
        std::vector<int> sorted_arr = arr;
        start = std::chrono::high_resolution_clock::now();
        long long ops2 = binary_search(sorted_arr, n - 1);
        end = std::chrono::high_resolution_clock::now();
        auto duration2 = std::chrono::duration_cast<std::chrono::microseconds>(end - start);
        
        std::cout << "线性搜索: 操作数=" << ops1 << ", 用时=" << duration1.count() << "μs" << std::endl;
        std::cout << "二分查找: 操作数=" << ops2 << ", 用时=" << duration2.count() << "μs" << std::endl;
        std::cout << "加速比: " << static_cast<double>(duration1.count()) / duration2.count() << "x" << std::endl;
    }
    std::cout << std::endl;
}

/**
 * @brief 排序算法性能比较
 */
void compare_sorting_algorithms() {
    std::cout << "=== 排序算法性能比较 ===" << std::endl;
    
    std::vector<int> test_sizes = {100, 500, 1000, 2000};
    
    for (int n : test_sizes) {
        std::cout << "\n输入规模: " << n << std::endl;
        
        // 准备测试数据
        std::vector<int> arr1(n), arr2(n);
        for (int i = 0; i < n; ++i) {
            arr1[i] = n - i; // 逆序
            arr2[i] = n - i;
        }
        
        // 测试冒泡排序
        auto start = std::chrono::high_resolution_clock::now();
        long long ops1 = bubble_sort(arr1);
        auto end = std::chrono::high_resolution_clock::now();
        auto duration1 = std::chrono::duration_cast<std::chrono::milliseconds>(end - start);
        
        // 测试归并排序
        start = std::chrono::high_resolution_clock::now();
        long long ops2 = merge_sort(arr2, 0, n - 1);
        end = std::chrono::high_resolution_clock::now();
        auto duration2 = std::chrono::duration_cast<std::chrono::milliseconds>(end - start);
        
        std::cout << "冒泡排序 (O(n²)): 操作数=" << ops1 << ", 用时=" << duration1.count() << "ms" << std::endl;
        std::cout << "归并排序 (O(n log n)): 操作数=" << ops2 << ", 用时=" << duration2.count() << "ms" << std::endl;
        
        if (duration2.count() > 0) {
            std::cout << "加速比: " << static_cast<double>(duration1.count()) / duration2.count() << "x" << std::endl;
        }
    }
    std::cout << std::endl;
}

## 算法分析

### 时间复杂度分析方法

#### 1. 循环分析

**单层循环**：
```cpp
for (int i = 0; i < n; ++i) {
    // O(1) 操作
}
// 时间复杂度: O(n)
```

**嵌套循环**：
```cpp
for (int i = 0; i < n; ++i) {
    for (int j = 0; j < n; ++j) {
        // O(1) 操作
    }
}
// 时间复杂度: O(n²)
```

**对数循环**：
```cpp
for (int i = 1; i < n; i *= 2) {
    // O(1) 操作
}
// 时间复杂度: O(log n)
```

#### 2. 递归分析

**递归式建立**：
- 确定基本情况
- 确定递归关系
- 计算递归深度和每层的工作量

**示例：二分查找**：
```
T(n) = T(n/2) + O(1)
T(1) = O(1)
解: T(n) = O(log n)
```

**示例：归并排序**：
```
T(n) = 2T(n/2) + O(n)
T(1) = O(1)
解: T(n) = O(n log n)
```

#### 3. 分摊分析

对于某些操作，单次操作的最坏情况时间可能很大，但一系列操作的平均时间很小。

**示例：动态数组扩容**：
- 单次插入操作最坏情况 O(n)
- n 次插入操作的总时间 O(n)
- 摊还时间 O(1)

### 空间复杂度分析

空间复杂度分析主要关注：
1. **输入空间**：存储输入所需的空间
2. **辅助空间**：算法执行过程中额外使用的空间
3. **递归栈空间**：递归算法的栈空间

**示例**：
- 归并排序的辅助空间：O(n)
- 快速排序的递归栈空间：O(log n) 平均，O(n) 最坏
- 原地排序算法（如堆排序）：O(1) 辅助空间

### 正确性证明

虽然本章主要讨论时间复杂度，但算法正确性证明仍然重要：

1. **循环不变式**：用于证明迭代算法
2. **数学归纳法**：用于证明递归算法
3. **反证法**：用于证明某些性质

## 常见误区

### 误区1：忽略常数因子

**错误观点**：O(n) 和 O(2n) 是一样的

**正确理解**：虽然渐近上界相同，但常数因子在实际情况中很重要。对于小规模输入，常数因子大的算法可能更慢。

### 误区2：只考虑最坏情况

**错误观点**：只分析最坏时间复杂度

**正确理解**：应该同时考虑：
- 最坏情况（Worst Case）
- 平均情况（Average Case）
- 最好情况（Best Case）

### 误区3：过度优化

**错误观点**：总是追求最优的渐近复杂度

**正确理解**：应该权衡：
- 代码复杂度
- 实现难度
- 实际输入规模
- 常数因子的影响

### 误区4：混淆输入规模

**错误观点**：所有问题的输入规模都是 n

**正确理解**：输入规模可以是：
- 数组长度
- 数值大小（如整数位数）
- 图的边数和顶点数
- 矩阵的维度

## 练习题

1. **基础题**：分析以下代码的时间复杂度：
   ```cpp
   for (int i = 0; i < n; ++i) {
       for (int j = 0; j < i; ++j) {
           // O(1) 操作
       }
   }
   ```

2. **进阶题**：证明 $\log_b n = O(\log_a n)$ 对于任何 $a, b > 1$。

3. **挑战题**：分析快速排序的最好情况、平均情况和最坏情况时间复杂度，并解释为什么平均情况更接近实际情况。

4. **实现题**：实现一个函数，计算两个算法在不同输入规模下的性能比值，并绘制增长曲线。

5. **思考题**：为什么在实际工程中，O(n log n) 的算法往往比 O(n) 的算法更受欢迎？考虑哪些因素会影响这个选择？
