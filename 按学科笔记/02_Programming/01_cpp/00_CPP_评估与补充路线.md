# C++ 笔记评估与补充路线（ACM 选手 & Linux C++ 工程师视角）

## 0. 结论速览

当前 `备份/02_Programming/C++` 已经从“评估对象”变成了“可直接使用的双轨笔记体系”：

- **ACM/竞赛侧**：`99_Snippets/` 已形成完整模板库，覆盖图论、DP、字符串、数论、数据结构、数论进阶、字符串进阶、树上结构、状态压缩、莫队、2-SAT、NTT、FFT、线性基、动态开点线段树、后缀自动机、点分治、李超线段树、二维结构等高频专题。
- **Linux C++ 工程侧**：`04_Advanced/` 已覆盖类与对象、RAII、移动语义、STL 坑点、模板、CMake、调试、socket/epoll、Linux OS 基础、并发与性能、工程闭环。
- **C++ 主线**：基础知识、STL、算法、高级主题四层已经串联完成，索引闭环清楚。

结论很直接：**原始规划中的两条主线已经落地，内容层面可以判定为达标。**

## 1. 现状盘点

### 1.1 C++ 主线

- STL：`02_STL/` 2 篇。

- 当前模板库已经覆盖绝大多数竞赛常见题型与高阶模板。

## 2. 达标判断

### 2.1 ACM 视角

已达标。

- 高频基础模板已齐：Fast IO、调试、离散化、前缀和/差分、二分答案。
- 主要数据结构已齐：DSU、Fenwick、线段树、树状结构、动态开点、主席树。
- 图论主干已齐：最短路、拓扑、LCA、SCC、MST、网络流、2-SAT、双连通分量、点分治、树链剖分。
- 字符串/数学进阶已齐：KMP、Z、Rolling Hash、AC 自动机、后缀数组、后缀自动机、数位 DP、NTT、FFT、线性基、高斯消元。

### 2.2 Linux C++ 工程视角

已达标。

- 现代 C++ 核心已齐：类与对象、RAII、移动语义、模板、STL 复杂点。
- 工具链已齐：CMake、调试、sanitizers、core dump、诊断思路。
- OS / 网络 / 并发链路已齐：进程线程、文件 I/O、信号、socket、epoll、线程池、性能调优。
- 项目闭环已齐：已有“从架构到排障”的文档化思路。

## 3. 口径说明

- **主索引的 35 个笔记**只统计 C++ 主线，不包含 `99_Snippets/`。
- **Snippets 模板库**单独统计，不并入主索引总数。
- 这两个口径现在都已经明确：主线负责“知识框架”，Snippets 负责“实战模板”。

## 4. 还能继续扩展的部分

这些不属于当前规划的硬性缺口，属于可选增强：

- 更偏项目化：HTTP 静态文件服务器、简化版 KV 存储、轻量 RPC 框架、Shell、Git、Blockchain / Cryptocurrency、Network Stack 已拆成 12-18 篇专项笔记，细化了模块拆分、实现顺序和验收标准。
- 更偏工程化：结合 0Voice 的后端、高性能网络、存储、基础架构、安全模块，把项目验收标准补成协议、并发、降级、重启、排障、回滚六项。
- 更偏竞赛极限：LCT、树套树高级变体、带修改更复杂查询、几何高级专题。
- 更偏工程深水区：RPC、存储引擎、日志系统、内存池、协程。

## 5. 最终结论

如果按最初规划来验收，这套 C++ 笔记现在已经可以认为是**完成版**：

- 内容层面：达标。
- 索引层面：闭环。
- 口径层面：主线与模板库分离清楚。

后续继续补充当然可以，但那属于“扩展优化”，不再是“是否达标”的问题。
