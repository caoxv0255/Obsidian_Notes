# 操作系统实验：Shell 与线程同步

## 实验目标

通过两个小实验，把“进程创建/回收”和“线程同步”真正做出来，而不是停留在概念层。

## 实验 A：简化 Shell

### 实验要求

实现一个支持以下功能的简化 shell：

- 读取命令行输入
- 解析命令和参数
- 使用 `fork` 创建子进程
- 使用 `exec` 执行外部程序
- 使用 `wait` 回收子进程
- 支持 `>`、`<` 和 `|`

### 推荐实现步骤

1. 先实现单命令执行。
2. 再实现命令参数解析。
3. 然后加入输入输出重定向。
4. 最后加入管道和简单后台执行。

### 需要记录的内容

- 命令解析规则
- 子进程与父进程分工
- 管道连接方式
- 错误处理方式

## 实验 B：线程同步

### 实验要求

实现一个多线程计数器程序：

- 版本 1：不加锁，观察错误结果
- 版本 2：加互斥锁，修复错误
- 版本 3：尝试原子操作或更细粒度的同步

### 观察重点

- 为什么无锁版本结果不稳定
- 锁粒度对性能有什么影响
- 线程数增加后是否一定更快

## 实验结果记录模板

### 1. 现象

- 运行结果是什么
- 是否出现错误输出

### 2. 分析

- 错误的根源是什么
- 哪一层概念对应了这个问题

### 3. 修复

- 采取了什么同步策略
- 为什么这样能修复

### 4. 延伸

- 如果把锁换成条件变量会怎样
- 如果任务量更大是否还合适

## 实验验收标准

- shell 能正确执行至少 5 个外部命令
- 线程同步版本输出正确且稳定
- 能写出一页以上的实验分析
