# 操作系统概述

## 1. 为什么需要操作系统

操作系统的核心作用，是在硬件资源和应用程序之间建立一层稳定、可管理、可扩展的抽象。没有操作系统，程序必须直接面对 CPU、内存、磁盘和设备，开发成本高，资源冲突严重，也很难做到并发执行与隔离保护。

可以把操作系统理解为三类“管理者”：

- 资源管理者：决定 CPU 时间、内存空间、磁盘带宽和设备访问权如何分配。
- 抽象提供者：把复杂硬件包装成进程、线程、文件、虚拟内存、套接字等简单对象。
- 安全边界维护者：限制不同程序互相破坏，防止越权访问和错误传播。

## 2. 操作系统提供的关键抽象

### 2.1 进程

进程是程序运行时的实例。它至少包含：

- 代码段：可执行指令
- 数据段：全局变量与静态变量
- 堆：动态分配内存
- 栈：函数调用与局部变量
- 进程控制块：PID、状态、寄存器上下文、调度信息

进程的意义在于“隔离”和“调度”。不同进程之间默认隔离，操作系统可以在它们之间切换 CPU 使用权。

### 2.2 线程

线程是进程内部的执行单元。一个进程可包含多个线程，共享地址空间和大部分资源，但各自拥有独立的寄存器上下文和栈。

线程的价值在于：

- 提高并发性
- 降低通信成本
- 更适合多核执行

但线程也会带来同步问题，因为共享内存使竞争条件更容易出现。

### 2.3 虚拟内存

虚拟内存把“每个进程都拥有一片连续、私有、足够大的地址空间”作为抽象。它背后通常依赖分页和页表：

- 程序看到的是虚拟地址
- MMU 将虚拟地址转换为物理地址
- 缺页异常由操作系统处理

虚拟内存的价值：

- 简化程序视角
- 保护不同进程
- 支持按需加载和更大的可用空间

### 2.4 文件与文件系统

文件是一种持久化抽象。文件系统负责：

- 命名：让用户用路径访问数据
- 组织：目录、索引节点、块映射
- 持久化：把内存中的变化落到磁盘
- 一致性：在异常中尽量保持可恢复

## 3. 操作系统的五条主线

### 3.1 抽象

操作系统把复杂硬件抽象为更容易使用的对象。例如：

- CPU → 进程 / 线程
- 内存 → 虚拟地址空间
- 磁盘 → 文件
- 网络设备 → 套接字

### 3.2 并发

多个任务同时推进时，操作系统负责调度和同步。

关键问题：

- 哪个任务先运行？
- 运行多久？
- 任务之间如何互不破坏？

### 3.3 共享

资源是有限的，必须共享。共享带来的问题是冲突，因此 OS 需要通过锁、信号量、调度、缓存和权限控制来管理。

### 3.4 隔离

隔离是系统稳定性的基础。一个程序崩溃，不能拖垮整个系统。进程隔离、权限隔离和地址空间隔离都是实现手段。

### 3.5 性能

操作系统不仅要“正确”，还要“高效”。性能优化通常体现在：

- 减少上下文切换
- 提高缓存命中率
- 利用局部性
- 合理批处理 I/O

## 4. 课程中最重要的概念链

建议把操作系统主线理解为以下链条：

程序 → 进程 → 线程 → 调度 → 同步 → 内存管理 → 文件系统 → I/O → 性能与安全

这个链条中任何一个环节都不应孤立理解。比如，线程调度会影响锁竞争；内存分配会影响缓存局部性；I/O 策略会影响文件系统性能。

## 5. 一个简单的系统调用视角

用户程序通常不能直接操作硬件，而是通过系统调用请求内核服务。例如：

1. 程序调用 `read()`
2. CPU 切换到内核态
3. 内核检查参数与权限
4. 内核从文件或设备读取数据
5. 数据返回用户空间

这条路径体现了 OS 的双重角色：既提供服务，也做保护。

## 6. 典型误区

- 把进程和程序混为一谈：程序是静态文件，进程是运行实体。
- 把线程当成轻量进程：线程共享地址空间，通信更快，但同步更难。
- 把虚拟内存理解为“磁盘内存”：虚拟内存是地址抽象，不等于交换区。
- 认为 OS 只是调度器：实际上它同时管理、抽象、保护和优化。

## 7. 课堂例题

### 例题 1：为什么单线程程序也需要操作系统？

因为单线程程序仍然需要：

- 文件输入输出
- 内存分配
- 时间片调度
- 错误保护
- 设备访问

### 例题 2：为什么进程隔离重要？

如果两个程序共享同一物理内存且没有保护，一个程序的越界写入可能覆盖另一个程序的数据甚至内核数据。隔离能把故障限制在局部范围内。

## 8. 项目式学习入口

建议围绕下面三个小项目推进：

### 项目 A：进程观察日志

- 目标：认识进程状态、调度和上下文切换。
- 做法：使用命令行工具记录一个程序从启动到退出的生命周期。
- 输出：进程状态图、命令输出截图、分析笔记。

### 项目 B：简化 shell

- 目标：理解进程创建、前后台切换和 I/O 重定向。
- 做法：实现命令解析、`fork/exec/wait` 逻辑和简单管道。
- 输出：设计文档、模块图和测试用例。

### 项目 C：并发正确性实验

- 目标：理解锁和同步原语的必要性。
- 做法：实现生产者-消费者或读者-写者问题。
- 输出：错误版本、修复版本、性能对比和分析。
