# I/O 与设备管理

## 1. 为什么 I/O 特别重要

绝大多数程序最终都要和外部世界交互：键盘、屏幕、磁盘、网络、传感器、打印机。I/O 的核心难点是：设备速度差异巨大，且通常需要中断、缓冲和驱动来协同。

## 2. 同步 I/O 与异步 I/O

### 同步 I/O

调用发出后，调用者等待操作完成。

### 异步 I/O

调用发出后，调用者先返回，完成后通过回调、事件或轮询得到结果。

异步 I/O 更适合高并发场景，但编程复杂度更高。

## 3. 中断

中断是设备通知 CPU“我有事情要处理”的机制。

中断的意义在于：

- 避免 CPU 轮询浪费
- 提高设备响应效率
- 允许并发处理多个设备事件

## 4. DMA

直接内存访问（DMA）允许设备与内存直接交换数据，减少 CPU 参与。

DMA 常用于：

- 磁盘传输
- 网络收发
- 大块数据搬运

## 5. 设备驱动

驱动程序是操作系统与硬件设备之间的桥梁。驱动负责：

- 初始化设备
- 发送命令
- 处理中断
- 管理缓冲区
- 向上层提供统一接口

## 6. 缓冲、缓存与假脱机

- 缓冲：平滑速度差异
- 缓存：减少重复访问成本
- 假脱机：把慢设备变成可排队的服务

这些机制本质上都在解决设备慢、程序快的不匹配问题。

## 7. 设备管理的常见问题

- 设备繁忙时如何排队
- 多进程如何共享设备
- 如何处理中断风暴
- 如何保证数据写入顺序

## 8. 与文件系统和网络的关系

I/O 并不孤立。磁盘读写是文件系统的基础，网络收发则涉及套接字和协议栈。因此 I/O 的本质是“让设备能被操作系统统一管理”。

## 9. 编程实践建议

学习 I/O 时，应特别注意：

- 文件描述符和句柄的概念
- 缓冲区大小对性能的影响
- 阻塞、非阻塞与超时
- 日志和错误处理

## 10. 项目式学习入口

### 项目 A：文件复制工具

- 目标：理解块读取和缓冲。
- 做法：实现一个支持进度显示的文件复制程序。
- 输出：缓冲策略、性能对比、错误处理。

### 项目 B：事件驱动 I/O 示例

- 目标：理解异步事件循环。
- 做法：实现一个简单事件驱动的多连接服务器模型。
- 输出：事件循环图、状态说明、测试记录。

### 项目 C：设备中断模拟

- 目标：理解设备如何通知 CPU。
- 做法：用程序模拟设备完成后发出事件的流程。
- 输出：中断流程图、处理逻辑、设计说明。

## 11. 学习检查

- 能否说明为什么需要中断？
- 能否解释 DMA 的价值？
- 能否区分缓冲和缓存？
- 能否把 I/O 和文件系统、网络联系起来？
