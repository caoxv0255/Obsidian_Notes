# 进程与线程

## 1. 从程序到进程

程序是静态文件，进程是程序运行时的实例。操作系统把程序装入内存、分配资源、建立地址空间、初始化寄存器上下文之后，程序才真正变成进程。

一个进程通常包含：

- 可执行代码
- 数据段和静态区
- 堆
- 栈
- 打开文件表
- 进程控制块（PCB）

PCB 是操作系统管理进程的核心数据结构，保存进程状态、寄存器、调度信息、内存映射和资源引用。

## 2. 进程的生命周期

典型状态包括：

- 新建：刚被创建
- 就绪：等待 CPU
- 运行：正在占用 CPU
- 阻塞：等待 I/O 或某个事件
- 终止：执行结束或被杀死

状态转换反映了操作系统对 CPU 和事件的管理：

- 就绪 → 运行：被调度器选中
- 运行 → 就绪：时间片用完或被抢占
- 运行 → 阻塞：发起 I/O 或等待同步事件
- 阻塞 → 就绪：等待事件完成

## 3. 进程创建与退出

在类 Unix 系统中，常见流程是：

1. `fork()` 复制当前进程
2. 子进程在必要时调用 `exec()` 装载新程序
3. 父进程调用 `wait()` 或 `waitpid()` 收集子进程状态

这个模型体现了“先复制，再替换”的思想，既灵活又便于构造 shell、管道和作业控制。

## 4. 上下文切换

上下文切换是操作系统保存当前进程/线程的执行现场，并恢复另一个执行实体的现场。

保存和恢复的内容通常包括：

- 程序计数器
- 栈指针
- 通用寄存器
- 状态寄存器
- 内核相关调度信息

上下文切换的代价不来自“切换动作本身”，而来自它带来的缓存失效、流水线扰动和调度开销。

## 5. 线程为什么重要

线程是进程内部的执行流。多个线程共享：

- 地址空间
- 代码段
- 打开的文件

但各自拥有：

- 独立栈
- 独立寄存器上下文
- 独立调度状态

线程更轻量，适合并发任务拆分；但因为共享内存，所以同步难度更高。

## 6. 线程与进程的区别

| 维度 | 进程 | 线程 |
|---|---|---|
| 地址空间 | 独立 | 共享 |
| 创建代价 | 较高 | 较低 |
| 通信成本 | 通过 IPC | 通过共享内存 |
| 崩溃影响 | 通常局限于单进程 | 可能影响整个进程 |
| 调度单位 | 是 | 是 |

理解这个表格的关键是：线程并不是“更小的进程”，而是共享资源更多的执行单元。

## 7. 并发带来的问题

多个线程同时操作共享数据时，可能出现：

- 竞态条件
- 不一致读写
- 死锁
- 活锁
- 饥饿

因此，线程编程的核心不只是“并发”，而是“正确并发”。

## 8. 同步工具

常见同步机制包括：

- 互斥锁：保护临界区
- 读写锁：优化读多写少场景
- 信号量：控制资源计数
- 条件变量：等待某个条件成立

选择同步原语时要考虑：

- 保护的数据范围
- 竞争激烈程度
- 等待模型
- 性能开销

## 9. 一个经典案例：生产者-消费者

生产者向缓冲区放数据，消费者从缓冲区取数据。核心问题是：

- 缓冲区满了怎么办
- 缓冲区空了怎么办
- 如何防止两个线程同时修改计数器

这个模型可以用锁 + 条件变量，也可以用信号量解决。

## 10. 编程实践建议

在学习进程与线程时，最好做两类练习：

### 练习 A：观察进程

- 启动一个程序
- 记录 PID、状态、父子关系
- 观察其 I/O、内存和退出行为

### 练习 B：写一个多线程小程序

- 实现计数器
- 故意不加锁观察错误
- 再加锁修复并比较结果

## 11. 项目式学习入口

### 项目 A：简化 shell

- 目标：理解进程创建与回收。
- 产出：命令解析、`fork/exec/wait`、前后台控制。

### 项目 B：线程池

- 目标：理解线程复用与任务分发。
- 产出：任务队列、工作线程、停止机制、错误处理。

### 项目 C：并发错误实验

- 目标：亲眼看到竞态条件。
- 产出：错误版本、修复版本、运行日志和分析。

## 12. 学习检查

- 能否解释进程和线程的差异？
- 能否说清状态转换的条件？
- 能否解释上下文切换的开销来自哪里？
- 能否说明为什么共享数据一定要同步？
