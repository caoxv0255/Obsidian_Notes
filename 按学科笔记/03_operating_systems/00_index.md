## 操作系统（课程索引）

简短描述：面向学生的操作系统课程主页，按“概念理解 - 机制拆解 - 小型实现 - 综合项目”组织学习。

## 课程定位

操作系统是连接硬件与应用程序的核心系统软件。本课程不仅要掌握进程、线程、内存、文件系统和 I/O 等基础概念，更要理解“抽象、并发、共享、隔离、性能”这五条主线如何贯穿整个系统设计。

## 学习目标

- 理解操作系统提供的核心抽象：进程、线程、地址空间、文件、系统调用。
- 掌握并发控制与资源管理的基本方法：调度、同步、互斥、死锁避免。
- 理解虚拟内存、分页、缓存、文件系统与设备管理的实现思路。
- 能用实验或小项目把“概念”转化为“可观察的系统行为”。

## 先修知识

- 计算机组成基础：指令执行、内存层次、异常与中断。
- 程序设计基础：C/C++ 或 Java 的函数、指针/引用、数据结构。
- 基本离散数学：状态机、图、集合、关系。

## 知识地图

1. 操作系统概述：为什么需要 OS，OS 的边界在哪里。
2. 进程与线程：并发执行、上下文切换、用户态与内核态。
3. CPU 调度：响应时间、吞吐量、公平性与调度算法。
4. 同步与互斥：临界区、锁、信号量、条件变量。
5. 死锁：四个必要条件、处理策略、资源分配图。
6. 内存管理：分页、段页式、虚拟内存、缺页处理。
7. 文件系统：目录、inode、缓存、日志与一致性。
8. I/O 与设备管理：中断、DMA、驱动、缓冲与异步 I/O。

## 项目式学习路径

### 阶段 0：环境与观察

- 目标：理解系统调用与进程状态变化。
- 任务：使用命令行工具观察进程、线程、文件、内存与磁盘 I/O。
- 产出：一份“系统观察笔记”，记录 10 个你能直接观察到的 OS 行为。

### 阶段 1：调度与并发

- 目标：实现一个简化的任务调度器或线程池模型。
- 任务：设计任务队列、优先级、时间片与等待队列；比较 FIFO、RR、Priority 的差异。
- 产出：流程图、伪代码、指标表（等待时间/响应时间/吞吐量）。

### 阶段 2：同步与资源管理

- 目标：把“并发正确性”落到可验证的实验上。
- 任务：设计生产者-消费者、读者-写者或哲学家就餐场景。
- 产出：问题定义、错误示例、修复方案、正确性说明。

### 阶段 3：内存与文件系统

- 目标：理解虚拟内存与持久化。
- 任务：分别模拟页表访问和一个简化文件分配/目录管理机制。
- 产出：地址转换示意图、文件组织图、缓存与一致性说明。

### 阶段 4：综合项目

- 目标：将课程主线整合为一个可解释的小型系统。
- 任务建议：
  - 实现一个迷你 shell，支持作业控制、管道、重定向与简单内建命令。
  - 或实现一个简化内存分配器，记录分配、释放、碎片与回收。
- 产出：设计文档、模块图、实验日志、性能与局限分析。
