## 📚 核心内容

### 2.1 引言

#### 指令集架构(ISA, Instruction Set Architecture)

**定义**：ISA是硬件与软件之间的接口，定义了程序员可见的所有指令和寄存器

**ISA的组成部分**：
- 指令格式
- 指令操作码
- 寄存器集合
- 寻址方式
- 异常处理

#### RISC-V简介

**RISC-V**：一个开源的精简指令集计算机(RISC)架构

**特点**：
- 简洁优雅
- 模块化设计
- 开源免费
- 现代化设计

**版本**：
- RV32I：32位整数指令集
- RV64I：64位整数指令集
- 扩展：M（乘除）、A（原子操作）、F（单精度浮点）、D（双精度浮点）

### 2.2 计算机硬件的操作

#### RISC-V寄存器

**通用寄存器**：32个64位寄存器（x0-x31）

| 寄存器 | 别名 | 用途 | 保存者 |
|--------|------|------|--------|
| x0 | zero | 常数0 | - |
| x1 | ra | 返回地址 | 调用者 |
| x2 | sp | 栈指针 | 被调用者 |
| x3 | gp | 全局指针 | - |
| x4 | tp | 线程指针 | - |
| x5-x7 | t0-t2 | 临时寄存器 | 调用者 |
| x8 | s0/fp | 保存寄存器/帧指针 | 被调用者 |
| x9 | s1 | 保存寄存器 | 被调用者 |
| x10-x11 | a0-a1 | 函数参数/返回值 | 调用者 |
| x12-x17 | a2-a7 | 函数参数 | 调用者 |
| x18-x27 | s2-s11 | 保存寄存器 | 被调用者 |
| x28-x31 | t3-t6 | 临时寄存器 | 调用者 |

#### 算术运算指令

**基本算术指令**：

| 指令 | 格式 | 操作 | 示例 |
|------|------|------|------|
| add | add rd, rs1, rs2 | rd = rs1 + rs2 | add x5, x6, x7 |
| sub | sub rd, rs1, rs2 | rd = rs1 - rs2 | sub x5, x6, x7 |
| addi | addi rd, rs1, imm | rd = rs1 + imm | addi x5, x6, 10 |

**立即数指令**：
- `addi`：加立即数
- 立即数范围：-2048到+2047（12位有符号）

#### 示例

**C代码**：
```c
f = (g + h) - (i + j);
```

**RISC-V汇编**：
```assembly
add  t0, x6, x7    # t0 = g + h
add  t1, x8, x9    # t1 = i + j
sub  x5, t0, t1    # f = t0 - t1
```

### 2.3 计算机硬件的操作数

#### 操作数类型

1. **寄存器**：快速访问，数量有限
2. **立即数**：常量，编码在指令中
3. **内存**：大容量，访问较慢

#### 内存访问指令

**加载指令(Load)**：
- `lb rd, offset(rs1)`：加载字节
- `lh rd, offset(rs1)`：加载半字
- `lw rd, offset(rs1)`：加载字
- `ld rd, offset(rs1)`：加载双字

**存储指令(Store)**：
- `sb rs2, offset(rs1)`：存储字节
- `sh rs2, offset(rs1)`：存储半字
- `sw rs2, offset(rs1)`：存储字
- `sd rs2, offset(rs1)`：存储双字

#### 字节寻址与字寻址

**字节寻址**：每个字节有一个地址
**字寻址**：地址必须是字长的倍数

**示例**：
```assembly
lw  x5, 0(x10)     # 加载地址x10的字
lw  x6, 4(x10)     # 加载地址x10+4的字
sw  x5, 8(x10)     # 存储x5到地址x10+8
```

#### 示例

**C代码**：
```c
A[12] = h + A[8];
```

**假设**：
- `A`是整数数组
- `A`的地址在`x10`
- `h`在`x11`

**RISC-V汇编**：
```assembly
lw   x5, 32(x10)    # x5 = A[8] (8 * 4 = 32)
add  x5, x5, x11    # x5 = x5 + h
sw   x5, 48(x10)    # A[12] = x5 (12 * 4 = 48)
```

### 2.4 有符号数和无符号数

#### 数的表示

**无符号数**：所有位都表示数值
**有符号数**：最高位是符号位（0为正，1为负）

#### 补码表示

**正数**：原码表示
**负数**：补码表示 = 反码 + 1

**示例**：
```
+5  = 0000 0101
-5  = 1111 1011 (原码: 1000 0101, 反码: 1111 1010, 补码: 1111 1011)
```

#### 有符号运算指令

| 指令 | 操作 | 说明 |
|------|------|------|
| slt | rd = (rs1 < rs2) ? 1 : 0 | 有符号比较 |
| sltu | rd = (rs1 < rs2) ? 1 : 0 | 无符号比较 |
| slti | rd = (rs1 < imm) ? 1 : 0 | 有符号比较立即数 |
| sltiu | rd = (rs1 < imm) ? 1 : 0 | 无符号比较立即数 |

### 2.5 计算机中指令的表示

#### RISC-V指令格式

**R型（寄存器型）**：
```
31 25   24 20   19 15   14 12   11 7    6 0
funct7  rs2     rs1     funct3  rd      opcode
7 bits  5 bits  5 bits  3 bits  5 bits  7 bits
```

**I型（立即数型）**：
```
31 20           19 15   14 12   11 7    6 0
imm[11:0]      rs1     funct3  rd      opcode
12 bits        5 bits  3 bits  5 bits  7 bits
```

**S型（存储型）**：
```
31 25   24 20   19 15   14 12   11 7    6 0
imm[11:5] rs2   rs1     funct3  imm[4:0] opcode
7 bits  5 bits  5 bits  3 bits  5 bits  7 bits
```

**B型（分支型）**：
```
31    25   24 20   19 15   14 12   11 8  7   6 0
imm[12|10:5] rs2 rs1     funct3  imm[4:1|11] opcode
1+6 bits  5 bits 5 bits  3 bits  1+4+1 bits  7 bits
```

**U型（立即数型）**：
```
31 12                           11 7    6 0
imm[31:12]                      rd      opcode
20 bits                         5 bits  7 bits
```

**J型（跳转型）**：
```
31          30 21   20 19   18 12   11 7    6 0
imm[20|10:1|11|19:12]          rd      opcode
1+10+1+8 bits                  5 bits  7 bits
```

#### 指令编码示例

**add x5, x6, x7**：
- opcode = 0110011 (R型)
- rd = 00101 (x5)
- funct3 = 000
- rs1 = 00110 (x6)
- rs2 = 00111 (x7)
- funct7 = 0000000

**机器码**：`0000000 00111 00110 000 00101 0110011`

### 2.6 逻辑运算

#### 逻辑运算指令

| 指令 | 操作 | 说明 |
|------|------|------|
| and | rd = rs1 & rs2 | 按位与 |
| or | rd = rs1 \| rs2 | 按位或 |
| xor | rd = rs1 ^ rs2 | 按位异或 |
| andi | rd = rs1 & imm | 按位与立即数 |
| ori | rd = rs1 \| imm | 按位或立即数 |
| xori | rd = rs1 ^ imm | 按位异或立即数 |

#### 移位指令

| 指令 | 操作 | 说明 |
|------|------|------|
| sll | rd = rs1 << rs2 | 逻辑左移 |
| srl | rd = rs1 >> rs2 | 逻辑右移 |
| sra | rd = rs1 >> rs2 | 算术右移 |
| slli | rd = rs1 << shamt | 逻辑左移立即数 |
| srli | rd = rs1 >> shamt | 逻辑右移立即数 |
| srai | rd = rs1 >> shamt | 算术右移立即数 |

#### 示例

**位操作**：
```assembly
and  x5, x6, x7    # x5 = x6 & x7
or   x5, x6, x7    # x5 = x6 | x7
xor  x5, x6, x7    # x5 = x6 ^ x7
slli x5, x6, 2     # x5 = x6 << 2 (乘以4)
```

### 2.7 决策指令

#### 条件分支指令

| 指令 | 操作 | 说明 |
|------|------|------|
| beq | if (rs1 == rs2) goto PC + offset | 相等跳转 |
| bne | if (rs1 != rs2) goto PC + offset | 不等跳转 |
| blt | if (rs1 < rs2) goto PC + offset | 小于跳转 |
| bge | if (rs1 >= rs2) goto PC + offset | 大于等于跳转 |
| bltu | if (rs1 < rs2) goto PC + offset | 无符号小于跳转 |
| bgeu | if (rs1 >= rs2) goto PC + offset | 无符号大于等于跳转 |

#### 无条件跳转指令

| 指令 | 操作 | 说明 |
|------|------|------|
| jal | rd = PC + 4; PC = PC + offset | 跳转并链接 |
| jalr | rd = PC + 4; PC = rs1 + offset | 寄存器跳转并链接 |

#### 示例

**if语句**：
```c
if (i == j)
    f = g + h;
else
    f = g - h;
```

**RISC-V汇编**：
```assembly
bne  x12, x13, Else    # if (i != j) goto Else
add  x5, x6, x7        # f = g + h
beq  x0, x0, Done      # goto Done
Else:
    sub  x5, x6, x7    # f = g - h
Done:
```

**循环**：
```c
while (save[i] == k)
    i += 1;
```

**RISC-V汇编**：
```assembly
Loop:
    slli  t0, x10, 3     # t0 = i * 8 (双字大小)
    add   t0, x5, t0     # t0 = &save[i]
    ld    t1, 0(t0)      # t1 = save[i]
    bne   t1, x11, Exit  # if (t1 != k) goto Exit
    addi  x10, x10, 1    # i += 1
    beq   x0, x0, Loop   # goto Loop
Exit:
```

### 2.8 支持过程的计算机硬件

#### 过程调用指令

**调用过程**：
- `jal rd, offset`：跳转到目标地址，保存返回地址到rd
- `jalr rd, offset(rs1)`：跳转到rs1+offset，保存返回地址到rd

**返回**：
- `jalr x0, 0(x1)`：跳转到返回地址（x1是ra寄存器）

#### 寄存器使用约定

**调用者保存寄存器**（t0-t6, a0-a7）：
- 由调用过程保存
- 调用过程可以在过程调用后假设这些寄存器的值不变

**被调用者保存寄存器**（s0-s11）：
- 由被调用过程保存
- 被调用过程必须确保这些寄存器的值在返回时与调用时相同

#### 栈帧

**栈帧**：过程调用时在栈上分配的内存区域

**栈帧内容**：
- 返回地址
- 保存的寄存器
- 局部变量
- 参数（如果寄存器不够）

**示例**：
```
        ┌──────────────┐
        │              │
        │  栈帧(n)     │
        │              │
        ├──────────────┤
        │   返回地址   │  ← 旧sp
        ├──────────────┤
        │  保存的寄存器│
        ├──────────────┤
        │  局部变量    │
        ├──────────────┤
        │              │
        │  栈帧(n+1)   │
        │              │  ← 新sp
        └──────────────┘
```

#### 示例

**过程调用**：
```c
int sum(int a, int b) {
    return a + b;
}

int main() {
    int result = sum(3, 4);
    return result;
}
```

**RISC-V汇编**：
```assembly
main:
    addi  sp, sp, -16    # 分配栈帧
    sd    ra, 8(sp)      # 保存返回地址

    addi  a0, x0, 3      # a0 = 3
    addi  a1, x0, 4      # a1 = 4
    jal   ra, sum        # 调用sum

    addi  a0, a0, 0      # 返回值在a0

    ld    ra, 8(sp)      # 恢复返回地址
    addi  sp, sp, 16     # 释放栈帧
    jalr  x0, 0(ra)      # 返回

sum:
    add   a0, a0, a1     # a0 = a0 + a1
    jalr  x0, 0(ra)      # 返回
```

### 2.10 RISC-V的大立即数和地址寻址

#### 大立即数

**lui指令**：加载立即数高位
```
lui rd, imm
rd = imm << 12
```

**组合加载大立即数**：
```assembly
lui  x5, 0x12345        # x5 = 0x12345000
addi x5, x5, 0x678      # x5 = 0x12345678
```

#### 伪指令

**伪指令**：不是真实指令，由汇编器转换为真实指令

**常用伪指令**：
- `li rd, imm`：加载立即数
- `mv rd, rs`：移动寄存器
- `not rd, rs`：按位取反
- `neg rd, rs`：取负
- `ret`：返回（等同于`jalr x0, 0(x1)`）
- `call offset`：调用（等同于`jal x1, offset`）

### 2.11 并行性和指令：同步

#### 原子操作

**原子操作**：不可中断的操作

**LR/SC（Load-Reserved/Store-Conditional）**：
```assembly
lr.d  x5, (x6)      # 加载保留
# ... 修改x5 ...
sc.d  x5, x5, (x6)  # 条件存储（成功则x5=0，失败则x5=1）
```

**原子交换**：
```assembly
Loop:
    lr.d  x5, (x10)    # 加载值
    sc.d  x5, x5, (x10) # 尝试存储
    bnez  x5, Loop      # 如果失败则重试
```

### 2.13 C排序示例：综合应用

#### 冒泡排序

**C代码**：
```c
void sort(int a[], int n) {
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n-1; j++) {
            if (a[j] > a[j+1]) {
                int temp = a[j];
                a[j] = a[j+1];
                a[j+1] = temp;
            }
        }
    }
}
```

**RISC-V汇编**（简化版）：
```assembly
sort:
    addi  sp, sp, -32
    sd    ra, 24(sp)
    sd    s0, 16(sp)
    sd    s1, 8(sp)
    sd    s2, 0(sp)

    mv    s0, a0          # s0 = a
    mv    s1, a1          # s1 = n

    addi  s2, x0, 0       # i = 0
Loop_i:
    bge   s2, s1, End_i   # if (i >= n) goto End_i

    addi  t0, x0, 0       # j = 0
Loop_j:
    addi  t1, s1, -1      # t1 = n-1
    bge   t0, t1, End_j   # if (j >= n-1) goto End_j

    slli  t2, t0, 2       # t2 = j * 4
    add   t2, s0, t2      # t2 = &a[j]
    lw    t3, 0(t2)       # t3 = a[j]
    lw    t4, 4(t2)       # t4 = a[j+1]

    ble   t3, t4, No_swap # if (a[j] <= a[j+1]) goto No_swap

    # 交换
    sw    t4, 0(t2)       # a[j] = t4
    sw    t3, 4(t2)       # a[j+1] = t3

No_swap:
    addi  t0, t0, 1       # j++
    beq   x0, x0, Loop_j  # goto Loop_j

End_j:
    addi  s2, s2, 1       # i++
    beq   x0, x0, Loop_i  # goto Loop_i

End_i:
    ld    ra, 24(sp)
    ld    s0, 16(sp)
    ld    s1, 8(sp)
    ld    s2, 0(sp)
    addi  sp, sp, 32
    jalr  x0, 0(ra)
```

### 2.16-2.19 实际案例：不同指令集架构

#### MIPS指令集

**特点**：
- RISC架构
- 固定长度指令（32位）
- 32个32位寄存器

**与RISC-V对比**：
- MIPS：更复杂，有更多指令
- RISC-V：更简洁，更现代化

#### ARM指令集

**ARMv7（32位）**：
- Thumb-2指令集
- 条件执行
- 16/32位混合指令

**ARMv8（64位）**：
- AArch64
- 31个64位寄存器
- 更大的立即数

#### x86指令集

**特点**：
- CISC架构
- 变长指令
- 复杂寻址方式
- 向后兼容

**与RISC-V对比**：
- x86：复杂，历史包袱
- RISC-V：简洁，现代化

### 2.20 实际案例：RISC-V的其他指令

#### RISC-V扩展

**M扩展（乘除）**：
```assembly
mul  rd, rs1, rs2    # 乘法
mulh rd, rs1, rs2    # 有符号乘法高位
div  rd, rs1, rs2    # 除法
rem  rd, rs1, rs2    # 余数
```

**A扩展（原子操作）**：
```assembly
lr.w  rd, (rs1)      # 加载保留

amoswap.w rd, rs2, (rs1)  # 原子交换
```

**F/D扩展（浮点）**：
```assembly
fadd.s fd, fs1, fs2  # 单精度加法
fsub.d fd, fs1, fs2  # 双精度减法
fmul.s fd, fs1, fs2  # 单精度乘法
fdiv.d fd, fs1, fs2  # 双精度除法
```

### 2.21 加速运行：C语言矩阵乘法

#### 优化技术

1. **循环展开**：减少循环开销
2. **分块**：提高缓存利用率
3. **SIMD指令**：数据级并行
4. **多线程**：任务级并行

#### 示例

**优化后的矩阵乘法**：
```c
void matrix_multiply_optimized(float *A, float *B, float *C, int n) {
    for (int i = 0; i < n; i += BLOCK_SIZE) {
        for (int j = 0; j < n; j += BLOCK_SIZE) {
            for (int k = 0; k < n; k += BLOCK_SIZE) {
                // 分块计算
                for (int ii = i; ii < i + BLOCK_SIZE; ii++) {
                    for (int jj = j; jj < j + BLOCK_SIZE; jj++) {
                        float sum = 0;
                        for (int kk = k; kk < k + BLOCK_SIZE; kk++) {
                            sum += A[ii*n + kk] * B[kk*n + jj];
                        }
                        C[ii*n + jj] += sum;
                    }
                }
            }
        }
    }
}
```

### 2.22 误区与陷阱

#### 常见误区

1. **误区**：汇编语言总是比高级语言快
   **事实**：现代编译器优化很好，汇编语言不一定更快

2. **误区**：寄存器越多越好
   **事实**：寄存器越多，上下文切换开销越大

3. **误区**：立即数越大越好
   **事实**：大立即数需要更多指令位，增加指令长度

#### 常见陷阱

1. **陷阱**：忘记保存被调用者保存的寄存器
   **后果**：程序崩溃

2. **陷阱**：栈不平衡
   **后果**：返回地址错误

3. **陷阱**：混淆有符号和无符号比较
   **后果**：逻辑错误

### 检查问题

7. 比较RISC-V和MIPS指令集的差异。
8. 什么是伪指令？举例说明。
9. 如何在RISC-V中访问数组元素？
10. 解释汇编程序、编译器和链接器的作用。

### 答案

<details>
<summary>点击查看答案</summary>

1. **32个**通用寄存器（x0-x31）

2. **六种指令格式**：
   - R型（寄存器型）
   - I型（立即数型）
   - S型（存储型）
   - B型（分支型）
   - U型（立即数型）
   - J型（跳转型）

3. **区别**：
   - **调用者保存寄存器**：由调用过程保存，调用过程可以在过程调用后假设这些寄存器的值不变
   - **被调用者保存寄存器**：由被调用过程保存，被调用过程必须确保这些寄存器的值在返回时与调用时相同

4. **栈帧**：过程调用时在栈上分配的内存区域
   - **内容**：返回地址、保存的寄存器、局部变量、参数

5. **条件分支**：使用分支指令（beq、bne、blt、bge等）
   - 例如：`beq x5, x6, label`（如果x5等于x6，跳转到label）

6. **LR/SC**：
   - **LR**：加载保留，加载内存值并标记该地址
   - **SC**：条件存储，如果地址未被修改则存储成功，否则失败
   - **作用**：实现原子操作

7. **RISC-V vs MIPS**：
   - RISC-V：更简洁、更现代化、开源
   - MIPS：更复杂、有更多指令、历史更长

8. **伪指令**：不是真实指令，由汇编器转换为真实指令
   - **示例**：
     - `mv rd, rs` → `add rd, rs, x0`
     - `ret` → `jalr x0, 0(x1)`

9. **访问数组**：
   - 计算地址：`base + index * element_size`
   - 示例：`slli t0, x10, 2; add t0, x5, t0; lw x6, 0(t0)`

10. **作用**：
    - **汇编器**：将汇编代码转换为机器代码
    - **编译器**：将高级语言代码转换为汇编代码
    - **链接器**：将多个目标文件合并，解析符号引用，生成可执行文件

</details>
