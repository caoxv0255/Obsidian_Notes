### 1.2 计算机架构的七大思想

这些思想是过去60年计算机设计的核心，至今仍在影响现代计算机架构：

#### 1. 使用抽象简化设计 ⬇️
**核心思想**：通过在不同层次隐藏细节，简化设计复杂性

**图标**：抽象画 🎨

**应用示例**：
- 软件层次：高级语言 → 汇编语言 → 机器语言
- 硬件层次：ISA → 微架构 → 电路
- 存储层次：寄存器 → 缓存 → 主存 → 磁盘

**优势**：
- 提高设计效率
- 降低错误率
- 便于协作开发

#### 2. 加速常见情况 🏎️
**核心思想**：优化最常发生的情况比优化罕见情况更能提升整体性能

**图标**：跑车（跑车比面包车更容易做得快）

**应用示例**：
- CPU设计：优化常用指令
- 缓存设计：提高局部性访问
- 分支预测：预测常见分支方向

**前提条件**：
- 需要通过测量确定"常见情况"
- 需要量化分析

#### 3. 通过并行提升性能 ✈️
**核心思想**：同时执行多个操作以提高性能

**图标**：多引擎飞机

**并行层次**：
- **位级并行**：同时处理多个位
- **指令级并行**：同时执行多条指令
- **数据级并行**：同时对多个数据执行相同操作
- **任务级并行**：同时执行多个任务

#### 4. 通过流水线提升性能 🔄
**核心思想**：将任务分解为多个阶段，不同阶段可以重叠执行

**图标**：管道系统

**应用示例**：
- CPU流水线：取指 → 译码 → 执行 → 访存 → 写回
- 工业流水线：汽车装配线
- 数据流水线：数据流处理

**关键问题**：
- 流水线冒险
- 流水线停顿
- 流水线转发

#### 5. 通过预测提升性能 🔮
**核心思想**：在不确定的情况下猜测并开始执行，如果猜测错误则回退

**图标**：水晶球

**应用示例**：
- 分支预测：预测跳转方向
- 预取：预取数据到缓存
- 推测执行：推测性地执行可能需要的代码

**关键要素**：
- 预测准确率
- 回退成本
- 预测收益

#### 6. 存储层次结构 🏔️
**核心思想**：通过多级存储器平衡速度、容量和成本

**图标**：分层三角形

**层次结构**：
```
        寄存器 (最快、最小、最贵)
          ↓
         L1缓存
          ↓
         L2缓存
          ↓
         L3缓存
          ↓
        主存 (DRAM)
          ↓
        磁盘/SSD
          ↓
        网络存储 (最慢、最大、最便宜)
```

**设计原则**：
- 上层存储器是下层存储器的缓存
- 利用时间局部性和空间局部性
- 管理数据一致性

#### 7. 通过冗余提高可靠性 🚛
**核心思想**：通过重复组件提高系统可靠性

**图标**：双轮卡车

**应用示例**：
- **冗余硬件**：双电源、双网络
- **冗余数据**：RAID、校验码、ECC
- **冗余计算**：容错计算、检查点
- **冗余时间**：重试机制

| 机器语言 | 二进制表示，硬件执行 | `1001010100101110` | CPU |

#### 代码示例

**C语言代码**：
```c
void swap(int v[], int k) {
    int temp;
    temp = v[k];
    v[k] = v[k+1];
    v[k+1] = temp;
}
```

**RISC-V汇编代码**：
```assembly
swap:
    slli  x6, x11, 3     # x6 = k * 8 (字长乘法)
    add   x6, x10, x6    # x6 = &v[k]
    lw    x5, 0(x6)      # temp = v[k]
    lw    x7, 4(x6)      # x7 = v[k+1]
    sw    x7, 0(x6)      # v[k] = v[k+1]
    sw    x5, 4(x6)      # v[k+1] = temp
    jalr  x0, 0(x1)      # 返回
```

**机器代码**（十六进制）：
```
swap:
    00000000001101011001001100010011  # slli x6, x11, 3
    00000000011001010000001100110011  # add x6, x10, x6
    00000000000000110011001010000011  # lw x5, 0(x6)
    00000000100000110011001110000011  # lw x7, 4(x6)
    00000000011100110011000000100011  # sw x7, 0(x6)
    00000000010100110011010000100011  # sw x5, 4(x6)
    00000000000000001000000001100111  # jalr x0, 0(x1)
```

### 1.4 计算机内部

#### 计算机的五大经典组件 🖥️

```
┌───────────────┐
│    输入设备   │  ← 麦克风、键盘、鼠标、触摸屏
└───────┬───────┘
        │
        ↓
┌───────────────────────────────┐
│           处理器              │
│  ┌──────────┐  ┌──────────┐  │
│  │  数据通路│  │  控制单元│  │
│  └──────────┘  └──────────┘  │
└─────────┬─────────────────────┘
          │
          ↓
┌───────────────────────────────┐
│         存储器                │  ← RAM、缓存
└─────────┬─────────────────────┘
          │
          ↓
┌───────┴───────┐
│    输出设备   │  → 显示器、扬声器、打印机
└───────────────┘
```

**详细说明**：

1. **输入设备(Input)**
   - 作用：向计算机输入数据
   - 示例：键盘、鼠标、触摸屏、麦克风、摄像头
   - 特点：将物理信号转换为数字信号

2. **输出设备(Output)**
   - 作用：从计算机输出数据
   - 示例：显示器、扬声器、打印机
   - 特点：将数字信号转换为物理信号

3. **存储器(Memory)**
   - 作用：存储程序和数据
   - 类型：DRAM（主存）、SRAM（缓存）、Flash（存储）
   - 特点：快速访问、容量有限

4. **数据通路(Datapath)**
   - 作用：执行算术和逻辑运算
   - 组成：ALU、寄存器、多路选择器
   - 特点：执行实际的计算操作

5. **控制单元(Control Unit)**
   - 作用：控制数据通路、存储器和I/O设备
   - 组成：状态机、译码器、控制信号
   - 特点：解释和执行指令

### 1.5 构建处理器和存储器的技术

#### 集成电路(IC, Integrated Circuit)
**定义**：将数十到数十亿个晶体管集成在一个芯片上

**制造工艺**：
- **光刻**：使用光刻技术在硅片上绘制电路
- **蚀刻**：去除不需要的材料
- **掺杂**：添加杂质改变硅的导电性
- **金属化**：添加金属连接电路

**工艺节点**：
- 7nm（2018年）：A12芯片
- 5nm（2020年）：A14芯片
- 3nm（2022年）：A16芯片
- 2nm（2024年）：A18芯片

**摩尔定律**：
- **原始预测**（1965年）：集成电路资源每年翻倍
- **修正预测**（1975年）：集成电路资源每两年翻倍
- **当前状态**：摩尔定律已显著放缓

#### 摩尔定律的终结 ⚠️

**终结原因**：
1. **物理极限**：原子尺寸限制
2. **功耗问题**：Dennard缩放定律失效
3. **成本问题**：先进工艺成本指数增长
4. **经济问题**：少数公司能够承担研发成本

**影响**：
- 通用处理器性能提升放缓
- 专用领域架构(DSA)兴起
- 并行化成为提升性能的主要途径

#### 存储器技术

##### DRAM (Dynamic Random Access Memory)
**特点**：
- 动态：需要定期刷新
- 随机访问：访问时间与位置无关
- 密度高：每比特成本低
- 速度中等：访问时间约50ns

**应用**：主存储器

##### SRAM (Static Random Access Memory)
**特点**：
- 静态：不需要刷新
- 速度快：访问时间约1-10ns
- 密度低：每比特成本高
- 功耗低：工作时功耗高，待机功耗低

**应用**：缓存

##### Flash Memory
**特点**：
- 非易失：断电后数据不丢失
- 可擦除：可以多次擦写
- 密度高：每比特成本中等
- 速度慢：擦写速度慢于DRAM

**应用**：SSD、手机存储、USB闪存盘

### 1.6 性能

#### 性能的定义

**响应时间(Response Time)**
- 也称为执行时间(Execution Time)
- 从开始到完成的时间
- 包括CPU时间、I/O时间等

**吞吐量(Throughput)**
- 单位时间内完成的任务数量
- 也称为带宽(Bandwidth)

**关系**：响应时间越短，吞吐量越高

#### 性能评估公式

```
性能 = 1 / 执行时间
```

**相对性能**：
```
性能A / 性能B = 执行时间B / 执行时间A
```

#### 执行时间的组成

```
CPU时间 = 指令数 × CPI × 时钟周期时间
     或
CPU时间 = 指令数 × CPI / 时钟频率
```

**其中**：
- **指令数(Instructions)**：程序执行的指令数量
- **CPI (Cycles Per Instruction)**：每条指令的平均时钟周期数
- **时钟周期时间(Clock Cycle Time)**：一个时钟周期的长度
- **时钟频率(Clock Rate)**：每秒的时钟周期数（Hz）

#### 性能公式示例

假设：
- 指令数 = 1,000,000,000
- CPI = 2.0
- 时钟频率 = 2.5 GHz

则：
```
CPU时间 = 1,000,000,000 × 2.0 / 2,500,000,000
       = 0.8秒
```

#### 影响性能的因素

1. **算法效率**
   - 好的算法减少指令数
   - 时间复杂度：O(1) < O(log n) < O(n) < O(n log n) < O(n²)

2. **编程语言**
   - 高级语言通常生成更多指令
   - 编译器优化很重要

3. **编译器优化**
   - 减少指令数
   - 降低CPI
   - 提高流水线效率

4. **指令集架构**
   - RISC vs CISC
   - 指令复杂度
   - 寄存器数量

5. **处理器实现**
   - 时钟频率
   - 流水线深度
   - 乱序执行
   - 缓存大小

6. **存储系统**
   - 缓存命中率
   - 主存带宽
   - 虚拟内存

7. **I/O系统**
   - 磁盘速度
   - 网络带宽
   - 设备驱动效率

#### Amdahl定律

**公式**：
```
加速比 = 1 / [(1 - f) + f / s]
```

**其中**：
- **f**：可优化部分的比例
- **s**：可优化部分的加速比

**含义**：
- 性能提升受限于不可优化部分
- 只有优化整个系统的瓶颈才能显著提升性能

**示例**：
假设程序中有90%可以优化10倍，10%无法优化：

```
加速比 = 1 / [(1 - 0.9) + 0.9 / 10]
      = 1 / [0.1 + 0.09]
      = 1 / 0.19
      ≈ 5.26倍
```

**启示**：即使优化大部分代码，整体加速也受限于未优化部分

#### 性能评估指标

| 指标 | 定义 | 示例 |
|------|------|------|
| MIPS | 百万指令每秒 | 3000 MIPS |
| FLOPS | 每秒浮点运算次数 | 10 GFLOPS |
| 带宽 | 数据传输速率 | 10 GB/s |
| 延迟 | 响应时间 | 10 ms |

### 1.7 功耗墙 ⚡

#### 功耗与性能的关系

**基本公式**：
```
功耗 = 电容 × 电压² × 频率
```

**动态功耗**：
```
动态功耗 = 活动因子 × 电容 × 电压² × 频率
```

**静态功耗**：
```
静态功耗 = 漏电流 × 电压
```

#### Dennard缩放定律
**原始定律**：随着晶体管缩小，功耗密度保持不变
**失效时间**：约2005年
**失效原因**：
- 电压不能继续线性降低
- 漏电流问题日益严重
- 散热成为瓶颈

#### 功耗墙问题
**现象**：
- 继续提高时钟频率导致功耗指数增长
- 散热能力无法跟上功耗增长
- 性能提升与功耗提升不再同步

**解决方案**：
1. **多核处理器**：增加核心数量而不是提高频率
2. **功耗管理**：动态电压频率调整(DVFS)
3. **专用加速器**：专用领域架构(DSA)
4. **低功耗设计**：低功耗模式、电源门控

#### 能效指标

| 指标 | 定义 | 典型值 |
|------|------|--------|
| 功耗/性能 | 瓦特/MIPS | 0.1-1 W/MIPS |
| 性能/瓦特 | MIPS/瓦特 | 1-10 MIPS/W |
| 性能/瓦特-秒 | MIPS/焦耳 | 10-100 MIPS/J |

### 1.8 海变：从单处理器到多处理器

#### 单处理器的终结

**历史背景**：
- 单处理器性能持续提升数十年
- 摩尔定律提供更多晶体管
- 频率不断提高

**终结原因**：
1. **功耗墙**：无法继续提高频率
2. **功耗密度**：散热成为瓶颈
3. **并行化需求**：应用程序需要更多并发

#### 多核处理器的兴起

**定义**：在一个芯片上集成多个处理器核心

**优势**：
- 更好的能效
- 更高的吞吐量
- 更好的可靠性

**挑战**：
- **编程复杂**：并行编程困难
- **内存一致性**：多个核心共享内存
- **负载均衡**：合理分配任务
- **通信开销**：核心间通信成本

#### 并行类型

| 并行类型 | 描述 | 示例 |
|---------|------|------|
| **位级并行** | 同时处理多个位 | 64位处理器 |
| **指令级并行** | 同时执行多条指令 | 流水线、乱序执行 |
| **数据级并行** | 同时处理多个数据 | SIMD、GPU |
| **任务级并行** | 同时执行多个任务 | 多核处理器 |

#### 现代多核处理器示例

**Intel Core i7-6700**：
- 4个核心，8个线程
- 基础频率：3.4 GHz
- 最大加速频率：4.0 GHz
- 8 MB三级缓存

**ARM Cortex-A53**：
- 4个核心
- 频率：1.2-2.0 GHz
- 低功耗设计

### 1.10 加速运行：Python矩阵乘法

#### 矩阵乘法的Python实现

```python
def matrix_multiply(A, B, C, n):
    for i in range(n):
        for j in range(n):
            for k in range(n):
                C[i][j] += A[i][k] * B[k][j]
```

#### 性能问题

**Python性能**：
- 解释执行，速度慢
- 动态类型检查开销
- 缺乏优化

**性能瓶颈**：
- 三重循环：O(n³)复杂度
- 大量函数调用
- 缺乏并行化

#### 性能对比

| 实现 | 语言 | 相对性能 |
|------|------|---------|
| 原始Python | Python | 1x |

| C语言 | C | 1000x |
| 优化C + SIMD | C + SIMD | 10000x |
| 多核并行 | C + OpenMP | 50000x |

### 1.11 误区与陷阱

#### 常见误区

1. **误区**：提高时钟频率就能线性提升性能
   **事实**：受限于Amdahl定律，只能提升可优化部分

2. **误区**：更快的CPU总是能提升性能
   **事实**：受限于内存和I/O瓶颈

3. **误区**：功耗与性能成正比
   **事实**：功耗与电压的平方成正比，与频率成正比

4. **误区**：多核处理器自动提升性能
   **事实**：需要并行编程才能利用多核

#### 常见陷阱

1. **陷阱**：忽略Amdahl定律
   **后果**：过度优化非瓶颈部分

2. **陷阱**：过早优化
   **后果**：浪费时间，增加复杂度

3. **陷阱**：忽略功耗问题
   **后果**：设备过热，电池续航短

4. **陷阱**：假设硬件会自动优化
   **后果**：程序性能差，无法充分利用硬件

### 答案

<details>
<summary>点击查看答案</summary>

1. **五大经典组件**：输入、输出、存储、数据通路、控制

2. **七大架构思想**：
   - 使用抽象简化设计
   - 加速常见情况
   - 通过并行提升性能
   - 通过流水线提升性能
   - 通过预测提升性能
   - 存储层次结构
   - 通过冗余提高可靠性

3. **抽象的作用**：通过在不同层次隐藏细节，简化设计复杂性，提高设计效率，降低错误率

4. **Amdahl定律**：加速比 = 1 / [(1 - f) + f / s]
   - **启示**：性能提升受限于不可优化部分，只有优化整个系统的瓶颈才能显著提升性能

5. **摩尔定律终结原因**：物理极限、功耗问题、成本问题、经济问题
   - **影响**：通用处理器性能提升放缓，专用领域架构兴起，并行化成为提升性能的主要途径

6. **功耗与性能关系**：功耗 = 电容 × 电压² × 频率
   - 提高性能需要提高频率，但功耗与电压平方成正比，导致功耗增长快于性能增长

7. **多核处理器的原因**：单处理器性能提升受功耗墙限制，多核可以在不提高频率的情况下提升性能和吞吐量

8. **DRAM vs SRAM**：
   - DRAM：动态、需要刷新、密度高、速度中等、成本低
   - SRAM：静态、不需要刷新、密度低、速度快、成本高

9. **流水线**：将任务分解为多个阶段，不同阶段可以重叠执行
   - **提升性能**：通过并行执行不同阶段，提高指令吞吐量

10. **存储层次结构**：通过多级存储器平衡速度、容量和成本
    - **原理**：利用时间局部性和空间局部性，上层存储器是下层存储器的缓存
