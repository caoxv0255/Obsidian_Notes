# 计算机组成原理（课程索引）

简短描述：面向学生的计算机组成原理课程主页，按“数据表示 - 指令执行 - 处理器设计 - 性能与存储层次”组织学习。

## 课程定位

计算机组成原理研究的是“程序如何在硬件上被真正执行”。这门课的核心不是背概念，而是建立从逻辑门、数据通路、指令集到流水线、缓存、存储系统的完整硬件视角。

## 学习目标

- 理解数据的表示方式：补码、浮点、字符编码、地址与对齐。
- 理解指令系统与程序执行：取指、译码、执行、访存、写回。
- 理解 CPU 结构：单周期、多周期、流水线、分支预测与冒险处理。
- 理解存储层次：寄存器、Cache、主存、虚拟存储、外存。
- 能把一个程序的执行过程映射到硬件数据通路。

## 先修知识

- 程序设计基础：变量、控制流、函数、数组、指针/引用。
- 离散数学和逻辑基础：二进制、逻辑代数、组合电路。
- 基本系统知识：汇编、系统调用、内存和中断概念。

## 知识地图

1. 计算机抽象与性能：层次化结构、性能指标、冯·诺依曼模型。
2. 数据表示与运算：整数、浮点、编码、算术电路。
3. 指令系统：ISA、寻址方式、控制流、函数调用。
4. 处理器设计：单周期、多周期、流水线、控制信号。
5. 存储层次：Cache、主存、虚拟存储与局部性原理。
6. I/O 与并行：中断、DMA、总线、外设与多核基础。

## 项目式学习路径

### 阶段 0：数据与指令观察

- 目标：建立“机器如何表示数据”的直觉。
- 任务：分析补码、浮点和对齐在程序中的表现。
- 产出：数据表示对照表、位级别示意图、常见错误案例。

### 阶段 1：ISA 与汇编

- 目标：理解程序到指令的映射。
- 任务：写几个短汇编片段，观察函数调用、循环、条件分支和栈帧。
- 产出：指令编码表、调用栈示意图、寄存器使用总结。

### 阶段 2：处理器与流水线

- 目标：理解 CPU 是如何执行指令的。
- 任务：画出单周期和流水线的数据通路，标出冒险、停顿和转发位置。
- 产出：数据通路图、控制信号表、流水线时序图。

### 阶段 3：缓存与存储系统

- 目标：理解性能为何受存储层次影响。
- 任务：比较不同访问模式对 Cache 命中率的影响；模拟局部性。
- 产出：命中率分析、访存模式对比、性能瓶颈总结。

### 阶段 4：综合项目

- 目标：把 ISA、汇编、CPU 和存储层次连接起来。
- 任务建议：
	- 设计一个简化指令集解释器，模拟寄存器、内存和基本控制流。
	- 或设计一个简单流水线时序模拟器，观察冒险、停顿与转发。
- 产出：架构说明、指令格式文档、运行示例、性能分析。

## 课程检核清单

- 是否能解释一条指令从取指到写回的完整流程。
- 是否能区分单周期、多周期和流水线处理器。
- 是否能说明 Cache 命中、缺失与局部性的关系。
- 是否能解释虚拟地址如何映射到物理地址。
- 是否能从程序代码反推硬件行为。
