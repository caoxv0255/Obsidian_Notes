# 存储层次

## 1. 为什么需要存储层次

CPU 很快，内存较慢，磁盘更慢。如果只依赖一种存储，系统要么太贵，要么太慢。因此，计算机采用多级存储层次。

## 2. 局部性原理

- 时间局部性：最近访问的数据很可能再次访问
- 空间局部性：访问某个位置时，附近位置也可能被访问

这是 Cache 设计的基础。

## 3. Cache 的基本思想

Cache 把常用数据保存在更快、更小的存储器中，以减少平均访问时间。

Cache 设计要考虑：

- 映射方式
- 替换策略
- 写策略
- 命中率

## 4. 主存与虚拟存储

主存负责当前活跃数据。虚拟存储让程序看到更大的地址空间，并通过页面换入换出实现扩展。

## 5. 性能指标

- 命中率
- 缺失率
- 平均访问时间
- 带宽

## 6. 例题

### 例题 1：为什么数组访问比链表访问更容易命中 Cache？

因为数组具有更好的空间局部性，连续元素更可能被预取到 Cache。

### 例题 2：为什么 Cache 容量越大不一定越好？

因为更大的 Cache 可能带来更高的访问延迟和更大的硬件复杂度。
