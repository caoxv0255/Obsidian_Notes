# I/O 与并行

## 1. 为什么要学习 I/O 与并行

现代计算机不仅要“算得快”，还要“输入输出快、多个部件一起工作快”。I/O 与并行是把硬件性能释放出来的关键。

## 2. I/O 系统

I/O 涉及：

- 总线
- 中断
- DMA
- 设备控制器

这些部件负责让 CPU 与外设协同工作。

## 3. 中断机制

中断允许设备在需要 CPU 处理时发出通知，避免 CPU 轮询浪费。

## 4. DMA

DMA 让设备可以直接和内存交换数据，减少 CPU 参与，提高大块数据传输效率。

## 5. 并行处理基础

并行可以发生在多个层次：

- 指令级并行
- 数据级并行
- 任务级并行
- 多核并行

并行能提升性能，但也会带来同步、通信和负载均衡问题。

## 6. 多核与一致性

多核处理器需要考虑缓存一致性和任务调度，以避免同一数据在不同核心上出现不一致。

## 7. 例题

### 例题 1：为什么 DMA 能提高大文件传输效率？

因为 DMA 让设备直接搬运数据，减少 CPU 在数据搬运上的负担。

### 例题 2：为什么并行不一定线性加速？

因为并行会引入同步开销、通信开销和不可并行部分。
