# 第2章 完全信息动态博弈

## 章节学习目标

本章主要讨论完全信息动态博弈的基本理论，学生完成本章学习后应能够：

1. 掌握扩展型博弈的基本概念和表示方法
2. 深入理解子博弈完美纳什均衡的定义和求解方法
3. 掌握逆向归纳法的原理和应用
4. 理解重复博弈的基本概念和性质
5. 掌握有限重复博弈和无限重复博弈的均衡分析
6. 能够运用动态博弈分析现实中的序贯策略互动问题

## 2.1 扩展型博弈

### 2.1.1 扩展型博弈的定义

**定义**：扩展型博弈（Extensive-Form Game）是描述动态博弈的基本形式，它通过博弈树（Game Tree）来展示参与者行动的顺序、信息结构、策略选择和支付结果。

**扩展型博弈的组成要素**：

1. **参与者集合** N = {1, 2, ..., n}
2. **决策节点集合** X，包括：
   - 初始节点 x₀
   - 非终止节点（决策节点）
   - 终止节点（叶子节点）
3. **参与者函数** α: X → N ∪ {终止}，表示每个决策节点由哪个参与者决策
4. **行动集合** A(x)，表示在节点 x 可选的行动
5. **信息集集合** H = {H₁, H₂, ..., Hₖ}，满足：
   - 每个信息集包含同一参与者的决策节点
   - 同一信息集中的节点具有相同的可选行动
6. **支付函数** u: Z → ℝⁿ，其中 Z 是终止节点集合
7. **概率分布** p，对于自然（Nature）选择的节点

### 2.1.2 完美信息与不完美信息

**完美信息博弈**：
- 每个信息集只包含一个节点
- 参与者在决策时知道博弈的所有历史
- 例如：象棋、围棋

**不完美信息博弈**：
- 至少有一个信息集包含多个节点
- 参与者在决策时不知道某些过去的选择
- 例如：扑克、桥牌

**完美记忆博弈**：
- 参与者记得自己之前所做的所有选择和观察到的所有行动
- 大多数经济模型都假设完美记忆

### 2.1.3 策略的定义

**定义**：在扩展型博弈中，参与者 i 的策略是一个完整的行动计划，规定了在参与者 i 的每个信息集应该采取什么行动。

**策略的性质**：
1. 完整性：规定了所有可能情况下的行动
2. 事前性：策略在博弈开始前确定
3. 条件性：策略是信息的函数

**策略型表示**：
任何扩展型博弈都可以转换为策略型博弈，但可能造成信息损失。

### 2.1.4 博弈树的结构

**博弈树的组成部分**：

```
         根节点
          / \
         /   \
        /     \
      节点1   节点2
       / \     / \
      /   \   /   \
    叶子1 叶子2 叶子3 叶子4
```

**博弈树的性质**：
1. 每个节点（除根节点）只有一个父节点
2. 从根节点到任何节点只有一条路径
3. 每条路径对应一个可能的博弈结果

### 2.1.5 信息集的表示

**完美信息博弈**：每个节点是一个单独的信息集

```
         A
        / \
       a   b
      /     \
     B       B
    / \     / \
   c   d   c   d
```

**不完美信息博弈**：某些节点在同一信息集中

```
         A
        / \
       a   b
      /     \
     {B}   {B}
    / | \  / | \
   c  d  e c  d  e
```

这里两个 B 节点在同一信息集中，用虚线框表示。

### 2.1.6 扩展型博弈的例子

**例子1：进入博弈**

**博弈描述**：
企业A（在位者）已经在市场上，企业B（进入者）决定是否进入。如果B进入，A可以选择容忍或对抗。

**博弈树**：

```
         B
        / \
      进入 不进入
       /     \
      A       (0, 2)
     / \
  容忍 对抗
    \   \
  (1, 1) (-1, 0)
```

**支付**：
- B不进入：A得2，B得0
- B进入，A容忍：A得1，B得1
- B进入，A对抗：A得-1，B得0

**例子2：信任博弈**

**博弈描述**：
参与者1可以选择信任或不信任参与者2。如果1信任，2可以选择合作或背叛。

**博弈树**：

```
         1
        / \
      信任 不信任
       /     \
      2       (1, 1)
     / \
   合作 背叛
    \   \
  (2, 2) (0, 3)
```

**支付**：
- 1不信任：各得1
- 1信任，2合作：各得2
- 1信任，2背叛：1得0，2得3

### 2.1.7 策略与行为策略

**纯策略**：每个信息集确定选择一个行动

**混合策略**：纯策略上的概率分布

**行为策略**：在每个信息集独立地随机选择行动

**Kuhn 定理**：
在完美记忆博弈中，混合策略和行为策略是等价的。

**区别**：
- 混合策略：在博弈开始前选择一个完整的行动计划
- 行为策略：在每个决策节点独立地随机选择

## 2.2 子博弈完美纳什均衡

### 2.2.1 子博弈的定义

**定义**：扩展型博弈的子博弈满足：
1. 从单个信息集的节点开始
2. 包含该节点及其所有后续节点
3. 不分割任何信息集

**子博弈的性质**：
- 子博弈本身是一个完整的博弈
- 子博弈可以被独立分析
- 原博弈本身也是一个子博弈

**例子**：

在进入博弈中：
- 整个博弈是一个子博弈
- 从A的决策节点开始的部分也是一个子博弈

```
         B
        / \
      进入 不进入
       /     \
      A*      (0, 2)  ← 从A开始的子博弈
     / \
  容忍 对抗
    \   \
  (1, 1) (-1, 0)
```

### 2.2.2 子博弈完美纳什均衡的定义

**定义**：扩展型博弈的策略组合 s* 是子博弈完美纳什均衡（Subgame Perfect Nash Equilibrium, SPNE），如果：
1. s* 是原博弈的纳什均衡
2. s* 在每一个子博弈中都构成纳什均衡

**理解子博弈完美纳什均衡**：
- 消除了不可信威胁
- 每个子博弈都选择最优反应
- 是纳什均衡的精炼

**与纳什均衡的关系**：
- 每个子博弈完美纳什均衡都是纳什均衡
- 但纳什均衡不一定是子博弈完美的
- 子博弈完美纳什均衡剔除了不合理的纳什均衡

### 2.2.3 逆向归纳法

**逆向归纳法（Backward Induction）**：
从博弈树的最后阶段开始，逐步向前求解每个子博弈的均衡。

**逆向归纳法的步骤**：
1. 找到最后阶段的子博弈
2. 求解这些子博弈的纳什均衡
3. 用均衡支付替代这些子博弈
4. 重复上述步骤，直到初始节点

**逆向归纳法的条件**：
1. 博弈是有限期的
2. 博弈是完美信息的
3. 参与者数量有限

**例子：进入博弈的逆向归纳**

**第一步**：从A的决策开始
- A选择容忍：支付(1, 1)
- A选择对抗：支付(-1, 0)
- A的最优选择：容忍

**第二步**：从B的决策开始
- B进入：A会容忍，B得1
- B不进入：B得0
- B的最优选择：进入

**子博弈完美纳什均衡**：
(进入，容忍)

### 2.2.4 不可信威胁

**不可信威胁**：
在纳什均衡中，某些策略在子博弈中不是最优的，因此是不可信的。

**例子：进入博弈中的纳什均衡**

转换为策略型博弈：
- B的策略：{进入，不进入}
- A的策略：{如果进入则容忍，如果进入则对抗}

**策略型表示**：

| B \ A | 容忍 | 对抗 |
|:---:|:---:|:---:|
| **进入** | (1, 1) | (0, -1) |
| **不进入** | (0, 2) | (0, 2) |

**纳什均衡**：
1. (进入，容忍) - 子博弈完美
2. (不进入，对抗) - 不是子博弈完美

**分析**：
在第二个纳什均衡中，A威胁说如果B进入就对抗，但这个威胁是不可信的，因为如果B真的进入，A的最优选择是容忍（1 > -1）。

**子博弈完美纳什均衡**：
只有 (进入，容忍)

### 2.2.5 子博弈完美纳什均衡的例子

**例子1：序贯囚徒困境**

**博弈描述**：
囚徒A先选择是否认罪，然后囚徒B决定是否认罪。

**博弈树**：

```
         A
        / \
      认罪 不认罪
       /     \
      B       B
     / \     / \
   认罪 不认罪 认罪 不认罪
```

**支付矩阵**（同静态囚徒困境）：

| A \ B | 认罪 | 不认罪 |
|:---:|:---:|:---:|
| **认罪** | (-5, -5) | (-1, -10) |
| **不认罪** | (-10, -1) | (-2, -2) |

**逆向归纳**：
- 无论A选择什么，B的最优反应都是认罪
- A知道B会认罪，A的最优选择是认罪（-5 > -10）

**子博弈完美纳什均衡**：
(认罪，认罪)

**例子2：斯塔克尔伯格博弈**

**博弈描述**：
企业1（领导者）先选择产量 q₁，企业2（跟随者）观察到 q₁ 后选择产量 q₂。

**市场需求**：P = a - q₁ - q₂
**成本函数**：Cᵢ(qᵢ) = c·qᵢ

**逆向归纳**：

**第一阶段**：企业2的决策
给定 q₁，企业2最大化利润：
π₂(q₁, q₂) = (a - q₁ - q₂)q₂ - cq₂

一阶条件：
∂π₂/∂q₂ = a - q₁ - 2q₂ - c = 0
⇒ q₂ = (a - c - q₁)/2

这是企业2的最佳反应函数。

**第二阶段**：企业1的决策
企业1知道企业2的反应，最大化利润：
π₁(q₁) = (a - q₁ - q₂(q₁))q₁ - cq₁
= (a - q₁ - (a - c - q₁)/2)q₁ - cq₁
= (a - c - q₁)q₁/2

一阶条件：
∂π₁/∂q₁ = (a - c - 2q₁)/2 = 0
⇒ q₁ = (a - c)/2

**均衡**：
q₁* = (a - c)/2
q₂* = (a - c - q₁*)/2 = (a - c - (a - c)/2)/2 = (a - c)/4

**支付**：
π₁ = (a - c)²/8
π₂ = (a - c)²/16

**与古诺模型的比较**：
- 古诺：q₁ = q₂ = (a - c)/3
- 斯塔克尔伯格：q₁ = (a - c)/2，q₂ = (a - c)/4

领导者获得先动优势。

### 2.2.6 多阶段博弈

**多阶段可观察行动博弈**：
- 多个阶段
- 每个阶段所有参与者同时行动
- 所有参与者观察到之前的行动

**定义**：
参与者 i 的策略是一个从历史到行动的映射：
sᵢ: Hᵢ → Aᵢ

其中 Hᵢ 是参与者 i 观察到的历史集合。

**求解方法**：
对于有限多阶段博弈，可以使用逆向归纳法求解子博弈完美纳什均衡。

## 2.3 重复博弈

### 2.3.1 重复博弈的定义

**定义**：重复博弈是同一个基础博弈（阶段博弈）重复多次的动态博弈。

**重复博弈的要素**：
1. 阶段博弈（Stage Game）：每次重复的基础博弈
2. 重复次数：有限或无限
3. 贴现因子：δ ∈ [0, 1]
4. 信息结构：可观察行动或不可观察行动

**历史**：
t 时期的历史是前 t-1 期的行动序列：
hᵗ = (a¹, a², ..., aᵗ⁻¹)

**策略**：
参与者 i 的策略是从历史到行动的映射：
sᵢ: H → Aᵢ

**支付**：
有限重复博弈（T 期）：
Uᵢ = Σ_{t=1}^{T} δ^{t-1}uᵢ(aᵗ)

无限重复博弈：
Uᵢ = Σ_{t=1}^{∞} δ^{t-1}uᵢ(aᵗ)

### 2.3.2 无名氏定理（有限情形）

**定理**：如果阶段博弈有纳什均衡，那么对于任意 ε > 0，存在 T 使得重复博弈存在子博弈完美纳什均衡，其平均支付在 ε 范围内优于阶段博弈的纳什均衡支付。

**理解**：
- 在足够长的有限重复博弈中，参与者可以达成接近帕累托最优的结果
- 合作可以通过"触发策略"（Trigger Strategy）实现

**触发策略**：
- 在初始阶段合作
- 如果有人背叛，永远转向阶段博弈的纳什均衡

### 2.3.3 无限重复博弈的重要性

**无限重复博弈的特点**：
1. 没有最后一个阶段，逆向归纳法不直接适用
2. 未来威胁（永远不合作）可以维持当前的合作
3. 有更多实现合作的可能性

**贴现因子的含义**：
δ = 1/(1+r)，其中 r 是利率
- δ 接近 1：参与者很重视未来
- δ 接近 0：参与者只关注眼前利益

### 2.3.4 触发策略

**冷峻触发策略（Grim Trigger Strategy）**：
- 开始时合作
- 一旦有人背叛，永远不合作

**温和触发策略（Forgiving Trigger Strategy）**：
- 开始时合作
- 如果有人背叛，惩罚 K 期后重新合作

**有限惩罚策略（Finite Punishment）**：
- 开始时合作
- 如果有人背叛，惩罚 L 期后重新合作

### 2.3.5 重复囚徒困境

**阶段博弈**：

| 参与者1 \ 参与者2 | 合作 | 背叛 |
|:---:|:---:|:---:|
| **合作** | (3, 3) | (0, 5) |
| **背叛** | (5, 0) | (1, 1) |

**纳什均衡**：(背叛, 背叛)，支付(1, 1)

**帕累托最优**：(合作, 合作)，支付(3, 3)

**触发策略**：
- 开始时选择合作
- 如果对方曾经背叛，永远选择背叛

**合作的条件**：

如果双方都采取触发策略：
- 每期合作，每期支付 3
- 总支付：3 + 3δ + 3δ² + ... = 3/(1-δ)

如果一方在某期背叛：
- 当期支付：5
- 此后永远背叛，每期支付 1
- 总支付：5 + 1δ + 1δ² + ... = 5 + δ/(1-δ)

合作的条件：
3/(1-δ) ≥ 5 + δ/(1-δ)
3 ≥ 5 + δ - 5δ
3 ≥ 5 - 4δ
4δ ≥ 2
δ ≥ 1/2

**结论**：
当 δ ≥ 1/2 时，合作可以作为子博弈完美纳什均衡实现。

## 2.4 有限重复博弈

### 2.4.1 有限重复囚徒困境

**阶段博弈**：同上

**T 期重复博弈**：
- 使用逆向归纳法
- 在最后一期，双方都背叛
- 倒数第二期，双方知道最后一期会背叛，因此也背叛
- ...
- 从第一期开始，双方都背叛

**结论**：
有限重复囚徒博弈的唯一子博弈完美纳什均衡是每期都背叛。

**与直觉的矛盾**：
即使重复很多次，只要有限，合作也无法维持

**解释**：
- 最后一期的不可信威胁
- 逆向归纳的"连锁反应"

### 2.4.2 不完全信息与有限重复博弈

**关键问题**：
如果在有限重复囚徒困境中，引入不完全信息（不确定对手的类型），合作是否能维持？

**Kreps-Milgrom-Roberts-Wilson (KMRW) 模型**：
- 每个参与者有 ε 的概率是"合作型"（永远合作）
- 有 1-ε 的概率是"理性型"（最大化自身利益）
- 参与者不知道对手的类型

**结论**：
对于任意有限 T，存在 ε̄ > 0，使得如果 ε < ε̄，在前 T-K 期合作可以维持（K 依赖于 T）

**意义**：
- 不完全信息可以维持合作
- "声誉"的重要性
- 解释现实中的合作行为

### 2.4.3 阶段博弈有多个均衡的情况

**阶段博弈**：

| 参与者1 \ 参与者2 | A | B |
|:---:|:---:|:---:|
| **A** | (2, 2) | (0, 0) |
| **B** | (0, 0) | (1, 1) |

**纳什均衡**：(A, A) 和 (B, B)

**有限重复博弈**：
- 在最后一期，可以选择任意一个纳什均衡
- 因此，可以在倒数第二期使用威胁，选择另一个均衡作为惩罚
- 这可以支持更好的结果

**例子**：
如果双方承诺一直选择 (A, A)，如果有人偏离就转向 (B, B)

### 2.4.4 不完美可观察行动

**问题**：
如果行动不能完美观察，或者观察有噪声，合作是否能维持？

**影响**：
- 错误的指责可能导致合作的崩溃
- 需要更复杂的策略来处理噪声

**策略**：
- 带噪声的触发策略
- 有限惩罚策略
- 基于统计检验的策略

## 2.5 无限重复博弈

### 2.5.1 无名氏定理（无限情形）

**定理（Folk Theorem）**：
设阶段博弈的可行且个人理性支付集合为 V，则对于任何 v ∈ V，存在 δ̄ < 1 使得当 δ > δ̄ 时，存在子博弈完美纳什均衡实现平均支付 v。

**可行支付集合**：
所有凸组合 Σ_{a∈A} α(a)u(a)，其中 Σ α(a) = 1, α(a) ≥ 0

**最小最大支付**：
参与者 i 的最小最大支付：
vᵢ^min = min_{σ₋ᵢ} max_{σᵢ} uᵢ(σᵢ, σ₋ᵢ)

**个人理性支付集合**：
V = {v: vᵢ ≥ vᵢ^min, 对所有 i}

### 2.5.2 无名氏定理的应用

**例子：无限重复囚徒困境**

**阶段博弈**：

| 参与者1 \ 参与者2 | 合作 | 背叛 |
|:---:|:---:|:---:|
| **合作** | (3, 3) | (0, 5) |
| **背叛** | (5, 0) | (1, 1) |

**最小最大支付**：
- 参与者1：min_{s₂} max_{s₁} u₁(s₁, s₂)
  - 如果参与者2合作：max{3, 5} = 5
  - 如果参与者2背叛：max{0, 1} = 1
  - 最小最大：min{5, 1} = 1
- 参与者2：同理，最小最大 = 1

**可行支付集合**：
所有 (3α₁ + 0α₂ + 5α₃ + 1α₄, 3α₁ + 5α₂ + 0α₃ + 1α₄)，其中 Σ αᵢ = 1

**个人理性支付集合**：
V = {v: v₁ ≥ 1, v₂ ≥ 1}

**结论**：
任何 v₁ ≥ 1, v₂ ≥ 1 的支付都可以作为子博弈完美纳什均衡实现，只要 δ 足够大。

### 2.5.3 重复古诺模型

**阶段博弈**：古诺双寡头模型
- 市场需求：P = a - q₁ - q₂
- 成本：Cᵢ(qᵢ) = cqᵢ
- 纳什均衡：q₁ = q₂ = (a-c)/3，支付 (a-c)²/9
- 垄断（合谋）：Q = (a-c)/2，支付 (a-c)²/8
- 合谋下每家：qᵢ = (a-c)/4，支付 (a-c)²/8

**触发策略**：
- 开始时选择合谋产量
- 如果有人偏离，永远转向古诺均衡

**合谋的条件**：

合谋时的总支付：
(1-δ)((a-c)²/8) + δ((a-c)²/8) + δ²((a-c)²/8) + ...
= ((a-c)²/8)/(1-δ)

偏离时的总支付：
给定对手选择 (a-c)/4，最优偏离：
max_q (a - (a-c)/4 - q)q - cq
= max_q (3a/4 + c/4 - q)q - cq
= max_q (3a/4 - 3c/4 - q)q

一阶条件：3a/4 - 3c/4 - 2q = 0
q = 3(a-c)/8

偏离当期支付：(3a/4 - 3c/8 - 3c/8)(3(a-c)/8) = (3a/4 - 3c/4)(3(a-c)/8) = 9(a-c)²/32

此后永远得到古诺支付：(a-c)²/9

总支付：9(a-c)²/32 + δ((a-c)²/9)/(1-δ)

合谋的条件：
((a-c)²/8)/(1-δ) ≥ 9(a-c)²/32 + δ((a-c)²/9)/(1-δ)

两边除以 (a-c)²：
1/(8(1-δ)) ≥ 9/32 + δ/(9(1-δ))

整理：
1/8 ≥ 9(1-δ)/32 + δ/9
1/8 ≥ 9/32 - 9δ/32 + δ/9
4/32 ≥ 9/32 - 9δ/32 + δ/9
-5/32 ≥ -9δ/32 + δ/9
9δ/32 - δ/9 ≥ 5/32
(81δ - 32δ)/288 ≥ 5/32
49δ/288 ≥ 5/32
49δ ≥ 45
δ ≥ 45/49 ≈ 0.918

**结论**：
当 δ ≥ 45/49 时，合谋可以作为子博弈完美纳什均衡实现。

### 2.5.4 重复博弈中的均衡选择

**均衡多重性**：
无限重复博弈通常有多个均衡

**选择标准**：
1. 效率：选择帕累托最优的均衡
2. 稳健性：选择对误差不敏感的均衡
3. 公平：选择相对公平的均衡
4. 简单性：选择策略简单的均衡

### 2.5.5 无限重复博弈的其他应用

**应用1：工资合同**
- 雇主可以选择低工资或高工资
- 员工可以选择偷懒或努力
- 重复博弈可以支持高工资和努力

**应用2：国际贸易**
- 国家可以选择贸易保护或自由贸易
- 重复博弈可以支持自由贸易

**应用3：环境保护**
- 企业可以选择污染或清洁
- 重复博弈可以支持环保行为

## 本章总结

### 核心要点

1. **扩展型博弈**通过博弈树描述动态博弈，包含参与者、决策节点、信息集和支付。

2. **子博弈完美纳什均衡**是纳什均衡的精炼，要求在每个子博弈中都构成纳什均衡。

3. **逆向归纳法**是求解有限完美信息博弈子博弈完美纳什均衡的标准方法。

4. **不可信威胁**是指在某些情况下，参与者在博弈中的威胁是不合理的，因为威胁者在实际执行威胁时不会这样做。

5. **重复博弈**是同一阶段博弈的多次重复，可以支持比一次性博弈更好的结果。

6. **无名氏定理**表明在重复博弈中，很多支付结果都可以作为均衡实现。

7. **触发策略**是维持合作的重要机制，通过威胁未来的惩罚来维持当前的合作。

### 重要概念表格

| 概念 | 定义 | 性质 |
|:---:|:---:|:---:|
| 扩展型博弈 | 通过博弈树描述的动态博弈 | 包含决策节点、信息集、支付 |
| 子博弈 | 从单一信息集开始的部分博弈 | 不分割信息集 |
| 子博弈完美纳什均衡 | 在每个子博弈中都构成纳什均衡 | 消除不可信威胁 |
| 逆向归纳法 | 从最后阶段向前求解 | 适用于有限完美信息博弈 |
| 重复博弈 | 阶段博弈的多次重复 | 可以支持合作 |
| 触发策略 | 合作直到有人背叛 | 维持合作的重要机制 |
| 无名氏定理 | 可行且个人理性的支付可实现 | δ 足够大时成立 |
| 贴现因子 | δ = 1/(1+r) | 衡量对未来重要程度 |
