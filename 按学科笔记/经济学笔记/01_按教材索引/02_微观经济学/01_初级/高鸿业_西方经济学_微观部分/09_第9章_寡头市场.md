## 9.1 寡头市场的特征

### 9.1.1 寡头市场的定义

**寡头市场（Oligopoly）**：
只有少数几个卖者的市场结构。

### 9.1.2 寡头市场的特征

**寡头市场的特征**：

1. **少数卖者（Few Sellers）**：
   - 市场上只有少数几个卖者
   - 每个卖者占市场的较大份额

2. **产品同质或有差异（Homogeneous or Differentiated Products）**：
   - 产品可能同质（例如：石油、钢铁）
   - 产品可能有差异（例如：汽车、手机）

3. **进入障碍（Barriers to Entry）**：
   - 其他厂商难以进入市场
   - 进入障碍是寡头形成的原因

4. **相互依存（Interdependence）**：
   - 每个厂商的决策影响其他厂商
   - 厂商之间相互依存

5. **策略行为（Strategic Behavior）**：
   - 厂商考虑其他厂商的反应
   - 厂商进行策略决策

### 9.1.3 寡头市场的类型

**寡头市场的类型**：

**1. 纯粹寡头（Pure Oligopoly）**：
- 产品同质
- 例如：石油、钢铁、水泥

**2. 差别寡头（Differentiated Oligopoly）**：
- 产品有差异
- 例如：汽车、手机、软饮料

### 9.1.4 寡头市场的形成原因

**寡头市场的形成原因**：

1. **规模经济（Economies of Scale）**：
   - 大规模生产降低平均成本
   - 小厂商无法竞争

2. **进入障碍（Barriers to Entry）**：
   - 高进入成本
   - 技术壁垒
   - 法律壁垒

3. **资源控制（Control of Resources）**：
   - 控制关键资源
   - 例如：石油、矿产

4. **政府政策（Government Policy）**：
   - 政府授予垄断权
   - 例如：特许经营权

## 9.2 寡头市场的模型

### 9.2.1 古诺模型

**古诺模型（Cournot Model）**：
一种寡头竞争模型，假设厂商同时选择产量。

**假设**：
1. 两个厂商（双寡头）
2. 产品同质
3. 厂商同时选择产量
4. 厂商知道市场需求
5. 厂商知道对方的产量

**模型分析**：

**市场需求函数**：
```
P = a - bQ = a - b(q1 + q2)
```

**厂商1的利润函数**：
```
π1 = P × q1 - TC1 = [a - b(q1 + q2)] × q1 - TC1
```

**厂商2的利润函数**：
```
π2 = P × q2 - TC2 = [a - b(q1 + q2)] × q2 - TC2
```

**厂商1的反应函数**：
```
q1 = (a - b × q2 - c1) / (2b)
```

**厂商2的反应函数**：
```
q2 = (a - b × q1 - c2) / (2b)
```

**古诺均衡**：
- 反应函数的交点
- 两个厂商都选择了最优产量

- 古诺均衡：R1和R2的交点

### 9.2.2 伯特兰模型

**伯特兰模型（Bertrand Model）**：
一种寡头竞争模型，假设厂商同时选择价格。

**假设**：
1. 两个厂商（双寡头）
2. 产品同质
3. 厂商同时选择价格
4. 厂商知道市场需求
5. 厂商知道对方的价格

**模型分析**：

**厂商的利润**：
- 如果P1 < P2，厂商1获得全部需求
- 如果P1 > P2，厂商2获得全部需求
- 如果P1 = P2，两个厂商平分需求

**伯特兰均衡**：
- P1 = P2 = MC
- 厂商获得零利润

**与古诺模型的区别**：
- 古诺模型：厂商选择产量
- 伯特兰模型：厂商选择价格

### 9.2.3 斯塔克尔伯格模型

**斯塔克尔伯格模型（Stackelberg Model）**：
一种寡头竞争模型，假设厂商依次选择产量。

**假设**：
1. 两个厂商（双寡头）
2. 产品同质
3. 厂商依次选择产量
4. 领导者先选择产量
5. 追随者根据领导者的产量选择产量

**模型分析**：

**厂商2（追随者）的反应函数**：
```
q2 = (a - b × q1 - c2) / (2b)
```

**厂商1（领导者）的利润函数**：
```
π1 = [a - b(q1 + q2)] × q1 - TC1
```

**斯塔克尔伯格均衡**：
- 领导者先选择产量
- 追随者根据领导者的产量选择产量
- 领导者获得比古诺均衡更高的利润

### 9.2.4 卡特尔模型

**卡特尔模型（Cartel Model）**：
一种寡头合作模型，厂商通过合作最大化联合利润。

**假设**：
1. 两个厂商（双寡头）
2. 产品同质
3. 厂商合作
4. 厂商像一个垄断厂商一样行动

**模型分析**：

**联合利润最大化**：
```
max π = [a - b(q1 + q2)] × (q1 + q2) - TC1 - TC2
```

**联合边际收益等于边际成本**：
```
MR = MC
```

**卡特尔均衡**：
- 厂商合作选择产量
- 厂商像垄断厂商一样行动
- 厂商获得正利润

**卡特尔的不稳定性**：
- 每个厂商有欺骗的动机
- 如果一个厂商增加产量，可以获得更多利润
- 如果所有厂商都增加产量，卡特尔崩溃

### 9.2.5 弯折的需求曲线模型

**弯折的需求曲线模型（Kinked Demand Curve Model）**：
一种寡头模型，解释价格刚性。

**假设**：
1. 如果一个厂商提高价格，其他厂商不跟随
2. 如果一个厂商降低价格，其他厂商跟随

**模型分析**：

**需求曲线**：
- 提价：需求弹性较大
- 降价：需求弹性较小
- 需求曲线在当前价格处弯折

**边际收益曲线**：
- 边际收益曲线在弯折处不连续
- 存在垂直间断

## 9.3 博弈论在寡头市场的应用

### 9.3.1 博弈论的基本概念

**博弈论（Game Theory）**：
研究策略互动的数学理论。

**基本概念**：

1. **参与者（Players）**：
   - 博弈的决策者
   - 例如：两个寡头厂商

2. **策略（Strategies）**：
   - 参与者可以选择的行动
   - 例如：提高价格、降低价格

3. **收益（Payoffs）**：
   - 参与者从博弈中获得的收益
   - 例如：利润

4. **纳什均衡（Nash Equilibrium）**：
   - 没有参与者可以通过改变策略而获得更高收益的策略组合
   - 每个参与者的策略都是对其他参与者策略的最优反应

### 9.3.2 囚徒困境

**囚徒困境（Prisoner's Dilemma）**：
一个经典的博弈论模型，说明合作困难。

**问题描述**：

两个囚徒被警察抓住，警察分开审问：

1. 如果两个囚徒都沉默，每个囚徒判1年
2. 如果一个囚徒坦白，一个囚徒沉默，坦白的囚徒释放，沉默的囚徒判5年
3. 如果两个囚徒都坦白，每个囚徒判3年

**收益矩阵**：

| 囚徒2 | 沉默 | 坦白 |
|-------|------|------|
| 沉默 | 1年, 1年 | 5年, 0年 |
| 坦白 | 0年, 5年 | 3年, 3年 |

**分析**：

**囚徒1的策略**：
- 如果囚徒2沉默，坦白更好（0年 < 1年）
- 如果囚徒2坦白，坦白更好（3年 < 5年）
- 因此，坦白是优势策略

**囚徒2的策略**：
- 如果囚徒1沉默，坦白更好（0年 < 1年）
- 如果囚徒1坦白，坦白更好（3年 < 5年）
- 因此，坦白是优势策略

**纳什均衡**：
- 两个囚徒都坦白
- 每个囚徒判3年

**帕累托最优**：
- 两个囚徒都沉默
- 每个囚徒判1年

**困境**：
- 纳什均衡不是帕累托最优
- 合作是困难的

### 9.3.3 囚徒困境在寡头市场的应用

**囚徒困境在寡头市场的应用**：

**描述**：

两个寡头厂商可以合作（保持高价）或不合作（降低价格）：

1. 如果两个厂商都保持高价，每个厂商获得高利润
2. 如果一个厂商降低价格，一个厂商保持高价，降低价格的厂商获得更高利润，保持高价的厂商获得更低利润
3. 如果两个厂商都降低价格，每个厂商获得低利润

**收益矩阵**：

| 厂商2 | 高价 | 低价 |
|-------|------|------|
| 高价 | 10, 10 | 2, 15 |
| 低价 | 15, 2 | 5, 5 |

**分析**：

**厂商1的策略**：
- 如果厂商2保持高价，降低价格更好（15 > 10）
- 如果厂商2降低价格，降低价格更好（5 > 2）
- 因此，降低价格是优势策略

**厂商2的策略**：
- 如果厂商1保持高价，降低价格更好（15 > 10）
- 如果厂商1降低价格，降低价格更好（5 > 2）
- 因此，降低价格是优势策略

**纳什均衡**：
- 两个厂商都降低价格
- 每个厂商获得5

**帕累托最优**：
- 两个厂商都保持高价
- 每个厂商获得10

**困境**：
- 纳什均衡不是帕累托最优
- 合作是困难的
- 卡特尔是不稳定的

## 9.4 寡头市场的效率

### 9.4.1 配置效率损失

**配置效率损失（Allocative Inefficiency）**：
寡头市场导致配置效率损失。

**原因**：
- 寡头价格高于完全竞争价格
- 寡头产量低于完全竞争产量
- 存在无谓损失

**图形表示**：

```plotly

  -
    type: scatter
    mode: lines
    name: 曲线 1
    x: [0, 20, 40, 60, 80, 100]
    y: [25, 20, 15, 10, 5, 0]
    line:
      color: "#1f77b4"
      width: 4
      shape: spline

  title:
    text: "需求曲线"
  xaxis:
    title:
      text: "数量Q"
  yaxis:
    title:
      text: "价格"
    range:
      - 0
      - 25
  template: "plotly_white"
  showlegend: true
config:
  displayModeBar: false
  responsive: true
```

**分析**：

**完全竞争均衡**：
- 价格：Pc
- 产量：Qc
- 总剩余最大

**寡头均衡**：
- 价格：Po > Pc
- 产量：Qo < Qc
- 总剩余减少

**无谓损失（Deadweight Loss, DWL）**：
- 寡头导致的无谓损失
- 由于产量不足造成

### 9.4.2 生产效率损失

**生产效率损失（Productive Inefficiency）**：
寡头厂商不以最低成本生产。

**原因**：
- P > min(ATC)
- 厂商不以最低成本生产
