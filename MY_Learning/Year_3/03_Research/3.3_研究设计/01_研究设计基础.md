# 3.3 研究设计

> **专业视角**：好的研究设计是成功的70%。研究设计决定了你能回答什么问题、不能回答什么问题——在开始之前就想清楚，比中途改设计要高效得多。本模块覆盖从研究问题到假设生成、从实验设计到伦理审查的完整流程。

## 4.1 从问题到假设

### 研究问题的类型

| 类型 | 描述 | 回答方式 | 举例 |
|------|------|---------|------|
| 描述性 | 描述现象 | 观察/调查 | 市场波动率的分布特征是什么？ |
| 相关性 | 探索关系 | 相关分析 | 散户比例与市场波动的关系？ |
| 因果性 | 验证因果 | 实验/准实验 | X策略是否导致Y收益？ |
| 预测性 | 预测未来 | 机器学习 | 明天股价走势？ |

### 好的研究问题标准（FINER）

- **F**easible：可执行（数据、方法是否可行）
- **I**nteresting：对领域有意思（新发现有价值）
- **N**ovel：新奇（不是简单重复）
- **E**thical：伦理（不伤害）
- **R**elevant：有意义（对理论和实践有贡献）

### 假设生成

```
研究问题 → 文献回顾 → 理论框架 → 假设演绎

示例路径：
问题：为什么散户在期权市场表现差？
    ↓
文献：期权定价理论 + 行为金融学 + 散户交易数据
    ↓
理论：散户缺乏波动率风险管理能力 + 处置效应导致错误行权
    ↓
假设H1：波动率预测能力强的散户，收益显著更高
    ↓
假设H2：实施止损规则的账户，回撤显著更小
```

## 4.2 实验设计范式

### 量化研究设计类型

| 类型 | 特点 | 内部效度 | 外部效度 | 适用场景 |
|------|------|---------|---------|---------|
| **随机对照实验（RCT）** | 随机分组、控制变量 | 高 | 中 | 因果推断 |
| **准实验** | 非随机分组 | 中 | 中 | 真实市场环境 |
| **观察性研究** | 无干预 | 低 | 高 | 探索性研究 |
| **回测研究** | 历史数据模拟 | 中 | 低 | 量化策略 |

### A/B测试设计

```python
import numpy as np
from scipy import stats

class ABTest:
    """A/B测试设计与分析"""

    def __init__(self, alpha=0.05, power=0.8, mde=0.01):
        """
        alpha: 第一类错误率
        power: 统计功效
        mde: 最小可检测效应（Minimum Detectable Effect）
        """
        self.alpha = alpha
        self.power = power
        self.mde = mde

    def sample_size(self, p_control):
        """
        计算每组所需样本量
        p_control: 对照组基准转化率
        """
        from scipy.stats import norm
        z_alpha = norm.ppf(1 - self.alpha / 2)
        z_beta = norm.ppf(self.power)
        p_treatment = p_control + self.mde

        n = (z_alpha + z_beta)**2 * (
            p_control * (1 - p_control) + p_treatment * (1 - p_treatment)
        ) / self.mde**2

        return int(np.ceil(n))

    def analyze(self, control_results, treatment_results):
        """
        分析A/B测试结果
        control_results: 对照组结果列表
        treatment_results: 实验组结果列表
        """
        n_c, n_t = len(control_results), len(treatment_results)
        mean_c = np.mean(control_results)
        mean_t = np.mean(treatment_results)

        # 独立样本t检验
        t_stat, p_value = stats.ttest_ind(treatment_results, control_results)

        # 效应量（Cohen's d）
        pooled_std = np.sqrt(
            ((n_c - 1) * np.var(control_results, ddof=1) +
             (n_t - 1) * np.var(treatment_results, ddof=1)) /
            (n_c + n_t - 2)
        )
        cohens_d = (mean_t - mean_c) / pooled_std

        return {
            "mean_control": mean_c,
            "mean_treatment": mean_t,
            "diff": mean_t - mean_c,
            "t_statistic": t_stat,
            "p_value": p_value,
            "cohens_d": cohens_d,
            "significant": p_value < self.alpha
        }
```

```python
class BacktestDesign:
    """避免过度拟合的回测设计"""

    def __init__(self, data, train_ratio=0.6, val_ratio=0.2, test_ratio=0.2):
        self.data = data
        self.train_ratio = train_ratio
        self.val_ratio = val_ratio
        self.test_ratio = test_ratio
        self.split_data()

    def split_data(self):
        """三层数据分割"""
        n = len(self.data)
        train_end = int(n * self.train_ratio)
        val_end = int(n * (self.train_ratio + self.val_ratio))

        self.train = self.data[:train_end]
        self.val = self.data[train_end:val_end]
        self.test = self.data[val_end:]

    def walk_forward_validation(self, strategy, n_splits=5):
        """
        步行_forward验证：模拟真实交易
        每轮用历史数据训练，用未来数据测试
        """
        results = []
        chunk_size = len(self.val) // n_splits

        for i in range(n_splits):
            train_chunk = self.train
            val_chunk = self.val[i*chunk_size:(i+1)*chunk_size]

            # 滚动更新训练集
            if i > 0:
                train_chunk = np.vstack([self.train, self.val[:i*chunk_size]])

            strategy.fit(train_chunk)
            predictions = strategy.predict(val_chunk)
            metrics = self.evaluate(predictions, val_chunk)
            results.append(metrics)

        return results

    def evaluate(self, predictions, actual):
        """评估指标"""
        return {
            "sharpe": self.sharpe_ratio(predictions, actual),
            "max_drawdown": self.max_drawdown(predictions),
            "win_rate": np.mean(predictions > 0)
        }
```

## 4.3 因果推断框架

### 潜在结果框架（Rubin Causal Model）

```
每个单位i有两种潜在结果：
- Y_i(1): 接受处理后的结果
- Y_i(0): 未接受处理的结果

因果效应 = Y_i(1) - Y_i(0)

问题：我们只能观察到其中一个
解决方案：随机分组（使处理组和对照组可比）
```

### 常用因果推断方法

```python
class CausalInference:
    """因果推断方法集合"""

    @staticmethod
    def difference_in_means(Y_treatment, Y_control):
        """差分均值法"""
        ate = np.mean(Y_treatment) - np.mean(Y_control)
        se = np.sqrt(
            np.var(Y_treatment) / len(Y_treatment) +
            np.var(Y_control) / len(Y_control)
        )
        return {"ate": ate, "se": se, "ci": (ate - 1.96*se, ate + 1.96*se)}

    @staticmethod
    def propensity_score_matching(df, treatment, outcome, covariates):
        """
        倾向得分匹配（PSM）
        treatment: 处理组指示变量
        outcome: 结果变量
        covariates: 协变量
        """
        from sklearn.linear_model import LogisticRegression

        # 估计倾向得分
        X = df[covariates]
        ps_model = LogisticRegression(max_iter=1000)
        ps_model.fit(X, df[treatment])
        propensity_scores = ps_model.predict_proba(X)[:, 1]

        # 匹配
        treated = df[df[treatment] == 1].copy()
        control = df[df[treatment] == 0].copy()
        treated["ps"] = propensity_scores[df[treatment] == 1]
        control["ps"] = propensity_scores[df[treatment] == 0]

        matched = []
        for _, row in treated.iterrows():
            # 最近邻匹配
            distances = np.abs(control["ps"] - row["ps"])
            nearest_idx = distances.idxmin()
            matched.append({
                "treated_outcome": row[outcome],
                "control_outcome": control.loc[nearest_idx, outcome]
            })

        matched_df = pd.DataFrame(matched)
        ate = (matched_df["treated_outcome"] - matched_df["control_outcome"]).mean()

        return {"ate": ate}

    @staticmethod
    def synthetic_control(df, treated_unit, control_units, outcome, time_index):
        """
        合成控制法
        用于政策评估（禁售令、交易限制等）
        """
        treated_data = df[df[time_index] == treated_unit][outcome]
        control_data = df[df[time_index].isin(control_units)][outcome]

        # 构建合成对照组（加权平均）
        weights = np.linalg.lstsq(control_data.values, treated_data.values, rcond=None)[0]
        synthetic = control_data.values @ weights

        effect = treated_data.values - synthetic
        return {"effect": effect, "weights": weights}
```

## 4.4 样本与统计功效

### 样本量计算

```python
from scipy.stats import norm

def calculate_sample_size(effect_size, alpha=0.05, power=0.8, method="two_sample"):
    """
    样本量计算
    effect_size: Cohen's d（标准化效应量）
    """
    z_alpha = norm.ppf(1 - alpha / 2)
    z_beta = norm.ppf(power)

    if method == "two_sample":
        n_per_group = 2 * (z_alpha + z_beta)**2 / effect_size**2
    elif method == "one_sample":
        n_per_group = (z_alpha + z_beta)**2 / effect_size**2

    return int(np.ceil(n_per_group))


# 效应量参考标准（Cohen's d）
# 小效应：0.2
# 中效应：0.5
# 大效应：0.8
```

### 时间序列样本量

```
对于日频交易策略回测：
```

## 4.6 研究设计常见错误

- **混淆相关性与因果性**：观察到两个变量相关就声称因果
- **选择性报告**：只报告支持假设的结果（p-hacking）
- **样本选择偏差**：样本不具代表性（如只用牛市数据）
- **多重比较问题**：做很多检验，只报告显著的
- **生态学谬误**：用aggregate数据推导个体结论
- **内部效度威胁**：历史事件、时间趋势、实验者效应
