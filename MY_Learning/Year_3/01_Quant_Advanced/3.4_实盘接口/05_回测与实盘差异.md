# 回测与实盘差异

## 核心概念

回测与实盘之间的差距（Backtest-Live Discrepancy）是量化交易中最核心、最棘手的问题之一。一个在历史数据上表现优异的策略，在实盘中可能表现平庸甚至亏损。这种差距源于回测环境的诸多假设与真实市场的本质差异。

理解这些差异对于正确评估策略、避免过度拟合、以及构建更稳健的交易系统至关重要。对于申请顶尖量化金融项目的学生，能够深入分析回测-实盘差距是学术能力的重要体现。

## 方法论

### 核心差异分类

| 差异类型 | 回测假设 | 实盘现实 |
|---------|---------|---------|
| 执行延迟 | 假设即时成交 | 订单传输、交易所处理存在延迟 |
| 流动性 | 假设可以任意量成交 | 大单可能冲击市场、滑点扩大 |
| 市场冲击 | 忽略或简化 | 订单簿被改变、价格偏移 |
| 数据质量 | 清洗后的干净数据 | 噪声、缺失、错误数据 |
| 心理因素 | 无情绪的机械执行 | 手动干预、恐慌抛售 |
| 费用 | 固定费率 | 滑点、价差、流动性成本 |
| 幸存者偏差 | 只用现存资产 | 回测中可能包含已退市资产 |
| 前视偏差 | 使用可用数据 | 信号产生与确认之间的时滞 |

### 幸存者偏差（Survivorship Bias）

回测中如果只使用当前仍存在的股票，会忽略历史上已经退市、破产的股票。这些"失败者"在历史上可能贡献了巨大的亏损。正确的回测需要包含**已退市股票的历史数据**（Point-in-Time Data）。

### 前视偏差（Look-Ahead Bias）

回测中使用的信息在产生信号时实际不可用。例如：
- 使用T日收盘价生成信号，却假设在T日开盘时已交易
- 使用财务报表发布日期而非实际公告日期
- 在计算中使用未来数据（data snooping）

### 流动性约束

回测中假设大额订单可以按照回测价格全部成交，但实盘中：
- 市场冲击成本（Market Impact）：大单会推动价格不利移动
- 滑点（Slippage）：实际成交价偏离预期价格
- 部分成交：流动性不足导致订单被拆分或无法完成

### 交易成本

| 成本类型 | 回测考虑 | 实盘影响 |
|---------|---------|---------|
| 佣金 | 通常固定 | 按笔或成交比例 |
| 印花税 | 简化处理 | 按地区不同 |
| 买卖价差 | 简化 | 资产、时段的真实价差 |
| 市场冲击 | 忽略或简单估算 | 难以精确建模 |
| 融资利息 | 固定利率 | 实际融资成本 |

## 代码实现

以下代码展示如何构建更真实的回测引擎，考虑交易成本与滑点：

```python
import numpy as np
from dataclasses import dataclass
from typing import List, Tuple, Optional

@dataclass
class BacktestConfig:
    initial_capital: float = 1000000.0
    commission_rate: float = 0.0003      # 佣金万分之三
    stamp_tax_rate: float = 0.001        # 印花税千分之一（卖出时）
    slippage_pct: float = 0.0005         # 滑点0.05%
    market_impact_factor: float = 0.1    # 市场冲击系数
    
@dataclass
class Trade:
    timestamp: int
    symbol: str
    side: str  # 'BUY' or 'SELL'
    quantity: float
    price: float
    slippage: float
    commission: float
    market_impact: float
    total_cost: float

class RealisticBacktester:
    """考虑真实交易成本的回测引擎"""
    
    def __init__(self, config: BacktestConfig):
        self.config = config
        self.capital = config.initial_capital
        self.positions: dict = {}
        self.trades: List[Trade] = []
        self.equity_curve: List[float] = []
    
    def calculate_slippage(self, price: float, quantity: float) -> float:
        """计算滑点成本（与订单规模相关）"""
        # 简化模型：滑点与订单量占日均成交量的比例相关
        participation_rate = quantity / 10000  # 假设日均成交量
        slippage = price * self.config.slippage_pct * (1 + participation_rate)
        return slippage
    
    def calculate_market_impact(self, price: float, quantity: float) -> float:
        """计算市场冲击成本（简化平方根模型）"""
        # 市场冲击 ∝ 订单规模 / 日均成交量 的平方根
        participation_rate = quantity / 10000
        impact = price * self.config.market_impact_factor * np.sqrt(participation_rate)
        return impact
    
    def execute_trade(self, timestamp: int, symbol: str, side: str, 
                     quantity: float, base_price: float) -> Optional[Trade]:
        """执行交易并计算真实成本"""
        if quantity <= 0:
            return None
        
        # 计算滑点
        slippage = self.calculate_slippage(base_price, quantity)
        
        # 计算市场冲击
        market_impact = self.calculate_market_impact(base_price, quantity)
        
        # 实际成交价格
        if side == 'BUY':
            execution_price = base_price + slippage + market_impact
        else:
            execution_price = base_price - slippage - market_impact
        
        # 交易成本
        turnover = execution_price * quantity
        commission = turnover * self.config.commission_rate
        
        # 印花税（仅卖出）
        stamp_tax = 0.0
        if side == 'SELL':
            stamp_tax = turnover * self.config.stamp_tax_rate
        
        total_cost = commission + stamp_tax
        
        # 检查资金是否足够
        required_capital = turnover + total_cost if side == 'BUY' else total_cost
        if self.capital < required_capital:
            print(f"Insufficient capital for trade: {required_capital} > {self.capital}")
            return None
        
        # 更新资金和持仓
        self.capital -= turnover + total_cost if side == 'BUY' else -turnover + total_cost
        
        if symbol not in self.positions:
            self.positions[symbol] = 0.0
        
        if side == 'BUY':
            self.positions[symbol] += quantity
        else:
            self.positions[symbol] -= quantity
        
        trade = Trade(
            timestamp=timestamp,
            symbol=symbol,
            side=side,
            quantity=quantity,
            price=execution_price,
            slippage=slippage,
            commission=commission,
            market_impact=market_impact,
            total_cost=total_cost
        )
        self.trades.append(trade)
        
        return trade
    
    def calculate_performance(self) -> dict:
        """计算回测绩效指标"""
        total_return = (self.capital - self.config.initial_capital) / self.config.initial_capital
        
        # 计算交易成本占比
        total_commission = sum(t.commission for t in self.trades)
        total_slippage = sum(t.slippage * t.quantity for t in self.trades)
        total_market_impact = sum(t.market_impact * t.quantity for t in self.trades)
        total_trading_cost = total_commission + total_slippage + total_market_impact
        
        win_trades = [t for t in self.trades if t.side == 'SELL']
        total_trades = len(win_trades)
        
        return {
            'total_return': total_return,
            'final_capital': self.capital,
            'total_trades': total_trades,
            'total_commission': total_commission,
            'total_slippage': total_slippage,
            'total_market_impact': total_market_impact,
            'total_trading_cost': total_trading_cost,
            'cost_to_return_ratio': total_trading_cost / max(abs(self.capital - self.config.initial_capital), 1)
        }
    
    def generate_report(self) -> str:
        """生成回测报告"""
        perf = self.calculate_performance()
        return f"""
=== Realistic Backtest Report ===
Initial Capital: {self.config.initial_capital:,.2f}
Final Capital: {perf['final_capital']:,.2f}
Total Return: {perf['total_return']:.2%}

Trading Activity:
  Total Trades: {perf['total_trades']}

Trading Costs:
  Commission: {perf['total_commission']:,.2f}
  Slippage: {perf['total_slippage']:,.2f}
  Market Impact: {perf['total_market_impact']:,.2f}
  Total Cost: {perf['total_trading_cost']:,.2f}
  Cost/Return Ratio: {perf['cost_to_return_ratio']:.2f}

Note: Strategy must beat trading costs to be profitable.
"""
```

## 应用与局限

### 如何缩小差距

1. **使用点对点数据（Point-in-Time Data）**：避免幸存者偏差
2. **加入交易成本模型**：更真实地模拟执行损耗
3. **考虑市场冲击**：大单分批执行（TWAP/VWAP）
4. **样本外测试（Out-of-Sample）**：留出时间窗口验证策略泛化能力
5. **蒙特卡洛模拟**：模拟不同市场情景下的策略表现
6. **Paper Trading**：实盘前进行模拟交易验证

### 局限性

1. **模型永远简化**：即使最复杂的回测引擎也无法完全复制真实市场
2. **行为金融学因素**：恐慌、贪婪等心理因素难以量化
3. **市场适应性问题**：历史规律可能在未来失效
4. **极端事件**：历史数据可能不包含足够多的极端事件样本
