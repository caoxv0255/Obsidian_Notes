# 订单管理系统

## 核心概念

订单管理系统（Order Management System, OMS）是量化交易系统中负责**订单生命周期管理**的核心组件。从策略生成交易信号，到订单最终成交或撤销，OMS全程追踪、管理每一个订单的状态转换，并处理与券商API的交互。

OMS的核心职责包括：**订单路由**（将订单发送到哪个交易所/券商）、**订单类型转换**（将通用订单转换为具体券商支持的形式）、**订单状态追踪**（维护每个订单的实时状态）、**成交匹配**（处理部分成交、更新剩余数量）、**手续费计算**（实时计算交易成本）。

在现代量化交易架构中，OMS通常与**订单执行系统（Order Execution System, OES）**配合工作：OMS负责"做什么"，OES负责"怎么做"——具体以什么价格、什么方式成交。

## 方法论

### 订单生命周期

订单从创建到终结会经历多个状态，每个状态的转换都需要精确处理：

1. **NEW（新建）**：订单对象创建完成，等待提交
2. **SUBMITTED（已提交）**：订单已发送给券商，等待确认
3. **PARTIAL_FILLED（部分成交）**：订单部分成交，剩余部分继续等待
4. **FILLED（全部成交）**：订单完成
5. **CANCELLED（已取消）**：用户主动取消
6. **REJECTED（已拒绝）**：券商拒绝

### 订单类型

| 订单类型 | 说明 | 适用场景 |
|---------|------|---------|
| 市价单（Market） | 以当前最优价格立即成交 | 追求成交、不在乎价格 |
| 限价单（Limit） | 指定价格或更优价格成交 | 希望控制价格 |
| 止损单（Stop） | 价格达到触发价后转为市价单 | 止损保护 |
| 冰山订单（Iceberg） | 大单分批小单成交 | 大额订单 |
| TWAP/VWAP | 时间/成交量加权平均执行 | 大宗交易 |

## 代码实现

```python
import asyncio
import time
from dataclasses import dataclass, field
from enum import Enum
from typing import Dict, List, Optional, Callable
import logging

logger = logging.getLogger(__name__)

class OrderStatus(Enum):
    NEW = "NEW"
    SUBMITTED = "SUBMITTED"
    PARTIAL_FILLED = "PARTIAL_FILLED"
    FILLED = "FILLED"
    CANCELLED = "CANCELLED"
    REJECTED = "REJECTED"

class OrderSide(Enum):
    BUY = "BUY"
    SELL = "SELL"

class OrderType(Enum):
    MARKET = "MARKET"
    LIMIT = "LIMIT"
    STOP = "STOP"

@dataclass
class Order:
    order_id: str
    symbol: str
    side: OrderSide
    order_type: OrderType
    quantity: float
    filled_quantity: float = 0.0
    price: Optional[float] = None
    status: OrderStatus = OrderStatus.NEW
    created_at: float = field(default_factory=time.time)
    broker_order_id: Optional[str] = None

@dataclass
class Fill:
    fill_id: str
    order_id: str
    symbol: str
    quantity: float
    price: float
    commission: float = 0.0

class OrderManagementSystem:
    def __init__(self):
        self._orders: Dict[str, Order] = {}
        self._fills: List[Fill] = []
        self._fill_callbacks: List[Callable] = []
        self._lock = asyncio.Lock()
    
    def generate_order_id(self) -> str:
        return f"OMS_{int(time.time() * 1000000)}"
    
    async def create_order(self, symbol: str, side: OrderSide, 
                          order_type: OrderType, quantity: float,
                          price: Optional[float] = None) -> Order:
        order = Order(
            order_id=self.generate_order_id(),
            symbol=symbol,
            side=side,
            order_type=order_type,
            quantity=quantity,
            price=price
        )
        async with self._lock:
            self._orders[order.order_id] = order
        logger.info(f"Order created: {order.order_id}")
        return order
    
    async def submit_order(self, order: Order, broker) -> Order:
        async with self._lock:
            if order.status != OrderStatus.NEW:
                raise ValueError(f"Invalid order status: {order.status}")
            order.status = OrderStatus.SUBMITTED
        
        try:
            broker_order = await broker.send_order(order)
            async with self._lock:
                order.broker_order_id = broker_order.get('broker_order_id')
            return order
        except Exception as e:
            async with self._lock:
                order.status = OrderStatus.REJECTED
            raise
    
    async def process_fill(self, order_id: str, quantity: float, price: float):
        async with self._lock:
            if order_id not in self._orders:
                return
            order = self._orders[order_id]
            order.filled_quantity += quantity
            fill = Fill(
                fill_id=f"FILL_{int(time.time() * 1000000)}",
                order_id=order_id,
                symbol=order.symbol,
                quantity=quantity,
                price=price
            )
            self._fills.append(fill)
            
            if order.filled_quantity >= order.quantity:
                order.status = OrderStatus.FILLED
            else:
                order.status = OrderStatus.PARTIAL_FILLED
        
        for callback in self._fill_callbacks:
            await callback(fill)
    
    def register_fill_callback(self, callback: Callable):
        self._fill_callbacks.append(callback)
    
    async def get_active_orders(self) -> List[Order]:
        async with self._lock:
            return [o for o in self._orders.values() 
                    if o.status in [OrderStatus.NEW, OrderStatus.SUBMITTED, OrderStatus.PARTIAL_FILLED]]
    
    async def calculate_pnl(self) -> float:
        async with self._lock:
            pnl = 0.0
            for fill in self._fills:
                if fill.quantity > 0:
                    pnl -= fill.quantity * fill.price
                else:
                    pnl += abs(fill.quantity) * fill.price
            return pnl
```

## 应用与局限

### 应用场景

1. **日内交易**：快速订单执行与撤销需要完善的OMS支持
2. **多策略管理**：统一管理多个策略的订单，避免仓位冲突
3. **合规监控**：实时检查订单是否符合监管要求和内部风控规则

### 局限性

1. **状态同步延迟**：OMS与券商系统之间存在状态同步延迟，可能导致重复下单
2. **复杂订单处理**：篮子订单、条件触发等复杂订单逻辑实现成本高
3. **高并发瓶颈**：单个OMS实例可能成为性能瓶颈
