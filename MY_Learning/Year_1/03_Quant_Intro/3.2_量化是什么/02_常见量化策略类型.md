# 常见量化策略类型

> 本章将介绍量化交易中最常见的一些策略类型，包括趋势跟踪、均值回归、统计套利、高频交易、事件驱动以及机器学习量化策略。这些策略各有其核心理念和适用场景，理解它们是构建自己交易系统的基础。

---

## 1. 趋势跟踪策略

趋势跟踪（Trend Following）是量化交易中最经典、最古老的策略流派之一。其核心理念是**"趋势一旦形成，往往会延续"**——市场在某个方向上积累的动能不会瞬间消失，而是会持续一段时间。趋势跟踪策略的目标是捕捉这种持续性的价格变动。

### 1.1 均线策略

均线策略是最简单的趋势跟踪策略之一。它利用短期价格均线与长期价格均线的交叉来判断趋势方向。

**核心思想：** 当短期均线上穿长期均线（金叉）时，认为市场进入上升趋势，做多；当短期均线下穿长期均线（死叉）时，认为市场进入下降趋势，做空或平仓。

```python
import numpy as np
import pandas as pd

def moving_average_crossover(data, short_window=20, long_window=60):
    """
    均线交叉策略
    data: 包含 'close' 列的 pandas DataFrame
    short_window: 短期均线窗口
    long_window: 长期均线窗口
    """
    df = data.copy()
    df['SMA_short'] = df['close'].rolling(window=short_window).mean()
    df['SMA_long'] = df['close'].rolling(window=long_window).mean()

    # 生成交易信号
    df['signal'] = 0
    df.loc[df['SMA_short'] > df['SMA_long'], 'signal'] = 1   # 多头信号
    df.loc[df['SMA_short'] < df['SMA_long'], 'signal'] = -1  # 空头信号

    # 计算持仓变化（从空仓到持仓的瞬间）
    df['position'] = df['signal'].diff()

    return df

# 示例使用
# price_data = pd.read_csv('historical_prices.csv')
# result = moving_average_crossover(price_data)
# print(result[result['position'] != 0])  # 打印所有交易信号点
```

**均线周期选择的经验规律：**
- 短期均线：10日、20日——对噪音敏感，信号频繁但假信号多
- 长期均线：60日、120日、200日——信号稳定但滞后严重
- 实际使用中需要根据品种特性做参数优化

### 1.2 海龟交易策略

海龟策略（Turtle Trading）源于1980年代传奇交易员理查德·丹尼斯（Richard Dennis）的实验，其核心理念是**"用严格的规则捕捉趋势，用仓位管理控制风险"**。海龟策略包含完整的入场、出场、止损和仓位管理体系。

**核心思想：** 利用唐奇安通道（Donchian Channel）判断突破——当价格突破过去N日最高价时做多，跌破过去N日最低价时做空。

```python
def turtle_trading_strategy(prices, entry_period=20, exit_period=10, atr_period=20):
    """
    海龟交易策略简化版
    prices: 收盘价序列
    entry_period: 入场通道周期（默认20日）
    exit_period: 出场通道周期（默认10日）
    atr_period: ATR计算周期
    """
    df = pd.DataFrame({'close': prices})

    # 1. 入场信号：突破20日最高价
    df['entry_high'] = df['close'].rolling(window=entry_period).max()
    df['entry_signal'] = df['close'] > df['entry_high'].shift(1)

    # 2. 出场信号：跌破10日最低价
    df['exit_low'] = df['close'].rolling(window=exit_period).min()
    df['exit_signal'] = df['close'] < df['exit_low'].shift(1)

    # 3. ATR（真实波幅）用于仓位管理
    high_low = df['close'].diff()
    high_close = (df['close'] - df['close'].shift(1)).abs()
    low_close = (df['close'] - df['close'].shift(1)).abs()
    df['tr'] = pd.concat([high_low, high_close, low_close], axis=1).max(axis=1)
    df['atr'] = df['tr'].rolling(window=atr_period).mean()

    # 4. 仓位单位计算（每个标的的仓位单位）
    # 每次亏损不超过账户的2%，单笔交易风险1%
    # unit = (account * 0.01) / (atr * multiplier)
    # 这里简化处理
    df['position_size'] = 100000 / (df['atr'] * 20)  # 简化：固定金额/ATR

    df['signal'] = 0
    df.loc[df['entry_signal'], 'signal'] = 1   # 开多
    df.loc[df['exit_signal'], 'signal'] = 0    # 平仓

    return df
```

海龟策略还包含加仓规则（每盈利0.5倍ATR加仓一次，最多加4次）和止损规则（每笔交易最大亏损为2%的账户净值）。

### 1.3 趋势跟踪策略的优缺点

| 优点 | 缺点 |
|------|------|
| 逻辑简单，易于理解和实现 | 震荡行情中持续亏损（"被反复打脸"） |
| 收益来源于少数大趋势，赔率可能很高 | 需要极强的执行力，扛得住连续亏损 |
| 适用于多种资产类别（期货、外汇、股票） | 参数敏感，不同品种需要不同参数 |
| 有完整的仓位管理体系 | 低胜率（可能只有30%~40%） |

**适用市场：** 趋势明显的商品期货、外汇市场；不适合高频数据和震荡市。

## 2. 均值回归策略

均值回归（Mean Reversion）与趋势跟踪恰恰相反，其核心理念是**"涨多了会跌，跌多了会涨"**——价格偏离均值后，最终会向均值回归。均值回归策略试图在价格偏离时反向操作，赌价格回归。

### 2.1 配对交易（Pairs Trading）

配对交易是均值回归策略的经典代表。它利用两个高度相关的资产之间的价差，当价差偏离历史均值时，做空涨得多的资产、做多涨得少的资产，等待价差回归。

**核心思想：** 选择两只走势高度相关的股票（如可口可乐和百事可乐、中国石油和中国石化），当它们的价格比率（或价差）偏离历史均值时入场，预期价差回归。

```python
def pairs_trading_strategy(price_a, price_b, lookback=60, entry_threshold=2.0, exit_threshold=0.5):
    """
    配对交易策略
    price_a, price_b: 两只股票的价格序列
    lookback: 计算价差均值和标准差的回看窗口
    entry_threshold: 入场阈值（标准差倍数）
    exit_threshold: 出场阈值
    """
    df = pd.DataFrame({'price_a': price_a, 'price_b': price_b})

    # 计算价格比率（或使用协整方法计算价差）
    df['ratio'] = df['price_a'] / df['price_b']

    # 计算历史的均值和标准差
    df['ratio_mean'] = df['ratio'].rolling(window=lookback).mean()
    df['ratio_std'] = df['ratio'].rolling(window=lookback).std()

    # Z-Score：当前比率偏离均值的标准差倍数
    df['z_score'] = (df['ratio'] - df['ratio_mean']) / df['ratio_std']

    # 生成信号
    # Z-Score > entry_threshold：价差过大，预期回归，做空 price_a，做多 price_b
    # Z-Score < -entry_threshold：价差过小（负偏），做多 price_a，做空 price_b
    df['signal'] = 0
    df.loc[df['z_score'] > entry_threshold, 'signal'] = -1  # 价差高估：空A多B
    df.loc[df['z_score'] < -entry_threshold, 'signal'] = 1  # 价差低估：多A空B
    df.loc[abs(df['z_score']) < exit_threshold, 'signal'] = 0  # 回归了，平仓

    return df
```

**关键点：** 选择配对资产时，通常使用协整检验（Cointegration Test）来验证两只股票之间是否存在稳定的长期均衡关系。

### 2.2 Bollinger Bands（布林带策略）

布林带由约翰·布林格（John Bollinger）发明，由中轨（均线）和上下两条轨道线组成。上下轨道分别是中轨加减K倍标准差（通常K=2）。

**核心思想：** 当价格触及下轨时，认为价格被低估，可能反弹至中轨或上轨；当价格触及上轨时，认为价格被高估，可能回落。

```python
def bollinger_bands_strategy(prices, window=20, num_std=2):
    """
    布林带策略
    prices: 价格序列
    window: 均线窗口
    num_std: 标准差倍数
    """
    df = pd.DataFrame({'close': prices})

    # 计算布林带
    df['middle_band'] = df['close'].rolling(window=window).mean()
    df['std'] = df['close'].rolling(window=window).std()
    df['upper_band'] = df['middle_band'] + num_std * df['std']
    df['lower_band'] = df['middle_band'] - num_std * df['std']

    # 简单的布林带交易策略：
    # 价格触及下轨时买入，触及上轨时卖出
    # 配合均线方向过滤效果更好
    df['signal'] = 0
    df.loc[df['close'] < df['lower_band'], 'signal'] = 1      # 超卖信号
    df.loc[df['close'] > df['upper_band'], 'signal'] = -1     # 超买信号
    df.loc[df['close'] > df['middle_band'], 'signal'] = 0     # 回归均线，平仓

    # 计算当前价格位置（%B指标）
    df['percent_b'] = (df['close'] - df['lower_band']) / (df['upper_band'] - df['lower_band'])

    return df
```

布林带的**%B指标**（Percent B）含义：
- %B = 1：价格在上轨
- %B = 0：价格在下轨
- %B > 1：价格突破上轨（超买）
- %B < 0：价格跌破下轨（超卖）

### 2.3 均值回归策略的优缺点

| 优点 | 缺点 |
|------|------|
| 胜率相对较高（通常超过50%） | 单次盈利有限，依赖大数定律 |
| 适合震荡市场 | 需要精准的均值和标准差估计 |
| 逻辑清晰，有统计学支撑 | 极端行情（"肥尾"）可能导致巨大亏损 |
| 配对交易对冲了市场系统性风险 | 配对选择和参数优化需要大量回测 |

**适用市场：** 震荡市、两只高度相关资产的价差交易、需要严格止损配合。

## 3. 统计套利策略

统计套利（Statistical Arbitrage）是一类基于统计学原理，利用资产间的定价偏差进行交易以获取稳定收益的策略。它不赌方向，而是赌"价格偏差终将修正"。

### 3.1 因子中性策略（Factor Neutral）

因子中性策略要求投资组合对市场因子（如Beta）、规模因子、价值因子等保持中性暴露，从而剥离系统性风险，获取纯粹的因子溢价。

**核心思想：** 构建一个多头和空头相互对冲的组合，使得组合收益与市场涨跌无关（Beta=0），只取决于选股能力或因子暴露。

```python
import numpy as np
from sklearn.linear_model import LinearRegression

def factor_neutral_portfolio(stocks_returns, market_returns):
    """
    因子中性组合构建（简化版）
    stocks_returns: 各股票的收益率序列（N x T 矩阵）
    market_returns: 市场收益率序列
    """
    n = stocks_returns.shape[0]
    residual_returns = []

    # Step 1: 对每只股票进行回归，计算残差收益（剥离因子后的收益）
    for i in range(n):
        model = LinearRegression()
        model.fit(market_returns.values.reshape(-1, 1), stocks_returns[i])
        residual = stocks_returns[i] - model.predict(market_returns.values.reshape(-1, 1))
        residual_returns.append(residual)

    residual_returns = np.array(residual_returns)  # (N, T)

    # Step 2: 选择残差收益最高的几只股票做多，最低的几只做空
    mean_residual = residual_returns.mean(axis=1)
    long_stocks = np.argsort(mean_residual)[-5:]   # 做多TOP5
    short_stocks = np.argsort(mean_residual)[:5]   # 做空BOTTOM5

    # Step 3: 等权重建仓，组合即为因子中性
    weights = np.zeros(n)
    weights[long_stocks] = 1.0 / len(long_stocks)
    weights[short_stocks] = -1.0 / len(short_stocks)  # 空头为负权重

    portfolio_return = np.dot(weights, residual_returns)  # 因子中性收益

    return weights, portfolio_return
```

**常见因子：** Fama-French五因子模型中的Market（SMB）、Size、Value、Profitability、Investment等。

### 3.2 协整（Cointegration）方法

协整是统计套利的核心理论基础。两个非平稳序列如果存在协整关系，即使它们各自的价格随机游走，它们的线性组合却是一个平稳序列。

**核心思想：** 如果两只股票价格存在协整关系，那么它们的价差（或者线性组合）是均值回复的。找到协整对后，当价差偏离均值时做回归交易。

```python
import pandas as pd
import numpy as np
from statsmodels.tsa.stattools import coint, adfuller
import statsmodels.api as sm

def cointegration_pairs_trading(price_a, price_b, lookback=60):
    """
    基于协整的配对交易
    """
    df = pd.DataFrame({'a': price_a, 'b': price_b})

    # Step 1: 用OLS拟合价格A对价格B的线性关系
    X = sm.add_constant(df['b'])
    model = sm.OLS(df['a'], X).fit()
    hedge_ratio = model.params.iloc[1]  # 对冲比例

    # Step 2: 计算价差（spread）
    df['spread'] = df['a'] - hedge_ratio * df['b']

    # Step 3: 计算价差的均值和标准差（滚动窗口）
    df['spread_mean'] = df['spread'].rolling(lookback).mean()
    df['spread_std'] = df['spread'].rolling(lookback).std()

    # Step 4: 当价差偏离超过2个标准差时入场
    df['z_score'] = (df['spread'] - df['spread_mean']) / df['spread_std']

    df['signal'] = 0
    df.loc[df['z_score'] > 2.0, 'signal'] = -1   # 价差过高：空A多B
    df.loc[df['z_score'] < -2.0, 'signal'] = 1   # 价差过低：多A空B
    df.loc[abs(df['z_score']) < 0.5, 'signal'] = 0  # 回归后平仓

    # Step 5: 协整检验（用ADF检验价差的平稳性）
    adf_result = adfuller(df['spread'].dropna())
    print(f"ADF Statistic: {adf_result[0]:.4f}, p-value: {adf_result[1]:.4f}")

    return df, hedge_ratio

# 协整检验：验证两只股票是否存在长期均衡关系
def find_cointegrated_pairs(stock_prices, threshold=0.05):
    """
    寻找协整对
    stock_prices: DataFrame of multiple stock prices
    Returns: 协整对列表及p值
    """
    n = stock_prices.shape[1]
    pairs = []

    for i in range(n):
        for j in range(i+1, n):
            result = coint(stock_prices.iloc[:, i], stock_prices.iloc[:, j])
            pvalue = result[1]
            if pvalue < threshold:  # p值小于阈值认为存在协整关系
                pairs.append((stock_prices.columns[i], stock_prices.columns[j], pvalue))

    return sorted(pairs, key=lambda x: x[2])
```

### 3.3 统计套利策略的优缺点

| 优点 | 缺点 |
|------|------|
| 组合对冲市场风险，收益相对稳定 | 需要大量资金分散风险 |
| 有严格的统计学理论支撑 | 极端市场条件下，价差可能不会回归 |
| 策略容量可以做得很大 | 执行成本（手续费、价差）对收益影响显著 |
| 可以多市场、多品种同时运行 | 策略同质化严重，竞争激烈 |

## 4. 高频交易基本概念

高频交易（High-Frequency Trading, HFT）是指利用计算机算法在极短时间内（通常毫秒甚至微秒级别）完成大量交易的策略类型。高频交易的核心竞争力是**速度**和**技术基础设施**。

### 4.1 什么是高频交易

高频交易有以下几个显著特征：
- **持仓时间极短**：从几毫秒到几分钟不等
- **订单交易量巨大**：每天可能产生数万甚至数百万笔订单
- **对延迟（Latency）极度敏感**：微秒级别的速度差异就能决定策略是否盈利
- **依靠市场微结构（Market Microstructure）**：利用订单簿、限价单队列等微观信息

### 4.2 常见的高频交易策略

#### 4.2.1 做市商策略（Market Making）

做市商通过在买卖盘口同时挂出限价单，为市场提供流动性并赚取买卖价差（Bid-Ask Spread）。高频做市商的核心是利用速度优势，始终保持订单簿的最佳买卖价位。

**核心思想：**
