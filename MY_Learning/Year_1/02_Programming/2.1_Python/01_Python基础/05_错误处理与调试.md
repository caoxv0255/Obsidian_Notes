# 错误处理与调试

## 核心概念

程序运行过程中难免出现错误，Python 提供了完善的异常处理机制来处理这些情况，而不是让程序直接崩溃。调试则是找出并修复代码中错误的过程。

**Python 异常处理机制：**
- `try`：尝试执行可能抛出异常的代码
- `except`：捕获并处理异常
- `else`：try 块中没有发生异常时执行
- `finally`：无论是否发生异常都执行

**Python 内置异常层次：**
- `BaseException` → `Exception` → 具体异常类
- 常见异常：`ValueError`、`TypeError`、`KeyError`、`IndexError`、`FileNotFoundError`

**调试方法：**
- 打印调试（print 语句）
- 断言（assert）
- IDE 调试器
- logging 模块
- pdb 交互式调试器

## 代码实现

### 1. 异常处理基础

```python
# 基本 try-except 结构
try:
    result = 10 / 0
except ZeroDivisionError:
    print("不能除以零！")

# 捕获多个异常
try:
    int("abc")
    lst = [1, 2, 3]
    print(lst[10])
except ValueError:
    print("值错误")
except IndexError:
    print("索引超出范围")

# 捕获异常对象
try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(f"错误类型: {type(e).__name__}")
    print(f"错误信息: {e}")
```

### 2. 完整的异常处理结构

```python
try:
    num = int(input("输入一个数字: "))
    result = 10 / num
except ValueError:
    print("请输入有效的数字")
except ZeroDivisionError:
    print("不能除以零")
else:
    print(f"结果是: {result}")
finally:
    print("程序结束")

# 异常处理最佳实践：具体到宽泛
try:
    with open("data.csv", "r") as f:
        content = f.read()
except FileNotFoundError:
    print("文件不存在")
except PermissionError:
    print("没有读取权限")
except Exception as e:
    print(f"发生错误: {e}")
```

### 3. 抛出异常

```python
# raise 语句抛出异常
def divide(a, b):
    if b == 0:
        raise ValueError("除数不能为零")
    return a / b

try:
    result = divide(10, 0)
except ValueError as e:
    print(e)

# 自定义异常类
class ValidationError(Exception):
    def __init__(self, field, message):
        self.field = field
        self.message = message
        super().__init__(f"{field}: {message}")

def validate_age(age):
    if age < 0:
        raise ValidationError("age", "年龄不能为负数")
    return True

try:
    validate_age(-5)
except ValidationError as e:
    print(f"验证失败: {e.field} - {e.message}")
```

### 6. 数据科学中的错误处理

```python
import pandas as pd
import numpy as np

def safe_divide(a, b, default=np.nan):
    try:
        return a / b
    except (ZeroDivisionError, FloatingPointError):
        return default

def load_csv_safe(filepath):
    try:
        return pd.read_csv(filepath)
    except FileNotFoundError:
        print(f"文件不存在: {filepath}")
        return None
    except pd.errors.EmptyDataError:
        print(f"文件为空: {filepath}")
        return None
    except pd.errors.ParserError:
        print(f"CSV 格式错误: {filepath}")
        return None
```

**1. 上下文管理器简化资源管理**
```python
from contextlib import contextmanager

@contextmanager
def timer():
    import time
    start = time.time()
    try:
        yield
    finally:
        print(f"耗时: {time.time() - start:.2f}秒")

with timer():
    sum(range(1000000))
```

## 常见错误

**1. bare except 捕获所有异常**
```python
# 错误：会捕获 KeyboardInterrupt 和 SystemExit
try:
    dangerous_code()
except:
    pass

# 正确：明确指定异常类型
try:
    dangerous_code()
except Exception as e:
    logger.error(f"发生错误: {e}")
```

**2. 异常处理后丢失原始错误信息**
```python
# 错误
try:
    risky_operation()
except Exception:
    print("出错了")  # 丢失错误详情

# 正确
try:
    risky_operation()
except Exception as e:
    raise RuntimeError(f"操作失败: {e}") from e  # 保留原始错误
```

**3. 在循环中不当使用异常处理**
```python
# 错误：可能每次迭代都捕获异常
for item in items:
    try:
        process(item)
    except:
        pass

# 正确：分离正常流程和异常处理
errors = []
for item in items:
    try:
        process(item)
    except Exception as e:
        errors.append((item, e))
```

**4. 忽略调试信息的重要性**
```python
# 错误：使用 print 代替 logging
print("调试信息")  # 生产环境应该用 logging

# 正确：分层日志
logger.debug("调试信息")
logger.info("一般信息")
logger.warning("警告信息")
logger.error("错误信息")
```
