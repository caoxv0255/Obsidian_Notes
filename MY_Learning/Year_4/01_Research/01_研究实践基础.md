## 一、研究到底是什么？

### 1.1 研究 vs 学习：本质区别

| 维度 | 学习 (Learning) | 研究 (Research) |
|------|----------------|-----------------|
| **目标** | 掌握已知知识 | 探索未知、创造新知识 |
| **路径** | 线性：教材→习题→考试 | 非线性：问题→假设→验证→迭代 |
| **答案** | 已有标准答案 | 没有预设答案，需要自己发现 |
| **反馈** | 即时（对错立判） | 延迟（可能数月后才知对错） |
| **心态** | 吸收、内化 | 质疑、突破、失败是常态 |

### 1.2 研究的三个层次

第一层：验证性研究（Replicating）
  - 重复别人做过的实验，理解方法
  - 这是本科阶段最常见的形式

第二层：扩展性研究（Extending）
  - 在已有工作基础上做改进、创新
  - 暑期科研的常见形态

第三层：开创性研究（Original）
  - 提出全新的问题或方法
  - 这是博士水平的追求

### 1.3 为什么大一就要开始做研究？

申请硬通货：顶级项目录取最看重的就是 research potential
- CMU CS PhD：平均每位录取者有 2-3 篇论文
- MIT Neuroscience：科研经历几乎是必需项
- 量化项目：研究经历能证明数学建模能力

时间线压力：
- 大一：试错期 — 找方向、进组、打基础
- 大二：积累期 — 争取做核心贡献、出一篇论文
- 大三：冲刺期 — 争取顶会论文、拿强推
- 大四：收获期 — 申请材料全部就绪

## 二、研究流程全解析

### 2.1 六步研究法

选题 → 文献 → 假设 → 实验 → 论文 → 发表

第一步：选题 (Topic Selection)
选题三问：
1. 这个领域还有哪些未解决的问题？
2. 我的背景能做出什么独特贡献？
3. 这个问题的意义是什么（理论/应用）？

好选题的特征：
- ✅ 足够具体（不是AI而是Transformer在金融时序预测中的应用）
- ✅ 有可行性（6-12个月内能完成）
- ✅ 有影响力（能发表在目标会议/期刊）
- ✅ 有趣（你自己真的想知道答案）

选题来源：
- 导师给的题目（最省力，但要确认你真的感兴趣）
- 文献里的 future work（作者自己说这是值得探索的方向）
- 跨领域碰撞（把A领域的方法用到B领域的问题）

第二步：文献综述 (Literature Review)

文献查找工具：
- Google Scholar（最全）
- Semantic Scholar（AI总结功能强）
- arXiv（最新 preprint）
- Web of Science / Scopus（严谨期刊）
- Connected Papers（追踪论文关联）

文献阅读方法：

快速筛查（5分钟/篇）：
1. 标题 + 摘要 → 判断是否相关
2. 图表 → 快速理解核心贡献
3. 结论 → 确认结论是否有力

深度精读（1-2小时/篇）：
1. 通读全文，理解动机和方法
2. 记录关键贡献、局限、自己的思考
3. 复现关键结果（如果可能）

文献管理工具：
- Zotero（免费开源，浏览器插件强）
- Mendeley（PDF管理强）
- Notion（轻量，适合笔记整合）
- Obsidian（你已经在用的！可以链接笔记）

第三步：提出假设 (Hypothesis)

假设的特征：
- 具体可测
- 有理论依据
- 可能被证伪

好假设示例：
❌ Transformer在金融预测中表现好
✅ 在加入领域特定注意力机制后，Transformer在预测A股波动率时的MSE能降低至少5%

假设验证路径：
假设 → 推导预期结果 → 设计实验 → 收集数据 → 分析结果 → 验证/推翻假设 → 迭代

第四步：实验设计 (Experimentation)

实验设计原则：
1. 对照实验：有基线（baseline）对比
2. 变量控制：一次只改变一个因素
3. 可复现性：记录所有参数、环境、种子
4. 统计显著性：足够样本量，报告 p-value / CI

第五步：论文写作

详见《02_论文写作与发表》

第六步：发表 (Publication)

发表层次：
顶会（最强）> 一区期刊 > 二区期刊 > 会议 > 预印本

发表周期：
- 会议：3-6个月
- 期刊：6-12个月
- 所以大二下学期就要开始投稿！

## 三、导师选择指南

### 3.1 选择导师的六个维度

| 维度 | 权重 | 评估问题 |
|------|------|----------|
| 研究方向 | 30% | 这个方向是我真正感兴趣的吗？ |
| 资金情况 | 20% | 教授有足够经费支持我吗？ |
| 学生出口 | 15% | 毕业生的去向好吗？ |
| 指导风格 | 15% | 教授是放养型还是手把手型？ |
| 合作机会 | 10% | 组里有多学科合作机会吗？ |
| 地理位置 | 10% | 这个城市适合我吗？ |

### 3.2 导师类型对照

明星教授：
- 资金足、名气大、推荐信强
- 但可能很忙，你可能是众多学生之一

青椒（助理教授）：
- 更可能亲自指导、你可能做核心工作
- 但经费和经验可能有限

大牛组：
- 组里有副研究员/博士后带学生
- 但你的贡献可能被稀释

小而精课题组：
- 紧密合作、直接指导
- 但资源有限、发表竞争力可能弱

### 3.3 如何了解导师

必做的事：
1. 读导师近5年的论文（至少5篇）
2. 找组里现有学生聊（邮件请求informational meeting）
3. 查毕业生去向（组主页 / LinkedIn）
4. 看导师在学术会议上的评价

警惕信号：
- 🚩 学生流动率异常高
- 🚩 毕业生去向差或延毕严重
- 🚩 组里关系紧张的风评
- 🚩 多年没有发表记录
- 🚩 不允许学生发论文（某些行业应用型组）

### 3.4 发邮件联系导师模板

邮件主题：Prospective Research Assistant - [你的学校/专业]

正文模板：
尊敬的[教授姓名]教授：

您好！

我是[你的大学]数据科学专业的大三学生[你的名字]。我在阅读您关于[具体研究方向]的论文后，对此方向产生了浓厚兴趣，特别是您2023年在NeurIPS发表的关于[具体论文标题]的工作。

我目前的研究兴趣是[你的研究方向]，并已有[相关经历，如：使用PyTorch实现图神经网络的经验，参与过XX项目]的经历。我希望能在暑期/下学期加入您的课题组，参与[具体课题]的研究工作。

随信附上我的简历，供您参考。非常期待能有一次面谈的机会，向您请教研究方向的建议。

祝您工作顺利！

[你的名字]
[你的邮箱]
[你的简历附件]

## 四、研究工具栈

### 4.1 LaTeX 论文排版

为什么必须学 LaTeX：
- 所有顶会（NeurIPS, ICML, ACL, AAAI）只接收 LaTeX 投稿
- 公式、参考文献管理比 Word 专业太多
- 学术交流的标准语言

LaTeX 生态：
Overleaf（在线编辑）← 推荐新手从这里开始
- 模板丰富：NeurIPS/ICML/IEEE 官方模板都有
- 实时预览
- 合作编辑
- Git 同步

本地：VS Code + LaTeX Workshop
- 更灵活，适合大项目
- 需要自己配置编译链

Overleaf 入门步骤：
1. 注册 overleaf.com
2. 创建新项目 → 选择官方模板
3. 学习基本语法

### 4.2 Zotero 文献管理

Zotero 核心功能：
- 浏览器插件：一键保存论文
- PDF 自动下载和整理
- 文献引用（Word/LaTeX插件）
- 标签和笔记系统

Zotero + Obsidian 工作流：
1. Zotero 保存论文PDF
2. Obsidian 写阅读笔记（双向链接到其他笔记）
3. LaTeX 写作时一键插入引用

Zotero 插件推荐：
- Zotero DOI Manager
- ZotFile（PDF自动重命名）
- Better BibTeX（LaTeX引用key管理）

### 4.3 GitHub 科研代码管理

为什么 GitHub 对申请至关重要：
- 招生官会看你的GitHub！
- 证明你的工程能力
- 展示你的研究贡献

科研GitHub规范 - 标准项目结构：
my-research-project/
├── README.md          # 项目说明（最重要！）
├── requirements.txt   # 依赖包
├── setup.py          # 安装脚本
├── data/             # 数据（用.gitignore排除大文件）
├── models/           # 模型代码
├── experiments/      # 实验脚本
├── results/          # 结果图表
├── LICENSE           # 开源协议（MIT/Apache）
└── .gitignore

README.md 必含内容：
- 项目标题
- 研究背景（一句话描述）
- 快速开始（代码示例）
- 主要结果（表格对比）
- 引用格式

### 4.4 实验管理工具

Weights & Biases (wandb)：
- 实验超参数追踪
- 训练曲线可视化
- 团队协作
- 免费额度够个人用

MLflow：
- 更适合完整生命周期管理
- 企业级

DVC (Data Version Control)：
- 数据版本控制
- GitHub for data

## 五、暑期科研全攻略

### 5.1 暑期科研的三种形式

| 类型 | 特点 | 难度 | 适合人群 |
|------|------|------|----------|
| REU | 政府资助的正式项目，有工资 | ⭐⭐⭐⭐⭐ | 成绩好、有一定科研经历 |
| UROP | 大学自办的研究项目 | ⭐⭐⭐⭐ | 本校学生或合作院校 |
| 导师自费 | 对方教授同意接收你 | ⭐⭐⭐ | 有明确目标和联系 |
| 自费套磁 | 自己联系，没工资但有机会 | ⭐⭐⭐ | 想进特定组 |

### 5.2 REU 项目详解

什么是 REU：
- NSF (National Science Foundation) 资助的本科生暑期研究项目
- 全美100+所大学有REU站点
- 每月,000-7,000津贴 + 住宿

顶级REU推荐：

MIT Summer Research Program (MSRP)
- MIT各系都有，竞争极激烈
- 截止：12月中旬

Caltech SURF
- 著名，导师资源强
- 截止：2月初

Stanford Summer Research (SSRP)
- 同上

Carnegie Mellon Summer Undergraduate Research (SURF)
- CS/ML方向很强

REU 申请时间线：

大三上（9-11月）：
- 确定感兴趣的项目列表
- 准备个人陈述、研究兴趣

大三寒假（12-2月）：
- 开始提交申请（很多1月截止）
- 联系推荐人

大三下（3-4月）：
- 等待结果
- 如被拒，立即启动备选方案

大三暑假（6-8月）：
- 正式开始暑期科研

### 5.3 暑期科研目标清单

最低目标：
- [ ] 完成一个完整的研究小项目
- [ ] 写出完整的技术报告
- [ ] 获得导师的评价/推荐信意图

标准目标：
- [ ] 有可展示的研究成果（poster/demo）
- [ ] 撰写论文初稿（不一定发表）
- [ ] 建立与导师的良好关系

超额目标：
- [ ] 发表论文或顶会workshop paper
- [ ] 获得返回读博的承诺（但别太早！）
- [ ] 产出开源代码（GitHub项目）

### 5.4 暑期科研期间的行为准则

应该做的：
- 每周向导师汇报进展
- 主动问问题（但先自己思考过）
- 记录每天的工作（实验日志）
- 积极参加组会、做展示
- 和组里的PhD学生建立关系

不应该做的：
- 🚫 长时间不联系导师
- 🚫 只等待指令，不主动思考
- 🚫 在导师面前抱怨困难
- 🚫 独占功劳（如果是合作项目）
- 🚫 泄露导师的未发表工作

## 六、研究心态与习惯

### 6.1 科研人的日常

典型研究日程（以做机器学习研究为例）：

09:00  查看邮件，整理今日任务
09:30  跑实验（同时读论文）
12:00  午饭
13:30  继续跑实验，分析上午结果
15:00  读论文（保持对领域的了解）
17:00  和导师/同门讨论
18:00  整理一天的结果，记录问题
20:00  偶尔晚上继续写代码或写作

### 6.2 科研好习惯

每日记录：
- 今天做了什么？
- 结果如何？
- 明天要做什么？
- 遇到什么问题？

每周总结：
- 本周进展
- 下周计划
- 需要帮助的地方

论文阅读习惯：
- 每天读1-2篇论文（泛读）
- 每周精读1篇（做笔记，复现）
- 用Notion/Zotero整理文献

### 6.3 面对失败

科研失败的常态：

一个PhD平均要经历：
- 3-5个失败的项目/假设
- 1-2次被顶会/期刊拒稿
- 数不清的实验失败

这不是失败，这是科研！

正确心态：
- 失败是信息：告诉你什么方向不对
- 假设被证伪 = 排除错误答案 = 进步
- 记录失败：下次不再犯同样的错误

调整策略：
1. 遇到连续失败 → 降低期望，先做小实验
2. 长期没进展 → 和导师讨论，可能需要换方向
3. 被拒稿 → 认真读审稿意见，修改后投下一个会议

## 七、检查清单与时间线

### 7.1 大三学生研究进度检查

□ 大三上学期：
  □ 确定研究兴趣方向
  □ 加入一个研究组
  □ 开始阅读该方向的综述论文
  □ 学习必要的研究工具（LaTeX, GitHub, etc.）

□ 大三寒假：
  □ 开始第一个研究项目
  □ 准备暑期科研申请材料
  □ 继续读论文

□ 大三下学期：
  □ 推进研究，争取有初步结果
  □ 如有可能，写一篇论文初稿
  □ 申请暑期科研（如打算去海外）

□ 大三暑假（至关重要！）：
  □ 完成暑期科研项目
  □ 产出可展示的成果
  □ 确认推荐信

□ 大四上学期：
  □ 论文投稿
  □ 开始正式申请
  □ 准备面试
