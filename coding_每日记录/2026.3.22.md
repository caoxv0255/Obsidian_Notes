### 模块1：路径问题概述——网格中的DP

**问题本质**：在 m×n 网格中，从左上角到右下角，每次只能向下或向右移动，求不同路径的数量。

**状态设计**：`dp[i][j]` = 到达格子 (i,j) 的不同路径数

**状态转移**：
```cpp
dp[i][j] = dp[i-1][j] + dp[i][j-1];
```

**初始化**：
- `dp[0][j] = 1`（首行只能从左边来）
- `dp[i][0] = 1`（首列只能从上边来）

**62 vs 63 的区别**：
- 62：空网格 → 简单组合数或 DP
- 63：障碍物网格 → 遇到障碍跳过，dp[i][j] = 0（不可达）

### 模块2：打家劫舍（198）——环形房屋

**问题**：首尾相连的房屋，最多偷多少钱。不能偷相邻的两家。

**核心思想**：环形拆成两种情况：
1. 不偷第一间 → 在 [2, n] 范围内做普通打家劫舍
2. 不偷最后一间 → 在 [1, n-1] 范围内做普通打家劫舍

取两种情况的最大值。

**一维 DP 状态**：`dp[i]` = 考虑前 i 间房屋能偷的最大金额

**状态转移**：
```cpp
dp[i] = max(dp[i-1], dp[i-2] + nums[i-1]);
// 不偷第i间 vs 偷第i间
```

**初始化**：
```cpp
dp[0] = 0;        // 0间房屋
dp[1] = nums[0];  // 只有第1间
```

## 思考

1. **为什么 63 不同路径 II 中，初始化首行/首列时遇到障碍后，后面都是 0？**
   - 因为只能向右或向下移动，一旦某个格子被障碍挡住，从起点到它就不可能了
   - 首行只能从左向右，首列只能从上向下，所以遇到障碍后面就都无法到达

2. **198 打家劫舍为什么不能直接用一维 DP 从左到右遍历？**
   - 因为是环形！首尾相邻意味着第1间和第n间不能同时偷
   - 所以拆成两种情况：排除第1间（或排除最后一间），变成 [1..n-1] 的普通打家劫舍

   - 这次复习加深了对 O(n²) 暴力法的理解，以及二分优化的思路
