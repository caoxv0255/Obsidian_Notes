## 📚 模块1：LeetCode 695 岛屿的最大面积——BFS队列模板

### 问题描述

给定一个二进制矩阵，1 代表陆地，0 代表水域。找出其中最大的岛屿面积（即相连的 1 的数量）。如果没有岛屿，返回 0。

### 核心思想：广度优先搜索（BFS）

BFS使用队列实现层序遍历，从起点开始，先访问所有相邻节点，再访问相邻节点的相邻节点，以此类推。适合计算岛屿面积，因为可以逐层累加面积。

### 完整代码模板（C++）

```cpp
#include <vector>
#include <queue>
using namespace std;

class Solution {
public:
    int maxAreaOfIsland(vector<vector<int>>& grid) {
        if (grid.empty() || grid[0].empty()) {
            return 0;
        }
        
        int m = grid.size();      // 行数
        int n = grid[0].size();   // 列数
        int maxArea = 0;          // 最大面积
        
        // 遍历整个网格
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                // 发现未访问的陆地
                if (grid[i][j] == 1) {
                    maxArea = max(maxArea, bfs(grid, i, j, m, n));
                }
            }
        }
        
        return maxArea;
    }
    
private:
    // BFS队列实现
    int bfs(vector<vector<int>>& grid, int i, int j, int m, int n) {
        queue<pair<int, int>> q;
        q.push({i, j});
        grid[i][j] = 0;  // 标记为已访问
        int area = 0;     // 当前岛屿面积
        
        // 方向数组：下、上、右、左
        int dirs[4][2] = {{1, 0}, {-1, 0}, {0, 1}, {0, -1}};
        
        while (!q.empty()) {
            auto [x, y] = q.front();
            q.pop();
            area++;  // 每弹出一个陆地，面积加1
            
            // 遍历四个方向
            for (int d = 0; d < 4; d++) {
                int nx = x + dirs[d][0];
                int ny = y + dirs[d][1];
                
                // 检查边界和陆地条件
                if (nx >= 0 && nx < m && ny >= 0 && ny < n && grid[nx][ny] == 1) {
                    q.push({nx, ny});
                    grid[nx][ny] = 0;  // 入队时立即标记，避免重复入队
                }
            }
        }
        
        return area;
    }
};
```

### BFS队列的层序特性

BFS的"广度优先"体现在它按距离起点的层数逐层扩展。在岛屿面积问题中，这相当于从中心向外一圈圈地覆盖整个岛屿。

**关键认知**：BFS使用队列保证"先进先出"，确保先访问距离起点近的节点，后访问距离远的节点。

## 📚 模块2：LeetCode 200 岛屿数量——DFS递归模板

### 问题描述

给你一个由 '1'（陆地）和 '0'（水）组成的二维网格，请你计算网格中岛屿的数量。岛屿总是被水包围，并且每座岛屿只能由水平方向和/或竖直方向上相邻的陆地连接形成。

### 核心思想：深度优先搜索（DFS）

当我们遇到一个陆地（'1'）时，就从该位置开始进行DFS，将与之相连的所有陆地标记为已访问，这样整个岛屿就被"淹没"了。每启动一次DFS，就计数一个岛屿。

### 完整代码模板（C++）

```cpp
#include <vector>
using namespace std;

class Solution {
public:
    int numIslands(vector<vector<char>>& grid) {
        if (grid.empty() || grid[0].empty()) {
            return 0;
        }
        
        int m = grid.size();      // 行数
        int n = grid[0].size();   // 列数
        int count = 0;            // 岛屿计数器
        
        // 遍历整个网格
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                // 发现未访问的陆地
                if (grid[i][j] == '1') {
                    count++;                    // 新岛屿
                    dfs(grid, i, j, m, n);      // 淹没整个岛屿
                }
            }
        }
        
        return count;
    }
    
private:
    // DFS递归函数
    void dfs(vector<vector<char>>& grid, int i, int j, int m, int n) {
        // 边界条件：越界或不是陆地
        if (i < 0 || i >= m || j < 0 || j >= n || grid[i][j] != '1') {
            return;
        }
        
        // 标记当前陆地已访问（修改为'0'，避免重复访问）
        grid[i][j] = '0';
        
        // 四个方向递归搜索
        dfs(grid, i + 1, j, m, n);  // 下
        dfs(grid, i - 1, j, m, n);  // 上
        dfs(grid, i, j + 1, m, n);  // 右
        dfs(grid, i, j - 1, m, n);  // 左
    }
};
```

### DFS递归栈的直观理解

每次递归调用都会在调用栈中压入一层，递归返回时弹出。对于岛屿问题，递归深度等于岛屿的最大连通分量大小。

**关键认知**：DFS的"深度优先"体现在它会一直往一个方向深入，直到无法继续（遇到边界或水），然后回溯到上一个分叉点尝试其他方向。

## 📚 模块3：访问标记策略——避免无限递归的关键

### 两种常见标记方法

| 方法 | 实现 | 优点 | 缺点 |
|-----|-----|-----|-----|
| 修改原数组 | 将访问过的 '1' 改为 '0' | 空间复杂度O(1)，无需额外内存 | 破坏原数据，无法恢复 |
| 额外visited数组 | 创建vector<vector<bool>> visited记录访问状态 | 保留原数据，便于调试 | 空间复杂度O(m×n) |

### 常见错误示例

**错误代码（忘记标记导致无限递归）**：

```cpp
void dfs(vector<vector<char>>& grid, int i, int j, int m, int n) {
    if (i < 0 || i >= m || j < 0 || j >= n || grid[i][j] != '1') {
        return;
    }
    // 缺少标记语句：grid[i][j] = '0';
    
    dfs(grid, i + 1, j, m, n);
    dfs(grid, i - 1, j, m, n);
    dfs(grid, i, j + 1, m, n);
    dfs(grid, i, j - 1, m, n);
}
```

**后果**：由于没有标记已访问，DFS会在相邻陆地间无限递归，最终导致栈溢出。

### 工程选择建议

- **面试/竞赛**：优先使用修改原数组法，节省空间且代码简洁
- **生产环境**：如果需要保留原始数据，使用额外visited数组
- **调试阶段**：使用visited数组，便于跟踪访问状态

**零压力提示**：掌握修改原数组法即足够。visited数组法作为了解内容，知道其存在即可。

## 📚 模块4：回溯法思想铺垫——状态恢复的概念

### 从DFS到回溯的思维过渡

DFS在岛屿问题中直接修改状态（陆地→水），无需恢复，因为目标是遍历整个连通分量。但在某些问题中，我们需要在探索后恢复原始状态，以便尝试其他路径——这就是回溯法的核心。

### 简单示例：全排列问题

在LeetCode 46全排列中，我们需要：

1. 选择当前数字加入路径
2. 递归生成剩余数字的排列
3. 撤销选择（将数字从路径移除），以便尝试其他数字

```cpp
void backtrack(vector<int>& nums, vector<int>& path, vector<bool>& used) {
    if (path.size() == nums.size()) {
        // 找到一个排列
        result.push_back(path);
        return;
    }
    
    for (int i = 0; i < nums.size(); i++) {
        if (!used[i]) {
            used[i] = true;          // 标记使用
            path.push_back(nums[i]); // 选择
            
            backtrack(nums, path, used); // 递归
            
            path.pop_back();         // 撤销选择（状态恢复）
            used[i] = false;         // 撤销标记
        }
    }
}
```

### 状态恢复的必要性

| 场景 | 是否需要状态恢复 | 原因 |
|-----|----------------|-----|
| 岛屿计数 | 否 | 每个陆地只需要访问一次，访问后可以永久标记 |
| 全排列 | 是 | 同一个数字可以在不同位置出现，需要重复使用 |
| N皇后 | 是 | 放置皇后后需要尝试其他位置，必须恢复棋盘状态 |

**关键认知**：是否需要状态恢复取决于问题是否允许重复访问同一状态。岛屿问题不允许（每个陆地只属一个岛屿），排列问题允许（数字可重复使用）。

## 📚 模块5：LeetCode 79 单词搜索——回溯法标准框架

### 问题描述

给定一个 m × n 的字符网格 board 和一个字符串 word，判断 word 是否存在于网格中。单词必须按照字母顺序，通过相邻的单元格（水平或垂直）构成，同一个单元格内的字母不允许被重复使用。

### 核心思想：回溯法（选择-递归-撤销）

回溯法在DFS基础上增加了状态恢复机制，适用于需要尝试所有可能路径的问题。其标准流程为：

- **选择**：将当前单元格标记为已使用，记录匹配的字符位置
- **递归**：向四个方向探索下一个字符
- **撤销**：恢复单元格状态，以便其他路径可以使用

### 完整代码模板（C++）

```cpp
#include <vector>
#include <string>
using namespace std;

class Solution {
public:
    bool exist(vector<vector<char>>& board, string word) {
        if (board.empty() || board[0].empty()) return false;
        
        m = board.size();
        n = board[0].size();
        
        // 遍历所有起点
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (backtrack(board, word, i, j, 0)) {
                    return true;
                }
            }
        }
        return false;
    }
    
private:
    int m, n;
    
    bool backtrack(vector<vector<char>>& board, string& word, int i, int j, int idx) {
        // 终止条件：已匹配完整个单词
        if (idx == word.size()) {
            return true;
        }
        
        // 边界检查与字符匹配检查
        if (i < 0 || i >= m || j < 0 || j >= n || board[i][j] != word[idx]) {
            return false;
        }
        
        // 选择：标记当前单元格已使用（临时修改为特殊字符）
        char temp = board[i][j];
        board[i][j] = '#';  // 任意非字母字符，避免重复使用
        
        // 递归：四个方向探索下一个字符
        bool found = backtrack(board, word, i + 1, j, idx + 1) ||
                     backtrack(board, word, i - 1, j, idx + 1) ||
                     backtrack(board, word, i, j + 1, idx + 1) ||
                     backtrack(board, word, i, j - 1, idx + 1);
        
        // 撤销：恢复单元格状态
        board[i][j] = temp;
        
        return found;
    }
};
```

### 逐步调试指南（跟踪回溯路径）

**示例输入**：
```
board = [
  ['A','B','C','E'],
  ['S','F','C','S'],
  ['A','D','E','E']
]
word = "ABCCED"
```

**执行流程**：

1. 起点 (0,0)：字符 'A' 匹配 word[0]
2. 标记 board[0][0] = '#'
3. 递归尝试四个方向：(0,1) 匹配 'B'（word[1]）
4. 位置 (0,1)：字符 'B' 匹配 word[1]
5. 标记 board[0][1] = '#'
6. 递归尝试：(0,2) 匹配 'C'（word[2]）
7. 位置 (0,2)：字符 'C' 匹配 word[2]
8. 标记 board[0][2] = '#'
9. 递归尝试：(1,2) 匹配 'C'（word[3]）
10. 位置 (1,2)：字符 'C' 匹配 word[3]
11. 标记 board[1][2] = '#'
12. 递归尝试：(1,1) 不匹配（'F' ≠ 'E'），(2,2) 匹配 'E'（word[4]）
13. 位置 (2,2)：字符 'E' 匹配 word[4]
14. 标记 board[2][2] = '#'
15. 递归尝试：(2,1) 匹配 'D'（word[5]）
16. 位置 (2,1)：字符 'D' 匹配 word[5]
17. idx == word.size()，返回 true
18. 逐层返回，恢复各单元格状态

**关键认知**：回溯法通过临时修改状态（'#'）避免重复访问，并在递归返回后恢复原状态，确保同一单元格可在不同路径中被使用。

### 剪枝优化对比示例

**无剪枝的基础回溯**：
- 每个匹配点尝试四个方向
- 递归次数：最坏情况下，每个位置都尝试四个方向，直到单词长度 L
- 示例：3×4 网格，单词长度6，递归调用次数约 12 × 4^5 = 12,288 次

**增加简单剪枝**：
- 方向剪枝：只尝试有效方向（已实现）
- 提前终止：字符不匹配立即返回（已实现）
- 起点剪枝：若单词首字母出现频率低于尾字母，可从尾字母开始搜索（可选）
- 前缀检查：若使用字典树（Trie），可快速判断当前路径是否可能（高级优化）

**剪枝效果对比（概念性数据）**：

| 网格大小 | 单词长度 | 无剪枝递归次数 | 简单剪枝后递归次数 | 优化比例 |
|---------|---------|---------------|------------------|---------|
| 3×4     | 6       | ~12,288       | ~3,072           | 75%     |
| 5×5     | 8       | ~1,600,000    | ~200,000         | 87.5%   |

**工程建议**：掌握基础剪枝（方向+提前终止）即足够覆盖面试需求。高级剪枝（字典树）作为了解内容，知道其存在即可。

### 复杂度分析

**时间复杂度上限**：
设网格大小为 m × n，单词长度为 L。

- 最坏情况：每个单元格都可以作为起点，每个位置最多尝试四个方向，递归深度为 L
- 时间复杂度：O(m × n × 4^L)
  - m × n 个起点
  - 每个递归调用最多分支4个方向
  - 递归深度为 L（形成 4^L 的搜索树）

**剪枝后的实际复杂度**：
- 平均情况：由于字符不匹配提前终止，实际递归分支远小于4
- 经验估计：O(m × n × 3^L) 或更低（取决于单词与网格的匹配程度）

**空间复杂度**：
- 递归栈深度：O(L)（最坏递归深度）
- 修改原数组：O(1) 额外空间（通过修改 board 实现标记）

**关键认知**：回溯法的时间复杂度通常是指数级的（4^L），这是其本质特性。剪枝可以显著减少常数因子，但无法改变指数级增长趋势。因此，回溯法适用于 L 较小（通常 ≤ 10）的场景。
