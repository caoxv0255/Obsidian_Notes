### 第一部分：概念判断题（5题）

1. **快速排序的平均时间复杂度是O(n log n)，最坏时间复杂度是O(n²)，可通过随机化基准避免最坏情况。** ✅
   - 答案：正确。随机化基准避免最坏情况是快速排序的经典优化

2. **归并排序在链表上的实现可以达到O(1)额外空间复杂度（递归栈不计入），因为合并时只需调整指针。** ✅
   - 答案：正确。链表归并排序只需调整指针，额外空间O(1)

3. **DFS和BFS都可以用于求解无权图中的最短路径问题，且BFS保证首次到达的路径是最短的。** ✅
   - 答案：正确。BFS在无权图中首次到达的路径是最短的

4. **回溯法必须恢复状态（undo）是因为问题允许重复访问同一状态，如排列问题中同一个数字可在不同位置出现。** ✅
   - 答案：正确。状态恢复是回溯法的核心特征

5. **二分查找要求数组有序是因为需要保证每次比较后可以确定目标在左半或右半，否则无法缩小搜索范围。** ✅
   - 答案：正确。有序性是二分查找正确缩小搜索范围的前提

### 第二部分：代码填空题（5题）

#### 题目1：快速排序分区（Lomuto分区）

```cpp
int partition(vector<int>& nums, int left, int right) {
    int pivot = nums[right];  // 选择最右元素作为基准
    int i = left - 1;         // 指向小于基准区域的末尾
    
    for (int j = left; j < right; j++) {
        if (nums[j] < pivot) {
            i++;
            swap(nums[i], nums[j]);  // 将较小元素移到前面
        }
    }
    // 将基准放到正确位置
    swap(nums[i + 1], nums[right]);
    return i + 1;  // 返回基准的最终索引
}
```

**填空结果**：已完整，无需填空（示例代码）

#### 题目2：回溯法模板（LeetCode 46全排列的交换法实现）

```cpp
void backtrack(vector<int>& nums, int start, vector<vector<int>>& result) {
    if (start == nums.size()) {
        result.push_back(nums);
        return;
    }
    
    for (int i = start; i < nums.size(); i++) {
        swap(nums[start], nums[i]);        // 填空1：选择
        backtrack(nums, start + 1, result); // 填空2：递归
        swap(nums[start], nums[i]);        // 填空3：撤销选择
    }
}
```

**填空答案**：
- 填空1：`swap(nums[start], nums[i])`（做选择）
- 填空2：`backtrack(nums, start + 1, result)`（递归）
- 填空3：`swap(nums[start], nums[i])`（撤销选择）

#### 题目3：组合总和剪枝（LeetCode 39）

```cpp
void backtrack(vector<int>& candidates, int target, int start, 
               int sum, vector<int>& path, vector<vector<int>>& result) {
    if (sum == target) {
        result.push_back(path);
        return;
    }
    
    for (int i = start; i < candidates.size(); i++) {
        if (sum + candidates[i] > target) break;  // 填空4：剪枝条件
        
        path.push_back(candidates[i]);            // 填空5：选择
        backtrack(candidates, target, i, sum + candidates[i], path, result);
        path.pop_back();                          // 填空6：撤销选择
    }
}
```

**填空答案**：
- 填空4：`sum + candidates[i] > target`（剪枝条件）
- 填空5：`path.push_back(candidates[i])`（选择）
- 填空6：`path.pop_back()`（撤销选择）

void dfs(vector<vector<char>>& grid, int i, int j, int m, int n) {
    if (i < 0 || i >= m || j < 0 || j >= n || grid[i][j] != '1') {

#### 题目5：二分查找模板

```cpp
int binarySearch(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;  // 填空8：防溢出计算中点
        
        if (nums[mid] == target) {
            return mid;                       // 填空9：找到目标
        } else if (nums[mid] < target) {
            left = mid + 1;                   // 填空10：目标在右半
        } else {
            right = mid - 1;                  // 填空11：目标在左半
        }
    }
    
    return -1;  // 未找到
}
```

**填空答案**：
- 填空8：`left + (right - left) / 2`（防溢出计算中点）
- 填空9：`return mid`（找到目标）
- 填空10：`left = mid + 1`（目标在右半）
- 填空11：`right = mid - 1`（目标在左半）

### 第三部分：简单算法证明题

**题目**：解释二分查找为什么要求数组有序

**答题要点**：

1. **可预测的缩小方向**：有序数组保证每次比较后，目标值要么在左半（如果目标小于中间值），要么在右半（如果目标大于中间值），从而每次可以排除一半搜索空间。

2. **决策树有效性**：二分查找对应的决策树是一棵平衡二叉搜索树，有序性是构造这种高效决策结构的前提。

3. **无序的失败案例**：如果数组无序，即使目标值小于中间值，也可能出现在右半部分，导致二分查找错误地排除包含目标的一半空间。

**零压力评分**：只要回答中包含"有序性保证搜索范围正确缩小"的核心概念即视为理解，无需完整证明。

## 🤔 今日疑问（1个疑问）

- 动态规划的状态定义和转移方程设计是否与回溯法有本质区别？如何在实际问题中识别可以使用DP的问题特征？

 **核心区别**

| 维度   | 回溯法              | 动态规划                      |
| ---- | ---------------- | ------------------------- |
| 本质   | 暴力搜索 + 剪枝        | 记录子问题最优解                  |
| 计算方式 | top-down（可能重复计算） | top-down(记忆化) 或 bottom-up |
| 目标   | 找所有解/一个解         | 找最优解                      |
| 复杂度  | 指数级（剪枝后仍可能很大）    | 多项式级（消除重复计算）              |
 *本质区别：重叠子问题的处理*

  回溯法：会重复计算相同的子问题
   例如：斐波那契数列的递归
   f(5) = f(4) + f(3)
   f(4) = f(3) + f(2)  ← f(3)被计算了2次
   f(3) = f(2) + f(1)  ← f(2)被计算了3次

  动态规划：记录已计算的结果，避免重复
   DP核心思想：用一个数组 memo[i] 记录 f(i) 的结果
   计算过的直接查表，没计算过的才递归

  *如何识别DP问题？*

  三个关键特征

   1. 最优子结构：问题的最优解可以由子问题的最优解构造
      - 例如：最短路径问题，最短路径的子路径也是最短的

   2. 重叠子问题：递归过程中会反复求解相同的子问题
      - 例如：回溯解法中有大量重复计算

   3. 无后效性：当前状态确定后，未来决策不受影响
      - 例如：到达某个位置的最短路径确定后，后续选择与此前的选择无关

  常见DP问题模式

   - 背包系列：0-1背包、完全背包、多重背包
   - 序列问题：最长递增子序列(LIS)、最长公共子序列(LCS)
   - 路径问题：网格路径、障碍物路径
   - 字符串问题：编辑距离、单词拆分
   - 计数问题：斐波那契、爬楼梯、硬币组合

  简单判断题

  > "这个问题如果用回溯法，会有很多重复计算吗？"

  如果答案是"是"，那么这个问题很可能可以用DP优化。

  ---

  举例说明：LeetCode 70 爬楼梯
   - 回溯法：枚举所有走法，指数级
   - DP：记录每个台阶的走法数，O(n)

  这就是典型的"重叠子问题"特征，DP可以将指数级优化到多项式级。
