### 模块1：状态压缩的核心思想——从二维DP到一维滚动数组

**为什么可以压缩？**
- 很多DP问题中，`dp[i]` 只依赖 `dp[i-1]`、`dp[i-2]` 等**常数个**历史状态
- 分析状态依赖关系：如果只需要保留最近2个状态，就不需要整个数组

**滚动数组（Rolling Array）原理**：
- 用变量替代数组：只保存计算时需要的那些历史状态
- 每轮迭代结束后，旧变量废弃，新变量作为下一轮的历史

**打家劫舍（198）为例**：
```cpp
int prev2 = 0, prev1 = 0;  // prev2 = dp[i-2], prev1 = dp[i-1]
for (int i = 0; i < n; ++i) {
    int curr = max(prev1, prev2 + nums[i]);
    prev2 = prev1;
    prev1 = curr;
}
```
- 核心转移：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`
- 只依赖 `dp[i-1]` 和 `dp[i-2]`，所以两个变量就够

### 模块2：环形数组问题——打家劫舍II（213）

**问题**：首尾房屋相邻，不能同时偷第一间和最后一间

**拆解策略**：把环形问题拆成两个线性子问题
1. **不偷第一间** → 在 `[0, n-2]` 范围内做普通打家劫舍
2. **不偷最后一间** → 在 `[1, n-1]` 范围内做普通打家劫舍

```cpp
int case1 = robRange(0, n - 2); // 拆头
int case2 = robRange(1, n - 1); // 拆尾
return max(case1, case2);
```

**覆盖性证明**：任意一种最优方案，要么不包含第一间，要么不包含最后一间（首尾相邻，但两者不能同时出现在最优解中）

**边界处理**：
- `n == 1`：只有一间，偷它
- `n == 2`：两间房不相邻（中间隔了至少一间），直接取 max
- `n >= 3`：正常走拆分逻辑

### 模块3：路径问题补充——63.不同路径II（滚动数组版）

**一维优化**：将二维 `dp[i][j]` 压缩为一维 `f[j]`
- `f[j]` = 到达当前行的第j列的路径数
- 每行从左到右遍历，`f[j]` 会被当前行的左侧值更新（原地）
- 遇障碍：`f[j] = 0`，直接跳过

```cpp
vector<int> f(n);
f[0] = (obstacleGrid[0][0] == 0);
for (int i = 0; i < m; ++i) {
    for (int j = 0; j < n; ++j) {
        if (obstacleGrid[i][j] == 1) { f[j] = 0; continue; }
        if (j > 0) f[j] += f[j-1];
    }
}
```

## 思考

1. **状态压缩的本质是什么？**
   - 分析「当前状态依赖几个历史状态」，如果只依赖常数个（如2个），就可以用变量替代数组
   - 从 O(n) 到 O(1)，时间不变，空间大幅降低

2. **环形问题为什么要拆成两个线性问题？**
   - 首尾相邻 = 不能同时出现在最优解中
   - 任意最优方案必然「不包含第一间」或「不包含最后一间」，两者其一必成立
   - 所以取两种情况的最大值即可，覆盖性毋庸置疑

3. **213 边界情况 n≤2 怎么处理？**
   - n=1：只有一间房，最优解就是它
   - n=2：两间房不相邻（中间隔了至少一间），直接取 max
   - n≥3：正常走拆分逻辑
