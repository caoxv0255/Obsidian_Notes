# 贡献与编辑指南

本仓库旨在维护结构化、可检索的教学与学习笔记。请遵循下列约定：

1. 目录与命名
   - 所有目录与文件名默认采用 `数字 + 下划线 + 英文标题` 的小写命名法（snake_case）。
   - 命名示例：`01_programming/`、`02_java/`、`00_index.md`、`01_basic_syntax.md`。
   - 每门课程目录应包含 `00_index.md`（可使用 `记录/course_index_template.md` 作为模板）。
   - 章节笔记建议以 `NN_topic_name.md` 命名（例如 `01_computer_abstraction.md`）。
   - 习题与实验目录建议使用 `02_problems/` 与 `03_labs/`，每题或每次实验为独立 Markdown 文件。

2. 新增学科笔记约定
   - 新增学科时，默认在 `按学科笔记/` 下创建一级目录，不在仓库根目录单独新建同名学科目录。
   - 新学科目录应包含 `00_index.md` 作为入口索引，并保持与现有学科一致的编号与命名风格。
   - 学科内的课程或主题应遵循相似体系架构：
     - `01_lecture_notes/`：课堂笔记
     - `02_problems/`：习题与解答
     - `03_labs/`：实验与动手练习说明（代码以片段形式嵌入）
     - `04_references/` 或 `05_refs/`：参考资料
   - 若学科已有成熟结构（例如章节文件已按教材组织），可在不破坏既有内容的前提下逐步对齐以上架构。

3. 模板使用
   - 所有课堂笔记使用 `记录/lecture_note_template.md`。
   - 习题使用 `记录/problem_template.md`，实验使用 `记录/lab_template.md`。

4. 语言与代码片段
   - 本次优先维护 C++ 与 Java 示例。代码应为片段形式嵌入 Markdown，不要添加可执行脚本或二进制文件。
   - 代码片段应加注释，保持精简并突出处理逻辑。

5. 标签与元数据
   - 建议在文件顶部使用 YAML front-matter（`title`, `course`, `chapter`, `date`, `tags`），便于检索与自动索引。

6. 链接与引用
   - 在 `00_index.md` 中使用相对链接引用章节/习题/实验文件，确保路径有效。

7. 校对与提交
   - 提交前请检查 Markdown 渲染（本地或 Obsidian），确认链接与代码块显示正常。
   - 提交说明请包含修改内容摘要与影响范围。

如有疑问，请在仓库根目录的 `README.md` 中添加讨论条目或联系维护者。
