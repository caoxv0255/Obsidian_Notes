# Build Your Own X 实践路线图

> 适用人群：C++基础 + 数据结构阶段（线性表、栈、队列）
> 目标：提高代码能力 + 理解底层原理

---

## 阶段一：入门实战（现在可以开始）

### 项目1：正则表达式引擎
**难度：** ⭐⭐
**预计时间：** 1-2周
**数据结构应用：** 字符串、栈、状态机

**为什么从这个开始：**
- 代码量小（核心逻辑100-200行）
- 直接应用你刚学的栈
- 理解"编译原理"的最小概念

**学习资源：**
- [Regular Expression Matching Can Be Simple And Fast (C)](https://www.cs.princeton.edu/courses/archive/spr09/cos333/beautiful.html)
- [Build Your Own Regular Expression Engines (Python参考，可转C++)](https://build-your-own.org/bison/)

**核心任务：**
1. 实现基础字符匹配（`a`匹配`a`）
2. 实现连接（`ab`匹配`ab`）
3. 实现或操作（`a|b`匹配`a`或`b`）
4. 实现闭包（`a*`匹配0或多个`a`）
5. 实现括号分组

**学到的底层原理：**
- NFA（非确定性有限自动机）
- DFA（确定性有限自动机）
- 状态机思想

---

### 项目2：文本编辑器（微型版）
**难度：** ⭐⭐
**预计时间：** 2-3周
**数据结构应用：** 字符串、链表、栈

**为什么推荐：**
- 直接应用链表（存储文本行）
- 应用栈（撤销/重做功能）
- 可视化成果强，有成就感

**学习资源：**
- [Build Your Own Text Editor (C)](https://viewsourcecode.org/snaptoken/kilo/)
- [Designing a Simple Text Editor (C++)](https://www.ics.uci.edu/~pattis/ICS-33/lectures/texteditor.txt)

**核心任务：**
1. 文本存储（每行一个字符串，用链表或vector管理）
2. 光标移动、插入、删除
3. 搜索功能
4. 撤销/重做（栈应用）
5. 保存/读取文件

**学到的底层原理：**
- 文本缓冲区设计（gap buffer / rope）
- 增量更新思想
- 用户输入处理

---

## 阶段二：进阶挑战（数据结构学完后）

### 项目3：简易数据库（Redis迷你版）
**难度：** ⭐⭐⭐
**预计时间：** 3-4周
**数据结构应用：** 哈希表、链表、跳表

**为什么重要：**
- 直接应用你学的数据结构
- 理解数据库最核心的存储原理
- 为后续学习打下基础

**学习资源：**
- [Build Your Own Redis from Scratch (C++)](https://build-your-own.org/)
- [Let's Build a Simple Database (C)](https://cstack.github.io/db_tutorial/)

**核心任务：**
1. 内存键值存储（哈希表）
2. 持久化（写入文件）
3. 简单命令解析（SET/GET/DEL）
4. 过期时间支持
5. 跳表实现有序集合（进阶）

**学到的底层原理：**
- 哈希表冲突处理
- 内存管理
- 序列化/反序列化
- 简单SQL解析

---

### 项目4：命令行Shell（mini版）
**难度：** ⭐⭐⭐
**预计时间：** 2-3周
**前置知识：** 操作系统基础概念

**为什么推荐：**
- 理解进程、管道、重定向
- 深入理解Linux命令
- 字符串解析能力提升

**学习资源：**
- [Tutorial - Write a Shell in C](https://brennan.io/2015/01/16/write-a-shell-in-c/)
- [Writing a UNIX Shell (C)](https://indradhanush.github.io/blog/writing-a-unix-shell-part-1/)

**核心任务：**
1. 命令解析（分割参数）
2. 执行外部命令（fork + exec）
3. 内置命令（cd、exit）
4. 管道实现（`|`）
5. 重定向实现（`>`、`<`）

**学到的底层原理：**
- 进程创建与管理
- 管道通信
- 文件描述符
- 信号处理

---

### 项目5：编程语言解释器（微型Lisp）
**难度：** ⭐⭐⭐
**预计时间：** 3-4周
**数据结构应用：** 树、栈、递归

**为什么有价值：**
- 理解编程语言本质
- 深刻理解递归
- 为后续学习编译原理打基础

**学习资源：**
- [Build Your Own Lisp (C)](https://buildyourownlisp.com/) ⭐ 强烈推荐
- [Kaleidoscope: Implementing a Language with LLVM (C++)](https://llvm.org/docs/tutorial/)

**核心任务：**
1. 词法分析（字符串→token）
2. 语法分析（token→抽象语法树）
3. 求值器（执行AST）
4. 环境与变量
5. 函数定义与调用

**学到的底层原理：**
- 词法分析、语法分析
- 抽象语法树（AST）
- 作用域与闭包
- 递归下降解析

## 学习建议

### 时间安排
```
阶段一（2-5周）：正则引擎 + 文本编辑器
阶段二（8-14周）：数据库 + Shell + 解释器
阶段三（6-12周）：操作系统/渲染器
阶段四（3-4周）：区块链
```

### 每个项目的标准流程
1. **先读懂原理**：看教程/文档，理解核心设计
2. **最小实现**：先跑通最核心功能
3. **增量迭代**：逐步添加新特性
4. **写文档**：记录设计决策和踩坑点
5. **测试驱动**：每个功能都要有测试用例

### 避坑指南
- 不要追求完美，先求有再求好
- 每个项目控制在 500-2000 行代码
- 遇到卡点看源码（GitHub上搜类似项目）
- 建议每个项目单独建一个 GitHub 仓库

## 辅助资源

### 必备工具
- **编译器**：g++ / clang++（支持C++17）
- **构建工具**：CMake
- **调试**：GDB / LLDB
- **代码规范**：clang-format
- **版本控制**：Git（你正在学）

### 推荐书籍
- 《C++ Primer》（查漏补缺）
- 《深度探索C++对象模型》（进阶）
- 《程序员的自我修养》（链接、加载原理）
- 《编译原理》（龙书，解释器项目参考）
