# 第11章：不完全信息

> 本章学习目标：
> - 理解贝叶斯纳什均衡

## 11.1 不完全信息博弈

### 11.1.1 不完全信息博弈的定义

**不完全信息博弈（Game of Incomplete Information）**：
参与人有私有信息（类型），但不知道其他参与人的类型。

**符号**：
- I：参与人集合
- Θᵢ：参与人i的类型空间
- θᵢ：参与人i的类型
- pᵢ：参与人i的信念（其他参与人类型的概率分布）

### 11.1.2 海萨尼转换

**海萨尼转换（Harsanyi Transformation）**：
将不完全信息博弈转化为不完美信息博弈。

**步骤**：
1. 引入自然（Nature）作为参与人0
2. 自然选择类型组合θ = (θ₁, ..., θₙ)
3. 参与人i观察到自己的类型θᵢ，但不知道其他参与人的类型θ₋ᵢ
4. 参与人i根据类型θᵢ选择策略sᵢ(θᵢ)

### 11.1.3 贝叶斯纳什均衡

**贝叶斯纳什均衡（Bayesian Nash Equilibrium, BNE）**：
策略组合s*是贝叶斯纳什均衡，如果对于所有i、所有θᵢ和所有sᵢ，有：
```
E_{θ₋ᵢ|θᵢ}[uᵢ(s*ᵢ(θᵢ), s*₋ᵢ(θ₋ᵢ), θᵢ, θ₋ᵢ)] ≥ E_{θ₋ᵢ|θᵢ}[uᵢ(sᵢ(θᵢ), s*₋ᵢ(θ₋ᵢ), θᵢ, θ₋ᵢ)]
```

**含义**：
- 给定自己的类型和其他参与人的策略，参与人选择最优策略
- 策略是类型的最优应对

## 11.2 信号传递

### 11.2.1 信号传递的定义

**信号传递（Signaling）**：
参与人通过行动传递私有信息。

**例子（Spence教育模型）**：
- 工人选择教育水平e
- 企业根据教育水平e支付工资w(e)
- 工人的类型：能力θ（高或低）
- 教育水平e传递能力θ的信息

### 11.2.2 信号传递模型

**信号传递模型（Signaling Model）**：
G = {I, (Θᵢ)ᵢ∈I, (Sᵢ)ᵢ∈I, (Aᵢ)ᵢ∈I, (pᵢ)ᵢ∈I, (uᵢ)ᵢ∈I}
- I：参与人集合
- Θᵢ：参与人i的类型空间
- Sᵢ：参与人i的信号空间
- Aᵢ：参与人i的行动空间
- pᵢ：参与人i的信念
- uᵢ：参与人i的收益函数

### 11.2.3 分离均衡

**分离均衡（Separating Equilibrium）**：
不同类型的参与人选择不同的信号。

**性质**：
- 信号完全揭示类型
- 后验信念：μ(θ|s) = 1或0

**例子（Spence模型）**：
- 高能力工人选择高教育水平e_H
- 低能力工人选择低教育水平e_L
- 企业根据教育水平推断能力

### 11.2.4 混同均衡

**混同均衡（Pooling Equilibrium）**：
不同类型的参与人选择相同的信号。

**性质**：
- 信号不揭示类型
- 后验信念：μ(θ|s) = 先验信念p(θ)

**例子（Spence模型）**：
- 高能力和低能力工人都选择相同的教育水平e*
- 企业无法根据教育水平区分能力
- 企业支付平均工资

### 11.2.5 混杂均衡

**混杂均衡（Hybrid Equilibrium）**：
某些类型的参与人选择相同的信号，其他类型的参与人选择不同的信号。

**性质**：
- 信号部分揭示类型
- 后验信念：μ(θ|s)介于0和1之间

## 11.3 信息筛选

### 11.3.1 信息筛选的定义

**信息筛选（Screening）**：
不知情的一方设计机制，使得知情的一方揭示信息。

**例子（保险市场）**：
- 保险公司设计不同的保险合同
- 投保人根据风险类型选择合同
- 合同设计使得高风险和低风险投保人选择不同的合同

### 11.3.2 信息筛选模型

**信息筛选模型（Screening Model）**：
G = {I, (Θᵢ)ᵢ∈I, (Cᵢ)ᵢ∈I, (Sᵢ)ᵢ∈I, (pᵢ)ᵢ∈I, (uᵢ)ᵢ∈I}
- I：参与人集合
- Θᵢ：参与人i的类型空间
- Cᵢ：参与人i的合同空间
- Sᵢ：参与人i的策略空间
- pᵢ：参与人i的信念
- uᵢ：参与人i的收益函数

### 11.3.3 自选择约束

**自选择约束（Self-Selection Constraints）**：
类型为θ的参与人选择为θ设计的合同，而不是为其他类型θ'设计的合同。

**公式**：
```
uᵢ(θ, c(θ)) ≥ uᵢ(θ, c(θ'))
```
**其中**：
- uᵢ(θ, c(θ))：类型为θ的参与人选择合同c(θ)的收益
- uᵢ(θ, c(θ'))：类型为θ的参与人选择合同c(θ')的收益

### 11.3.4 参与约束

**参与约束（Participation Constraints）**：
类型为θ的参与人选择合同c(θ)的收益不低于保留收益。

**公式**：
```
uᵢ(θ, c(θ)) ≥ uᵢ(θ, 0)
```
**其中**：
- uᵢ(θ, 0)：类型为θ的参与人的保留收益

### 11.3.5 最优筛选机制

**最优筛选机制（Optimal Screening Mechanism）**：
设计者设计合同c(θ)，使得目标函数最大化，满足自选择约束和参与约束。

**问题**：
```
max Σ_θ p(θ)·W(c(θ))
s.t. uᵢ(θ, c(θ)) ≥ uᵢ(θ, c(θ')), 对于所有θ, θ'
uᵢ(θ, c(θ)) ≥ uᵢ(θ, 0), 对于所有θ
```
**其中**：
- W(c(θ))：设计者在合同c(θ)下的收益
- p(θ)：类型θ的概率

## 11.4 机制设计

### 11.4.1 机制设计的定义

**机制设计（Mechanism Design）**：
设计者设计机制，使得参与人按照设计者的目标行动。

**符号**：
- I：参与人集合
- Θᵢ：参与人i的类型空间
- Mᵢ：参与人i的消息空间
- g：结果函数，g: M₁ × ... × Mₙ → X
- uᵢ：参与人i的收益函数

### 11.4.2 显示原理

**显示原理（Revelation Principle）**：
对于任何机制，存在一个直接机制，使得参与人如实报告类型是贝叶斯纳什均衡。

**含义**：
- 可以只考虑直接机制（参与人直接报告类型）
- 参与人有动力如实报告类型

### 11.4.3 直接机制

**直接机制（Direct Mechanism）**：
参与人直接报告类型，结果函数根据报告的类型确定结果。

**符号**：
- θᵢ'：参与人i报告的类型
- g(θ')：结果函数，根据报告的类型θ'确定结果

### 11.4.4 激励相容

**激励相容（Incentive Compatibility, IC）**：
参与人有动力如实报告类型。

**公式**：
```
E_{θ₋ᵢ|θᵢ}[uᵢ(θᵢ, g(θᵢ, θ₋ᵢ'))] ≥ E_{θ₋ᵢ|θᵢ}[uᵢ(θᵢ, g(θᵢ', θ₋ᵢ'))], 对于所有θᵢ, θᵢ'
```
**其中**：
- θᵢ：参与人i的真实类型
- θᵢ'：参与人i报告的类型
- g(θᵢ, θ₋ᵢ')：结果函数

### 11.4.5 最优机制

**最优机制（Optimal Mechanism）**：
设计者设计直接机制，使得目标函数最大化，满足激励相容和参与约束。

**问题**：
```
max E_θ[W(g(θ'))]
s.t. E_{θ₋ᵢ|θᵢ}[uᵢ(θᵢ, g(θᵢ, θ₋ᵢ'))] ≥ E_{θ₋ᵢ|θᵢ}[uᵢ(θᵢ, g(θᵢ', θ₋ᵢ'))], 对于所有θᵢ, θᵢ'
E_θ[uᵢ(θᵢ, g(θ'))] ≥ uᵢ(θᵢ, 0), 对于所有θᵢ
```
**其中**：
- W(g(θ'))：设计者在结果g(θ')下的收益
- uᵢ(θᵢ, 0)：参与人i的保留收益

## 11.5 本章总结

### 核心要点

1. **不完全信息博弈**：
   - 不完全信息博弈：参与人有私有信息，但不知道其他参与人的类型
   - 海萨尼转换：引入自然，将不完全信息博弈转化为不完美信息博弈
   - 贝叶斯纳什均衡：给定类型，策略是最佳应对

2. **信号传递**：
   - 信号传递：参与人通过行动传递私有信息
   - 分离均衡：不同类型的参与人选择不同的信号
   - 混同均衡：不同类型的参与人选择相同的信号
   - 混杂均衡：部分类型的参与人选择相同的信号

3. **信息筛选**：
   - 信息筛选：不知情的一方设计机制，使得知情的一方揭示信息
   - 自选择约束：类型为θ的参与人选择为θ设计的合同
   - 参与约束：类型为θ的参与人的收益不低于保留收益
   - 最优筛选机制：设计合同，使得目标函数最大化

4. **机制设计**：
   - 机制设计：设计者设计机制，使得参与人按照设计者的目标行动
   - 显示原理：存在直接机制，使得参与人如实报告类型
   - 直接机制：参与人直接报告类型
   - 激励相容：参与人有动力如实报告类型
   - 最优机制：设计直接机制，使得目标函数最大化
